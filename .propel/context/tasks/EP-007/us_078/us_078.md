# US-078: Build Audit Log Query Interface (Admin Only)

**Story ID:** US-078  
**Epic:** EP-007 (Compliance & Audit Logging)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UI / Backend Engineer  
**Status:** Draft

## User Story

As an **admin user**, I want an audit log query interface so that I can search, filter, and review audit records for compliance and investigation.

## Business Value

- Speeds security and compliance investigations
- Provides practical access to audit evidence
- Reduces manual retrieval work for admin users

## Scope

In scope:
- Admin-accessible audit log query UI or API
- Filtering by actor, action type, date range, resource, and result
- Pagination and sort capabilities
- Export support for compliance review

Out of scope:
- Full analytics or dashboard reporting on audit logs
- External audit system integration

## Acceptance Criteria

1. Given an Admin user, when accessing the audit query interface, then only authorized users can use it.
2. Given filter criteria, then the system returns matching audit records and supports pagination.
3. Given a record is selected, then detailed metadata is presented including actor, resource, action, timestamp, and outcome.
4. Given an export request, then the results can be downloaded as CSV or JSON.
5. Given an unauthorized user attempts access, then the system returns 403 Forbidden.

## Technical Notes

- Use secure query patterns to avoid exposing PHI in audit search results if not needed.
- Implement query performance patterns for large volumes of audit entries.
- Keep interface read-only.

## Dependencies

- Immutable audit log infrastructure from US-074.
- Admin role access from EP-005.

## Definition of Done

- Audit query interface available to Admin role only.
- Filtering, pagination, and export work correctly.
- Access restrictions tested.
- Security review completed.
