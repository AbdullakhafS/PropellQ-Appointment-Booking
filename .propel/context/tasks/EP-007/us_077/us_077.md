# US-077: Implement Log Integrity Checking (HMAC)

**Story ID:** US-077  
**Epic:** EP-007 (Compliance & Audit Logging)  
**Priority:** CRITICAL  
**Story Points:** 5  
**Owner Role:** Security / Data Engineer  
**Status:** Draft

## User Story

As a **compliance engineer**, I want audit logs to include integrity checks such as HMAC so that any tampering can be detected and log integrity can be proved.

## Business Value

- Protects audit records from undetected modification
- Strengthens evidence validity for compliance reviews
- Enables tamper-detection without requiring immutable storage alone

## Scope

In scope:
- HMAC or similar integrity verification mechanism for each audit record
- Periodic integrity validation checks
- Documentation of integrity verification process

Out of scope:
- Blockchain-based audit log storage
- External tamper-proof hardware modules

## Acceptance Criteria

1. Given an audit log entry, then an integrity hash is stored alongside it.
2. Given a log validation process, then tampering is detected if the stored hash does not match the log contents.
3. Given a collection of audit logs, then periodic checks confirm the integrity of stored entries.
4. Given a compliance review, then integrity-verification design and results are documented.

## Technical Notes

- Use a strong secret key for HMAC generation and protect the key securely.
- Consider chaining hash values for a tamper-evident sequence.
- Store integrity metadata separately from log content if possible.

## Dependencies

- Audit log storage infrastructure from US-074.
- Secure key management service.

## Definition of Done

- HMAC-based integrity checks implemented for audit logs.
- Periodic validation process in place.
- Integrity check failures are detectable and logged.
- Documentation updated.
