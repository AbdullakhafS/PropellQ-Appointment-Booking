# US-073: Implement Session Timeout (15 min inactivity)

**Story ID:** US-073  
**Epic:** EP-007 (Compliance & Audit Logging)  
**Priority:** HIGH  
**Story Points:** 3  
**Owner Role:** Authentication Engineer  
**Status:** Draft

## User Story

As a **security engineer**, I want user sessions to expire after 15 minutes of inactivity so that access is automatically revoked if a user leaves an unattended device.

## Business Value

- Reduces the risk of unauthorized access from inactive sessions
- Aligns session management with HIPAA security controls
- Improves overall platform security posture

## Scope

In scope:
- Inactivity-based session timeout policy implementation
- Session renewal on active use
- Frontend and backend support for session expiration handling

Out of scope:
- Persistent session duration beyond 15 minutes without activity
- MFA step-up flows for extended sessions

## Acceptance Criteria

1. Given a user session with no activity for 15 minutes, then the session expires and subsequent requests require re-authentication.
2. Given user activity within the timeout window, then the session is extended appropriately.
3. Given an expired session is used, then the system returns 401 Unauthorized.
4. Given a session timeout occurs, then the event is audit-logged.

## Technical Notes

- Use sliding expiration if active use should extend the session.
- Ensure session timeout is enforced server-side, not only in the browser.
- Align with token or cookie expiration policies.

## Dependencies

- Session and token management from EP-005.
- Audit logging from EP-007.

## Definition of Done

- 15-minute inactivity timeout enforced consistently.
- Expired sessions require re-authentication.
- Timeout events captured in audit logs.
- Tests validate both expiration and refresh behavior.
