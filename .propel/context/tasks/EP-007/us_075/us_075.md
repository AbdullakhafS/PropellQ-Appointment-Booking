# US-075: Log All User Actions (Login, Access, Changes)

**Story ID:** US-075  
**Epic:** EP-007 (Compliance & Audit Logging)  
**Priority:** CRITICAL  
**Story Points:** 5  
**Owner Role:** Backend / Security Engineer  
**Status:** Draft

## User Story

As a **security engineer**, I want all user actions related to login, data access, and configuration changes logged so that we can provide a complete audit trail for compliance and incident investigation.

## Business Value

- Creates a reliable forensic record for HIPAA audits
- Enables detection of suspicious or unauthorized activity
- Supports operational troubleshooting for security incidents

## Scope

In scope:
- Logging successful and failed login attempts
- Logging data access and modification actions for PHI-related resources
- Logging admin/user account changes and role updates
- Capturing metadata: timestamp, user ID, role, IP, action, resource

Out of scope:
- Detailed application debug logging not related to audit events

## Acceptance Criteria

1. Given a login attempt, then the system logs success or failure with user identity and source IP.
2. Given access or modification of PHI-related resources, then the action is logged with user, resource, and outcome.
3. Given a user account or role change, then the audit log records the actor, target user, and change details.
4. Given a log retrieval request, then audit records include sufficient metadata for investigation.
5. Given a compliance review, then the audit log coverage is documented and complete.

## Technical Notes

- Differentiate audit logs from standard application logs.
- Ensure sensitive values are not stored in audit entries.
- Use structured logging format for consistent storage and query.

## Dependencies

- Audit log infrastructure from US-074.
- User access controls from EP-005.

## Definition of Done

- Login, access, and change actions are logged comprehensively.
- Audit log entries contain required metadata.
- Security review verifies log coverage and privacy controls.
- Audit retrieval works for compliance queries.
