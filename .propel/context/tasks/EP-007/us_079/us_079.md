# US-079: Implement MFA Support (TOTP)

**Story ID:** US-079  
**Epic:** EP-007 (Compliance & Audit Logging)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** Security Engineer  
**Status:** Draft

## User Story

As a **security-conscious user**, I want multi-factor authentication using TOTP so that staff and admin accounts have an additional layer of protection beyond passwords.

## Business Value

- Strengthens access security for high-risk roles
- Helps satisfy HIPAA strong authentication requirements
- Reduces account takeover risk

## Scope

In scope:
- TOTP enrollment and verification flows
- Backup codes or recovery options
- Enforcement policies for Staff and Admin users

Out of scope:
- Passwordless or biometric authentication
- MFA for patient self-service accounts in Phase 1

## Acceptance Criteria

1. Given a Staff or Admin user, when they enroll in MFA, then they receive TOTP configuration instructions.
2. Given a valid TOTP code, then login proceeds successfully.
3. Given an invalid TOTP code, then login is rejected with a clear error.
4. Given backup codes are used, then they are single-use and securely managed.
5. Given a policy requires MFA, then users cannot continue login without completing MFA setup.

## Technical Notes

- Use standard TOTP algorithms and secure enrollment flows.
- Protect MFA secrets in secure storage.
- Provide a recovery path for lost devices.

## Dependencies

- User authentication infrastructure from EP-005.
- Secure secret storage and user account management.

## Definition of Done

- TOTP MFA available and enforced for required roles.
- Enrollment and login flows validated.
- Backup/recovery options tested.
- Security review completed.
