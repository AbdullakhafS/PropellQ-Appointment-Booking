# Task 004: Define Question Phrasing & Create A/B Test Variants

**Task ID:** task_012_004  
**User Story:** US-012 (Design Multi-Step Chatbot Conversation Flow)  
**Epic:** EP-002  
**Priority:** MEDIUM  
**Story Points:** 2  
**Owner Role:** Product / Content Designer  
**Status:** Not Started

## Objective

Define clear, patient-friendly question phrasing for each conversation stage and create A/B test variants to validate which wording resonates with patients.

## Requirements

1. **Question Phrasing Library**
   - Define primary phrasing for each conversation stage (7 stages)
   - Create 2-3 alternative phrasings for key questions
   - Base on user research findings (task_012_001) for tone and clarity
   - Avoid medical jargon; use plain language alternatives
   - Include rationale for each phrasing choice

2. **A/B Test Candidates**
   - Chief Complaint: "What brings you in today?" vs. "What's bothering you?" vs. "What's your main concern?"
   - Medical History: "Do you have any chronic conditions?" vs. "Are you being treated for any long-term health issues?" vs. "Any ongoing health problems?"
   - Medications: "What medications are you currently taking?" vs. "What pills or inhalers do you use?" vs. "Tell me about your current medications"
   - Allergies: "Do you have any drug allergies?" vs. "Are you allergic to any medications?" vs. "Tell me about any medication allergies"
   - Insurance: "Do you have insurance?" vs. "Are you insured?" vs. "Do you have health insurance?"

3. **Rationale & Guidelines**
   - Explain why each phrasing was chosen
   - Document tone guidelines (warm, professional, not condescending)
   - Provide context rules (when to use variant A vs. B based on patient profile)
   - Include fallback phrasings for confusion scenarios

4. **Follow-Up Question Phrasing**
   - Define follow-up questions for each stage
   - Create clarification prompts (if patient doesn't understand)
   - Define skip/fallback prompts ("Would you like to skip this or switch to the form?")

5. **Confirmation Language**
   - Summary statement: "Here's what I captured: [data]"
   - Confirmation question: "Does this look correct?"
   - Edit prompt: "What would you like to change?"
   - Completion message: "Thanks! Your intake is complete."

## Implementation Steps

1. Review user research findings for phrasing preferences
2. Draft primary phrasings for all 7 conversation stages
3. Create 2-3 variant phrasings for high-impact questions (chief complaint, medical history, meds, allergies)
4. Document rationale for each variant
5. Create A/B test plan: which variants to test, sample size, success metrics
6. Define tone guidelines and example responses
7. Create fallback/clarification prompts for confusion scenarios
8. Document final question library in spec document
9. Get product and design review
10. Finalize for implementation
