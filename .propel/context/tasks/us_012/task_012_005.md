# Task 005: Document Branching Logic & Decision Matrix

**Task ID:** task_012_005  
**User Story:** US-012 (Design Multi-Step Chatbot Conversation Flow)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Product / Systems Designer  
**Status:** Not Started

## Objective

Document all branching logic and conditional paths in a decision matrix so that developers can implement the flow correctly and predictably.

## Requirements

1. **Decision Matrix**
   - Create decision table for each conversation stage
   - Define condition (if X) → action (then Y) for all branches
   - Document fallback paths (else Z)
   - Example: IF patient_mentions_chest_pain THEN ask_severity_and_duration; ELSE ask_next_main_question

2. **Branching Scenarios**
   - Chief Complaint branches:
     - IF symptom_detected THEN ask follow-up severity/duration
     - IF no_clear_symptom THEN ask for clarification (up to 2x, then skip)
   - Medical History branches:
     - FOR EACH condition_mentioned: ask medication_for_condition
     - IF no_conditions THEN skip to medications stage
   - Medications branches:
     - FOR EACH medication: ask_dosage_and_frequency
     - IF_medication_allergenic_to_known_allergy: flag_cross_reactivity
   - Allergies branches:
     - IF penicillin_allergy THEN flag_amoxicillin_and_other_beta_lactams
     - IF multiple_allergies THEN collect_all_before_moving_to_insurance
   - Insurance branches:
     - IF insurance_yes THEN ask_carrier_name_then_member_id
     - IF insurance_no THEN skip_to_summary

3. **Context Tracking**
   - Document what conversational context needs to be tracked between turns
   - Example: chief_complaint, symptoms_mentioned, conditions, medications, allergies, insurance_status
   - Define how context is carried forward to inform follow-up questions

4. **Error/Fallback Paths**
   - IF patient_response_unclear THEN ask_clarification (up to 2 attempts)
   - IF patient_still_unclear THEN offer: "Skip this question" or "Switch to manual form"
   - IF patient_frustrated THEN offer skip/form switch earlier
   - Document fallback responses for each stage

5. **Conversation State Machine**
   - Define conversation states: GREETING, CHIEF_COMPLAINT, HISTORY, MEDICATIONS, ALLERGIES, INSURANCE, SUMMARY, COMPLETE
   - Document valid transitions between states
   - Document state timeouts (if conversation inactive >5 min, offer to resume)

6. **Edge Cases**
   - Patient mentions allergy and then medication allergenic to that allergy
   - Patient lists >10 medications
   - Patient skips medical history but mentions medication
   - Patient switches from AI to manual mid-conversation

## Implementation Steps

1. Review conversation flow design from task_012_002
2. Extract all branching scenarios
3. Create decision table in spreadsheet or structured format
4. Define state machine diagram
5. Document all edge cases and how to handle
6. Create context tracking requirements list
7. Document fallback/error paths
8. Get product and tech review
9. Finalize decision matrix
10. Create implementation guide for developers
