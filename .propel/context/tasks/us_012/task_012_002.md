# Task 002: Design Conversation Flow & Create Flow Diagram

**Task ID:** task_012_002  
**User Story:** US-012 (Design Multi-Step Chatbot Conversation Flow)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 3  
**Owner Role:** Product / UX Designer  
**Status:** Not Started

## Objective

Design the complete chatbot conversation flow with all conversation stages, questions, branching logic, and transitions documented in a flowchart and specifications document.

## Requirements

1. **Conversation Stages (7 stages)**
   - Stage 1: Greeting (introduce chatbot, set time expectation)
   - Stage 2: Chief Complaint (open-ended, symptom capture)
   - Stage 3: Medical History (chronic conditions, past events)
   - Stage 4: Current Medications (medication list with dosage/frequency)
   - Stage 5: Allergies (drug allergies, reactions, side effects)
   - Stage 6: Insurance (carrier, member ID, group number)
   - Stage 7: Summary & Confirmation (review, edit, confirm)

2. **Branching Logic**
   - If patient mentions specific symptom → ask severity/duration follow-ups
   - If patient mentions condition → ask medication for that condition
   - If patient has no conditions → skip condition-specific medication follow-ups
   - If patient allergic to medication class → flag similar medications
   - If insurance "yes" → collect carrier and ID; if "no" → skip to summary

3. **Question Library**
   - Main question + follow-up questions for each stage
   - Alternate phrasings for A/B testing (2-3 variants per key question)
   - Fallback responses (if patient unclear, offer clarification or skip)
   - Conditional logic for question sequencing

4. **Flowchart & Visual Documentation**
   - Create BPMN or flowchart diagram showing all stages and branches
   - Document decision points and conditions for branching
   - Include error/fallback paths (patient confused, wants to skip, wants manual form)
   - Show progress indicator touchpoints ("Step 2 of 7")

5. **Conversation Tone & Personality**
   - Define chatbot voice: professional but warm, empathetic, non-judgmental
   - Create tone guidelines: avoid medical jargon (use plain language alternatives)
   - Example responses for different scenarios (patient hesitant, rushed, confused)

6. **Timing & Pacing**
   - Target 5-7 minutes total conversation time
   - Allocate time per stage: greeting (30s), chief complaint (1-2m), history (1-2m), meds (1m), allergies (1m), insurance (1m), summary (1m)
   - Identify optimization opportunities if flow exceeds 7 minutes

## Implementation Steps

1. Review user research findings from task_012_001
2. Draft conversation flow in pseudo-code or outline format
3. Identify all branching scenarios and decision points
4. Create branching logic matrix (if X then Y table)
5. Create flowchart in Figma, Miro, or draw.io
6. Define question library with main + alternate phrasings
7. Document tone guidelines and example responses
8. Calculate estimated conversation time per stage
9. Review with product manager and get feedback
10. Document final flow in specification document

## Acceptance Criteria

- [ ] Flowchart created and covers happy path + 3+ fallback paths
- [ ] Tone guidelines documented with example responses
- [ ] Estimated total conversation time is 5-7 minutes
- [ ] Conditional logic matrix created and reviewed
- [ ] Fallback/skip options defined for each stage
- [ ] Progress indicator touchpoints documented
