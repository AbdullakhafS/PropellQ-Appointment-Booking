# Task 004: Implement Form Validation & Error Handling

**Task ID:** task_013_004  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Frontend Developer (React)  
**Status:** Not Started

## Objective

Implement submit-time form validation with clear error messages, preventing submission if required fields are empty.

## Requirements

1. **Validation Rules**
   - Chief Complaint: Required (min 3 characters, max 500)
   - Medical History: Optional (at least one condition OR other text if section is filled)
   - Medications: Optional (all fields optional; if one field filled, name is required)
   - Allergies: Optional (all fields optional; if one field filled, name and type are required)
   - Insurance: Optional (if insurance selected, Name is required)

2. **Validation Triggers**
   - Validation runs on form submit only (not on blur or change)
   - No inline red error messages while typing
   - On submit attempt with errors: Prevent submission, scroll to first error, display error summary

3. **Error Display**
   - Error summary box at top of form: "Please fix 3 errors before submitting"
   - Below each field with error: Error message text
   - Error message color: Red (#d32f2f) with clear icon
   - Error message size: Readable (16px+)

4. **Error Messages**
   - "Chief complaint is required" (if empty)
   - "Chief complaint must be at least 3 characters" (if too short)
   - "Chief complaint must not exceed 500 characters" (if too long)
   - "At least one medication name is required" (if medication row missing name)
   - "At least one allergy name is required" (if allergy row missing name)
   - "Insurance provider is required" (if insurance selected but name empty)

5. **Success Handling**
   - If validation passes: Show loading state ("Submitting...")
   - Disable submit button during submission
   - Submit form data to backend API

6. **Field-Level Validation**
   - Use react-hook-form's `register()` with validation rules
   - Character count validation: Chief complaint max 500 chars
   - Required field validation: Only chief complaint required
   - Custom validation: Check for duplicate medications (warn but allow)

7. **Error Recovery**
   - User can fix errors and resubmit
   - Error messages clear when field is corrected (optional: clear on edit)
   - Focus management: After error, focus moved to first error field

8. **Accessibility**
   - Error summary box has `role="alert"` and `aria-live="polite"`
   - Each error message linked to field via `aria-describedby`
   - Form announce errors to screen readers: "Form has 3 errors"
   - Error message text is clear and actionable (not just "Error")

## Implementation Steps

1. Define validation schema with react-hook-form or Zod
2. Implement Chief Complaint validation (required, min 3, max 500)
3. Implement Medications validation (optional but name required if present)
4. Implement Allergies validation (optional but name required if present)
5. Implement Insurance validation (name required if insured)
6. Create ErrorSummary component for displaying errors
7. Create submit handler with validation check
8. Implement error display below each field
9. Implement error focus management (scroll to first error)
10. Add loading state during submission
11. Disable submit button during submission
12. Test validation with various error scenarios
13. Write unit tests for validation logic
