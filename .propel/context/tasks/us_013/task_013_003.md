# Task 003: Implement Repeating Fields (Medications & Allergies) with Add/Delete

**Task ID:** task_013_003  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Frontend Developer (React)  
**Status:** Not Started

## Objective

Implement dynamic repeating fields for Medications and Allergies sections with add/delete functionality using react-hook-form `useFieldArray` hook.

## Requirements

1. **Medications Section Implementation**
   - Use `useFieldArray` to manage medications array
   - Repeating field structure per medication:
     - Medication Name: Text input (required)
     - Dosage: Text input (e.g., "500mg", "2 tablets")
     - Frequency: Text input (e.g., "2x daily", "once at bedtime")
     - Delete Button: Remove this medication from list
   - "Add Medication" button at bottom of section
   - Minimum of 0 medications (all optional); no pre-filled defaults

2. **Allergies Section Implementation**
   - Use `useFieldArray` to manage allergies array
   - Repeating field structure per allergy:
     - Allergy Name: Text input (required) or dropdown with common allergies
     - Reaction Type: Dropdown ["Allergy", "Side Effect"]
     - Reaction Description: Text area (e.g., "causes rash", "upset stomach")
     - Delete Button: Remove this allergy from list
   - "Add Allergy" button at bottom of section

3. **Add Button Behavior**
   - Clicking "Add Medication" appends new empty row to medications array
   - New row has focus on first input (Medication Name)
   - Clicking "Add Allergy" appends new empty row to allergies array
   - Buttons text: "Add Medication" / "Add Allergy"
   - Buttons are touch-friendly (44px+ height)

4. **Delete Button Behavior**
   - Each row has delete button (trash icon or "Remove" text)
   - Clicking delete removes row immediately (no confirmation, but can undo if needed)
   - Delete button only shows if more than 0 items exist (optional: show always)
   - Delete button is touch-friendly and has clear hover state

5. **Field Validation (Submit-Time)**
   - Medication Name required if any medication in list
   - Allergy Name required if any allergy in list
   - No inline validation; only validate on form submit

6. **Keyboard Navigation**
   - Tab key navigates through all fields including add/delete buttons
   - Enter key on "Add" button adds new row
   - Delete button accessible via Tab and keyboard
   - Focus management: After deleting a row, focus moves to delete button of previous row (or add button if last row deleted)

7. **Mobile Responsiveness**
   - Repeating fields stack vertically on mobile (375px)
   - Each medication/allergy row is full width
   - Delete button positioned consistently (right side or below)
   - Add button is full width on mobile for easy tapping

8. **Accessibility**
   - Each repeating field has aria-label or aria-describedby
   - Remove/Delete buttons have aria-label: "Remove medication 1 of 3" (indicates position)
   - Add buttons labeled: "Add another medication" / "Add another allergy"
   - Fieldset and legend elements for logical grouping (optional but recommended)

## Implementation Steps

1. Import `useFieldArray` from react-hook-form
2. Initialize fieldArray for medications in form context
3. Create MedicationField component (single medication row)
4. Create "Add Medication" button with append logic
5. Create delete button with remove logic
6. Implement keyboard navigation (Tab, Enter, focus management)
7. Implement accessibility labels and descriptions
8. Repeat for Allergies section (reuse MedicationField pattern)
9. Add responsive CSS for mobile layout
10. Test add/delete functionality (add 5, delete middle one, verify array order)
11. Test keyboard navigation (Tab through all rows, arrow keys, Enter)
12. Write unit tests for fieldArray add/delete/reorder operations
