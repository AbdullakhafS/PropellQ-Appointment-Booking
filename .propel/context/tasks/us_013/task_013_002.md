# Task 002: Build Form Component Structure & React Hook Form Integration

**Task ID:** task_013_002  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 3  
**Owner Role:** Frontend Developer (React)  
**Status:** Not Started

## Objective

Implement core form component structure with React Hook Form for state management, supporting all 5 form sections (chief complaint, medical history, medications, allergies, insurance).

## Requirements

1. **Form Structure**
   - 5 main sections: Chief Complaint, Medical History, Medications, Allergies, Insurance
   - Section layout: Accordion or tabs (choose based on UX design)
   - Sticky submit button at bottom (desktop and mobile)
   - "Back" and "Next" navigation buttons between sections (if step-by-step flow)

2. **React Hook Form Integration**
   - Use `react-hook-form` for form state management
   - Initialize form with default values (empty or auto-populated)
   - Implement watch/subscribe pattern for conditional rendering
   - No validation errors on page load (validation on submit only)

3. **Chief Complaint Section**
   - Text input field: `maxLength="500"`
   - Label: "What brings you in today?"
   - Helper text: "(Required)"
   - Character counter: "500 characters remaining"

4. **Medical History Section**
   - Checkbox list: Common conditions (diabetes, asthma, hypertension, heart disease, arthritis, etc.)
   - Text area: "Other conditions" field for free-text entry
   - Label: "Do you have any chronic conditions?"
   - Multiple checkboxes can be selected

5. **Medications Section**
   - Repeating fields (see task_013_003 for detailed implementation)
   - Placeholder for now: Text showing "Medications will be managed in separate task"
   - Column structure: [Medication Name] [Dosage] [Frequency] [Delete]

6. **Allergies Section**
   - Checkbox list: Common drug allergies (Penicillin, Sulfa, NSAIDs, etc.)
   - Checkboxes for "Reaction type": Allergies vs. Side Effects (differentiator)
   - Text area for reaction description: "What happens when you take this?"
   - Can add multiple allergies

7. **Insurance Section**
   - Text inputs: Insurance Name, Member ID, Group Number, Plan Name
   - Label: "Do you have insurance?"
   - Option: Checkbox to indicate "No insurance" (skips fields if unchecked)

8. **Form Accessibility**
   - All form fields have associated `<label>` elements with `htmlFor`
   - Error messages linked via `aria-describedby`
   - Required fields marked with asterisk and aria-required="true"
   - Form has role="form" and aria-label="Patient Intake Form"
   - Submit button has aria-label="Submit intake form"

9. **Responsive CSS Structure**
   - Desktop (1024px+): 2-column layout where applicable
   - Tablet (768px): 1.5-column or 1-column
   - Mobile (375px+): Single column; all inputs full width
   - Minimum input height: 44px (touch-friendly on mobile)

10. **State Management**
    - Form state centralized with react-hook-form
    - Component structure: IntakeForm (parent) → FormSections (children)
    - Props: initialData, onSubmit, isLoading
    - No global state yet (simple local component state)

## Implementation Steps

1. Create IntakeForm component with react-hook-form setup
2. Create FormSection component (accordion/tab wrapper)
3. Implement Chief Complaint section with character counter
4. Implement Medical History section with checkboxes
5. Create placeholder for Medications section (to be implemented in task_013_003)
6. Create placeholder for Allergies section (to be implemented in task_013_003)
7. Implement Insurance section
8. Implement sticky submit button
9. Add accessibility attributes and labels
10. Implement responsive CSS (mobile-first approach)
11. Implement error message display (for submit-time errors)
12. Create Storybook stories for each section (for isolated testing)
13. Write component unit tests
