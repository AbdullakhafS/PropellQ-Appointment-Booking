## Objective

Design and implement database schema for storing chatbot conversations, extracted data, and metadata with proper indexing and audit trail support.

## Requirements

1. **IntakeConversations Table**
   - Store conversation metadata: appointment_id, patient_id, mode (ai/manual), timestamps
   - Store conversation transcript as JSON (array of messages with timestamps)
   - Store extracted structured data: chief_complaint, medical_history, medications, allergies
   - Store confidence scores for each extracted field
   - Track conversation status: in_progress, completed, switched_to_manual, abandoned

2. **ChatbotPrompts Table**
   - Versioned system prompts for A/B testing and audit trail
   - Tracks effective date and deprecation date
   - Allows rollback to previous prompt versions

3. **Indexes & Performance**
   - Primary key on id
   - Foreign keys: appointment_id, patient_id
   - Indexes on: patient_id, appointment_id, created_at (for pagination)
   - Covering index on (patient_id, created_at) for quick retrieval

4. **Data Retention & Compliance**
   - Conversation transcripts encrypted at rest (AES-256)
   - JSON data validated on insert
   - Audit logging for schema changes

## Implementation Steps

1. Write SQL migration script to create IntakeConversations table
2. Write SQL migration script to create ChatbotPrompts table
3. Create indexes for performance
4. Create ORM models/entities (C# DbContext or equivalent)
5. Implement data access layer for CRUD operations
6. Add encryption for sensitive fields (if needed)
7. Write schema documentation
8. Create rollback migration scripts
