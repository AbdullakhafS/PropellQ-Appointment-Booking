# Task 006: Implement Conversation Persistence & Audit Logging

**Task ID:** task_011_006  
**User Story:** US-011 (Implement AI Chatbot Engine)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Backend / Data Engineer  
**Status:** Not Started

## Objective

Implement conversation persistence to database, audit logging for compliance, conversation resumption, and data cleanup policies.

## Requirements

1. **Conversation Persistence**
   - Save each Q&A pair to IntakeConversations table immediately after exchange
   - Store conversation_transcript as JSON array with timestamps
   - Store extracted_data and confidence_scores from each turn
   - Track conversation state and status
   - Support incremental saves (append-only for audit trail)

2. **Conversation Resumption**
   - Retrieve conversation history on user return
   - Display previous messages in chatbot UI
   - Allow user to continue or start fresh
   - Track resumption timestamp and count

3. **Audit Logging**
   - Log all API calls with: timestamp, user_id, patient_id, appointment_id, message, response
   - Include request/response sizes and latency
   - Store correlation IDs for tracing
   - Implement immutable audit log (no updates, only inserts)
   - Encrypt sensitive data in logs (PII redaction)

4. **Data Retention & Cleanup**
   - Define retention policy: 7 years for audit, 2 years for transcripts (configurable)
   - Implement archival for old conversations
   - Implement secure deletion with audit trail
   - Handle GDPR right-to-be-forgotten requests

5. **Error Recovery**
   - Handle partial saves if API fails mid-conversation
   - Provide idempotent save operations (retry-safe)
   - Clean up orphaned conversation records

## Implementation Steps

1. Create data access layer for IntakeConversations table
2. Implement conversation save logic (incremental, append-only)
3. Create conversation retrieval and resumption flow
4. Implement audit logging service
5. Add PII detection and redaction for logs
6. Create conversation archival process
7. Implement deletion/anonymization for data retention
8. Add error handling and recovery
9. Write unit tests for persistence and logging
10. Create data retention documentation
