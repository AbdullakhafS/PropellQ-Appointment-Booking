# Task 003: Implement Chatbot System Prompt & Conversation Flow

**Task ID:** task_011_003  
**User Story:** US-011 (Implement AI Chatbot Engine)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 3  
**Owner Role:** Product / Platform Engineer  
**Status:** Not Started

## Objective

Engineer and test the system prompt for medical intake chatbot, define conversation flow, implement conversation state management, and validate prompt effectiveness.

## Requirements

1. **System Prompt Engineering**
   - Create base system prompt for medical intake context
   - Define chatbot persona: empathetic, professional, non-judgmental
   - Include guidelines: ask one question at a time, listen for follow-ups, avoid medical advice
   - Define structured data extraction format
   - Include fallback/clarification prompts

2. **Conversation Flow Design**
   - Define initial greeting with patient name
   - Chief complaint: "What brings you in today?" with follow-ups for severity, duration
   - Medical history: Chronic conditions, past surgeries, relevant medical events
   - Medications: Current prescriptions with dosages; over-the-counter meds
   - Allergies: Drug allergies vs. side effects distinction; cross-reactivity detection
   - Insurance: Basic info collection (optional in Phase 1)
   - Completion: Summary review and confirmation

3. **Conversation State Management**
   - Track conversation state: greeting, chief_complaint, medical_history, medications, allergies, insurance, summary, complete
   - Store context between turns (patient name, previously mentioned conditions, medications)
   - Implement conversation resumption for abandoned sessions

4. **Confidence Scoring & Data Extraction**
   - Define confidence thresholds for extracted fields (high: >85%, medium: 70-85%, low: <70%)
   - Implement tagging/classification for extracted data
   - Flag low-confidence extractions for manual review

5. **Misunderstanding Handling**
   - Define re-attempt logic: up to 2 clarification attempts
   - Implement fallback to manual intake if chatbot stuck

## Implementation Steps

1. Write and version system prompt in ChatbotPrompts table
2. Implement conversation state machine
3. Create conversation flow configuration (questions, transitions)
4. Implement context tracking and conversation history management
5. Create prompt templating system (inject context into prompts)
6. Implement structured data extraction from LLM responses
7. Add confidence scoring logic
8. Create misunderstanding detection and handling logic
9. Write unit tests for state transitions and data extraction
