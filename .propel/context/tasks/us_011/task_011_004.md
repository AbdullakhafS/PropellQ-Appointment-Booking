# Task 004: Build Chatbot UI Component (Frontend)

**Task ID:** task_011_004  
**User Story:** US-011 (Implement AI Chatbot Engine)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 3  
**Owner Role:** Frontend / UX Engineer  
**Status:** Not Started

## Objective

Implement React chatbot component with message display, input field, conversation history, loading states, and integration with backend chat API.

## Requirements

1. **Chatbot UI Component**
   - Message list displaying conversation history (user and assistant messages)
   - Timestamp for each message
   - Message styling: assistant (left-aligned, light background), user (right-aligned, blue background)
   - Input field with placeholder text: "Type your response here..."
   - Send button (keyboard Enter support)

2. **Conversation Display**
   - Scroll to latest message automatically (virtual scrolling for long conversations)
   - Loading indicator while waiting for chatbot response
   - Typing indicator (animated dots) while generating response
   - Error messages displayed inline

3. **Summary View**
   - Completion screen showing extracted data summary
   - Edit buttons for each field (optional for Phase 1)
   - Confirm/Submit button to save intake data

4. **State Management**
   - Loading state during API call
   - Error state with retry option
   - Conversation state: in_progress, completed, error
   - Fallback button: "Switch to manual intake form"

5. **Accessibility & Mobile**
   - Keyboard navigation support
   - ARIA labels for interactive elements
   - Mobile-responsive: works on 375px (phone) and 768px (tablet)
   - Touch-friendly button sizes (48px minimum)

6. **User Feedback**
   - Show response time feedback ("Processing..." / "Done")
   - Toast notifications for errors
   - Confirmation dialog before submitting incomplete intake

## Implementation Steps

1. Create React Chatbot component with hooks (useState, useEffect, useRef)
2. Implement message list with auto-scroll
3. Create message input component with keyboard support
4. Integrate with POST `/api/intake/chat` endpoint
5. Add loading and error states
6. Implement conversation history pagination
7. Create summary/confirmation view
8. Add accessibility attributes (ARIA, semantic HTML)
9. Implement mobile responsiveness
10. Add unit tests for component behavior
