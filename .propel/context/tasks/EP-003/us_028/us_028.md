# US-028: Build Code Verification UI (Accept/Reject/Override)

**Story ID:** US-028  
**Epic:** EP-003 (Clinical Data Intelligence & Coding)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Frontend Engineer  
**Status:** Draft

## User Story

As a **coding reviewer**, I want a verification UI for AI-suggested codes so that I can accept, reject, or override code suggestions efficiently.

## Business Value

- Improves coder productivity and accuracy
- Ensures human review of AI-suggested codes
- Reduces downstream billing and compliance errors

## Scope

In scope:
- UI for reviewing ICD-10 and CPT suggestions
- Actions for accept, reject, and override
- Display of confidence scores and supporting evidence

Out of scope:
- Full payer coding rules engine

## Acceptance Criteria

1. Given suggested codes, then the reviewer can accept, reject, or override each one.
2. Given a suggestion, then the UI shows confidence score and source evidence.
3. Given an override, then the reviewer’s choice is recorded and audit-logged.
4. Given rejected suggestions, then the system retains them for audit and feedback.

## Technical Notes

- Use pithy UI patterns for code review workflows.
- Ensure all review actions are audit-loggable.
- Persist reviewer decisions with metadata.

## Dependencies

- ICD-10 and CPT suggestion engines from US-026 and US-027.
- Audit logging framework from EP-007.

## Definition of Done

- Verification UI implemented and accessible.
- Accept/reject/override actions persist correctly.
- Audit logging covers review actions.
- Security and UI accessibility validated.
