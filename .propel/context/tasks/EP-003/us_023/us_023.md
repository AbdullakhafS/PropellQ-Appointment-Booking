# US-023: Display Document Sources with Traceability

**Story ID:** US-023  
**Epic:** EP-003 (Clinical Data Intelligence & Coding)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Data Engineer  
**Status:** Draft

## User Story

As a **clinical reviewer**, I want to see the source documents behind extracted data so that I can verify and trust the information in the patient profile.

## Business Value

- Improves confidence in AI-extracted data
- Supports clinical review and auditability
- Provides transparency into the source of diagnoses and medications

## Scope

In scope:
- Linking profile data elements to source documents or intake entries
- Displaying source metadata such as document name, type, and extraction confidence
- Allowing users to open or preview source documents securely

Out of scope:
- Document editing or annotation features

## Acceptance Criteria

1. Given an extracted data item, then the UI shows the source document or intake source.
2. Given source metadata, then document name, source type, and extraction confidence are visible.
3. Given a user clicks a source link, then they can preview or access the source document securely.
4. Given the source is intake data, then the intake entry and timestamp are shown.

## Technical Notes

- Secure source access with authorization checks.
- Store document source references and metadata with extracted data.
- Use secure preview mechanisms to avoid PHI leakage.

## Dependencies

- Document processing pipeline from US-020.
- Aggregated profile data from US-019.

## Definition of Done

- Source traceability implemented in profile UI.
- Metadata displayed for each data item.
- Secure preview or access flow works.
- Tests verify source links and permissions.
