# US-021: Extract Structured Data from PDF/DOCX

**Story ID:** US-021  
**Epic:** EP-003 (Clinical Data Intelligence & Coding)  
**Priority:** CRITICAL  
**Story Points:** 5  
**Owner Role:** AI / Document Extraction Engineer  
**Status:** Draft

## User Story

As a **clinical data processor**, I want structured data extracted from uploaded PDF and DOCX documents so that the system can use it for patient profile and coding workflows.

## Business Value

- Turns unstructured documents into actionable clinical data
- Reduces manual review time for staff
- Enables downstream intelligence and alerting

## Scope

In scope:
- Extraction of key structured entities from uploaded documents
- Detection of medications, allergies, diagnoses, and dates
- Confidence score attached to extracted elements

Out of scope:
- Full human-in-the-loop validation UI

## Acceptance Criteria

1. Given a processed document, then structured fields are extracted and stored.
2. Given extracted data, then each element includes a confidence score.
3. Given an extraction error, then the document is marked for review and the failure reason is recorded.
4. Given document metadata, then extraction source and timestamps are preserved.

## Technical Notes

- Use external AI/NLP services or rule-based extraction as appropriate.
- Return extracted entities in a normalized format for profile ingestion.
- Preserve original text references for auditability.

## Dependencies

- Document upload pipeline from US-020.
- AI or extraction service integration.

## Definition of Done

- Structured extraction works for PDF and DOCX documents.
- Confidence scores are stored with extracted data.
- Extraction errors are handled and tracked.
- Tests validate structured output and failure handling.
