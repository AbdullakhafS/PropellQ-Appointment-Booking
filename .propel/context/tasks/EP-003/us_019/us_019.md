# US-019: Aggregate Patient Data from Intake & Documents

**Story ID:** US-019  
**Epic:** EP-003 (Clinical Data Intelligence & Coding)  
**Priority:** CRITICAL  
**Story Points:** 5  
**Owner Role:** Data Integration Engineer  
**Status:** Draft

## User Story

As a **clinical data analyst**, I want patient intake and uploaded documents aggregated into a single profile so that clinical staff can review all relevant patient information in one place.

## Business Value

- Reduces prep time for clinical staff
- Improves care coordination by consolidating data
- Ensures important patient details are not missed

## Scope

In scope:
- Aggregation of intake form responses and document-derived structured data
- Source tracking for each data element
- Presentation of a unified 360° patient profile foundation

Out of scope:
- Full AI coding or conflict detection workflows

## Acceptance Criteria

1. Given patient intake and document data, then the system aggregates them into a unified patient profile.
2. Given aggregated data, then source metadata is available for each element.
3. Given a patient profile view, then the profile includes intake answers, medications, allergies, and key extracted fields.
4. Given the profile is updated, then data provenance is preserved.

## Technical Notes

- Use a normalized data model for intake and document-derived elements.
- Maintain source references and timestamps for traceability.

## Dependencies

- Intake capture from EP-002.
- Document processing pipeline from US-020.

## Definition of Done

- Patient data aggregation works across intake and document inputs.
- Source tracking is preserved.
- Unified profile data is available for display.
- Documentation and tests are completed.
