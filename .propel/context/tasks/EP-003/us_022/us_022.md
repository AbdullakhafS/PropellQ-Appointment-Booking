# US-022: Build 360° Patient Profile UI (Tabs: Overview, Meds, Allergies, Diagnoses)

**Story ID:** US-022  
**Epic:** EP-003 (Clinical Data Intelligence & Coding)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Frontend Engineer  
**Status:** Draft

## User Story

As a **clinical staff member**, I want a unified patient profile UI with overview, medication, allergy, and diagnosis tabs so that I can review key patient data quickly and comprehensively.

## Business Value

- Improves clinical efficiency by consolidating data
- Presents complex patient information in a structured way
- Enables faster decision-making for care preparation

## Scope

In scope:
- Patient profile UI with tabbed navigation
- Overview tab with intake summary and key patient details
- Medications, allergies, and diagnoses tabs with extracted data
- Data freshness indicators and source links

Out of scope:
- Full clinician charting or note editing workflows

## Acceptance Criteria

1. Given a patient profile, then the UI displays tabs for Overview, Medications, Allergies, and Diagnoses.
2. Given extracted data, then each tab presents the data with clear labels and source attribution.
3. Given a clinician views the profile, then the layout is responsive and usable across desktop and tablet.
4. Given a data element is selected, then source document or intake source details are available.

## Technical Notes

- Keep the UI performant by loading tab content on demand.
- Ensure PHI display is restricted to authorized staff roles.
- Use consistent styling and accessible tab patterns.

## Dependencies

- Aggregated profile data from US-019.
- Extracted clinical data from US-021.

## Definition of Done

- Profile UI implemented with all tabs.
- Data sources are displayed and traceable.
- Responsive behavior validated.
- Security review and access gating completed.
