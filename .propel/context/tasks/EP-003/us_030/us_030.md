# US-030: Build Conflict Resolution UI (Side-by-Side Comparison)

**Story ID:** US-030  
**Epic:** EP-003 (Clinical Data Intelligence & Coding)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Frontend Engineer  
**Status:** Draft

## User Story

As a **clinical reviewer**, I want a conflict resolution UI with side-by-side comparison so that I can evaluate and resolve data conflicts clearly.

## Business Value

- Improves accuracy of conflict resolution decisions
- Presents data discrepancies in a clear, actionable way
- Reduces time spent reconciling conflicting clinical information

## Scope

In scope:
- Side-by-side comparison of conflicting medication, allergy, or diagnosis data
- Display of source provenance and confidence metrics
- Actions to resolve, merge, or discard conflicting entries

Out of scope:
- Automatic conflict resolution without reviewer input

## Acceptance Criteria

1. Given conflicting clinical data, then the UI displays both versions side-by-side.
2. Given conflict details, then source provenance and confidence metrics are shown.
3. Given a reviewer chooses a resolution action, then the decision is persisted and audit-logged.
4. Given resolved conflicts, then they are removed from the active conflict queue.

## Technical Notes

- Use clear visual differentiation for conflicting values.
- Ensure actions are reversible or reviewed through audit logs.
- Keep the UI focused on high-severity conflicts.

## Dependencies

- Conflict detection outputs from US-024 and US-025.
- Audit logging from EP-007.

## Definition of Done

- Conflict resolution UI implemented.
- Side-by-side comparison works and sources are visible.
- Resolution actions persist and log properly.
- Usability and security tests completed.
