# Task 005: Integrate Mode Switching with Chatbot & Form Components

**Task ID:** task_014_005  
**User Story:** US-014 (Enable Intake Mode Switching - AI ↔ Manual)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer (React)  
**Status:** Not Started

## Objective

Integrate mode switching functionality with existing chatbot (US-011) and form (US-013) components to enable seamless data transfer between modes.

## Requirements

1. **Chatbot Component Integration**
   - On form/button render: Display "Switch to Form?" button/link
   - On switch trigger: Collect all chatbot conversation data
   - Map chatbot data to form structure using mapChatbotToForm()
   - Save partial intake data to context
   - Call draft save API: POST /api/intake/draft
   - Render form component with pre-filled data
   - Show indication: "You've already answered these questions via AI. Review and edit as needed."

2. **Form Component Integration**
   - On component render: Check for mode and existing data
   - If switching from chatbot: Pre-populate form with mapped data
   - Display "Switch to Chatbot?" button/link
   - On switch trigger: Collect form field values
   - Map form data to chatbot context using mapFormToChatbot()
   - Save partial intake data to context
   - Call draft save API: POST /api/intake/draft
   - Render chatbot with context: "We see you've already filled out [fields]. Let me continue..."

3. **Progress Indicator Consistency**
   - In AI mode: Show "Question X of Y in AI mode"
   - In form mode: Show "Step X of Y in form mode"
   - After switch: Update progress indicator based on new mode
   - Maintain approximate progress % (e.g., if 50% done in form, show 50% in chatbot)

4. **Data Pre-population on Mode Switch**
   - Chatbot → Form: Medications array auto-populated in form.medications (fieldArray)
   - Chatbot → Form: Allergies checkboxes pre-checked based on drug_allergies[]
   - Form → Chatbot: Resume chatbot with: "So you're taking [meds]. Any other health issues?"
   - Partial data: If only chief complaint filled, chatbot resumes from medical history
   - Full data: If form is complete, chatbot shows summary for confirmation

5. **Switch Data Preservation**
   - Before switch: Call context updateData() to persist current state
   - On switch: Apply data mapping (mapChatbotToForm or mapFormToChatbot)
   - After switch: Render new component with mapped data
   - No data loss: All fields that were filled remain filled in new mode
   - Visual feedback: Show what data is being transferred

6. **Session Resume (Draft Loading)**
   - On page load (intake start): Check for existing draft via GET /api/intake/draft
   - If draft exists and not expired: Display resume prompt: "You have a saved intake in progress. Resume?" [Yes] [Start New]
   - If resume: Load draft mode and data, render appropriate component
   - If start new: Clear draft, start fresh with mode selection

7. **Accessibility & Mobile**
   - All switch buttons keyboard accessible
   - Switch button doesn't obscure form/chatbot content
   - Modal focus trap working
   - Mobile: Switch button visible without scrolling
   - Toast notifications announced to screen reader
   - Progress indicator updated and re-announced

8. **Error Handling**
   - If data mapping fails: Show error "Could not switch modes. Please try again."
   - If draft save fails: Show warning "Draft save failed, but you can continue"
   - If draft load fails: Default to fresh start (don't block intake)
   - Retry logic: Allow user to retry switch/save

9. **Testing Scenarios**
   - Switch from chatbot (50% complete) → form (verify data auto-populated)
   - Switch from form (30% complete) → chatbot (verify chatbot resumes correctly)
   - Multiple switches (chatbot → form → chatbot) with data preservation
   - Page refresh during form (resume from draft)
   - Network error on switch (graceful fallback)
   - Switch limit exceeded (button disabled)

## Implementation Steps

1. Integrate IntakeContext with ChatbotComponent
2. Add "Switch to Form" button to chatbot component
3. On switch click: Collect chatbot data, map it, save to context and API
4. Render form with pre-populated data (from mapped chatbot data)
5. Integrate IntakeContext with FormComponent
6. Add "Switch to Chatbot" button to form component
7. On switch click: Collect form data, map it, save to context and API
8. Resume chatbot with context/previous responses
9. Implement progress indicator updates on switch
10. Add draft resume on page load (GET /api/intake/draft)
11. Display resume prompt if draft exists
12. Implement error handling for all switch scenarios
13. Test all integration scenarios
