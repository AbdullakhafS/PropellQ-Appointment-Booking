# Task 002: Implement Data Mapping Functions (Chatbot ↔ Form)

**Task ID:** task_014_002  
**User Story:** US-014 (Enable Intake Mode Switching - AI ↔ Manual)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Backend/Frontend Developer  
**Status:** Not Started

## Objective

Implement bidirectional data mapping functions to convert between chatbot output format and form field format, ensuring seamless data preservation across mode switches.

## Requirements

1. **Chatbot to Form Mapping**
   ```typescript
   function mapChatbotToForm(chatbotData: ChatbotOutput): FormData {
     // Input: Chatbot-extracted data structure
     // Output: Form field structure
     // Mappings:
     // - chatbot.chief_complaint → form.chief_complaint (direct)
     // - chatbot.medical_history.conditions[] → form.medical_history checkboxes
     // - chatbot.medications[] → form.medications fieldArray rows
     // - chatbot.allergies.drug_allergies[] → form allergies entries
     // - chatbot.insurance → form insurance fields
   }
   ```

2. **Form to Chatbot Mapping**
   ```typescript
   function mapFormToChatbot(formData: FormData): ChatbotConversation {
     // Input: Form field values
     // Output: Natural language summary for chatbot context
     // Mappings:
     // - form.chief_complaint → "Your chief complaint is: [value]"
     // - form.medications[] → "You're taking: [med1] [dosage], [med2]..."
     // - form.allergies → "Drug allergies: [allergy1], [allergy2]..."
     // - form.insurance → "Insurance: [name], Member ID: [id]"
   }
   ```

3. **Chatbot Output Format**
   ```typescript
   interface ChatbotOutput {
     chief_complaint: string;
     medical_history: {
       conditions: string[];
       other: string;
     };
     medications: Array<{
       name: string;
       dosage: string;
       frequency: string;
       confidence: number; // 0-1, extracted from NLP
     }>;
     allergies: {
       drug_allergies: string[];
       reactions: Record<string, string>; // { penicillin: 'rash' }
       side_effects: string[];
     };
     insurance: {
       name: string;
       member_id: string;
       group_number: string;
       plan_name: string;
     };
     extracted_data: object; // Raw NLP extraction
     confidence_scores: Record<string, number>; // Per-field confidence
   }
   ```

4. **Form Data Format**
   ```typescript
   interface FormData {
     chief_complaint: string;
     medical_history: {
       conditions: string[]; // Checkbox selections
       other: string; // Free text
     };
     medications: Array<{
       name: string;
       dosage: string;
       frequency: string;
     }>;
     allergies: Array<{
       name: string;
       reaction_type: 'allergy' | 'side_effect';
       reaction_description: string;
     }>;
     insurance: {
       name: string;
       member_id: string;
       group_number: string;
       plan_name: string;
     };
   }
   ```

5. **Mapping Edge Cases**
   - Empty/null values: Skip or show placeholder
   - Partial data: Only map populated fields (don't overwrite with nulls)
   - High-confidence extractions: Mark visually (e.g., "AI extracted: [value]")
   - Low-confidence extractions: Show as suggestions, not auto-fill
   - Medications with no name: Skip or show as "Unknown medication"
   - Allergies without reaction type: Default to "allergy"

6. **Validation on Mapping**
   - Check that required fields are present before mapping
   - Validate data types (array vs string vs object)
   - Warn if mapping results in data loss
   - Normalize data: trim whitespace, handle case sensitivity

7. **Round-Trip Consistency**
   - Test: Form → Chatbot → Form should preserve core data
   - Test: Chatbot → Form → Chatbot should preserve meaning
   - Test: Multiple rounds of switching should not degrade data

8. **Confidence Score Handling**
   - Chatbot provides confidence_scores for extracted data
   - Map high-confidence (>0.8) extractions → auto-filled form fields
   - Map medium-confidence (0.6-0.8) → prefilled but highlighted for review
   - Map low-confidence (<0.6) → shown as suggestion, not auto-filled

## Implementation Steps

1. Define TypeScript interfaces for all data structures

4. Handle edge cases: null/empty values, partial data

9. Write unit tests for each mapping scenario (12+ test cases)
