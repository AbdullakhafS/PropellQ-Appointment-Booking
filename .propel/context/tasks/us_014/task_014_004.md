# Task 004: Implement Backend Draft Save/Retrieve API

**Task ID:** task_014_004  
**User Story:** US-014 (Enable Intake Mode Switching - AI ↔ Manual)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Backend Developer  
**Status:** Not Started

## Objective

Implement backend API endpoints to save and retrieve draft intake data, allowing users to resume intake after page refresh or later session.

## Requirements

1. **Save Draft Endpoint**
   - Endpoint: `POST /api/intake/{appointmentId}/draft`
   - Authentication: Verify user owns appointment
   - Request body:
     ```json
     {
       "mode": "ai" | "manual",
       "data": { /* full intake data */ },
       "switchCount": 0-2,
       "lastSwitchedAt": "ISO timestamp or null"
     }
     ```
   - Response: `{ success: true, draftId: "...", savedAt: "..." }`
   - Error handling: 401 unauthorized, 404 appointment not found, 400 invalid data

2. **Retrieve Draft Endpoint**
   - Endpoint: `GET /api/intake/{appointmentId}/draft`
   - Authentication: Verify user owns appointment
   - Response:
     ```json
     {
       "mode": "ai" | "manual",
       "data": { /* full intake data */ },
       "switchCount": 0-2,
       "lastSwitchedAt": "ISO timestamp or null",
       "savedAt": "ISO timestamp",
       "expiresAt": "ISO timestamp (24 hours from savedAt)"
     }
     ```
   - If no draft exists: Return 404 or empty object
   - If draft expired: Return 404 (don't return stale data)

3. **Clear Draft Endpoint**
   - Endpoint: `DELETE /api/intake/{appointmentId}/draft`
   - Authentication: Verify user owns appointment
   - Trigger: When intake is completed/submitted
   - Response: `{ success: true, message: "Draft cleared" }`
   - Delete from database to free storage

4. **Database Schema**
   - Create IntakeDraft table if not exists:
     ```sql
     CREATE TABLE IntakeDrafts (
       id INT PRIMARY KEY,
       appointment_id INT FOREIGN KEY,
       patient_id INT FOREIGN KEY,
       mode VARCHAR ('ai', 'manual'),
       data JSON,
       switchCount INT,
       lastSwitchedAt DATETIME NULL,
       savedAt DATETIME,
       expiresAt DATETIME,
       created_at DATETIME,
       updated_at DATETIME
     );
     
     CREATE INDEX idx_appointment_id ON IntakeDrafts(appointment_id);
     CREATE INDEX idx_expires_at ON IntakeDrafts(expiresAt);
     ```

5. **Draft Expiration**
   - Set expiry to 24 hours from save time
   - Scheduled job: Delete expired drafts daily at midnight
   - Don't return expired drafts (validation on retrieval)

6. **Data Validation**
   - Validate mode is 'ai' or 'manual'
   - Validate switchCount is 0-2
   - Validate data structure matches expected schema
   - Validate timestamps are ISO format

7. **Concurrency Handling**
   - If multiple saves within 1 second, use last one (UPDATE not INSERT)
   - Don't create duplicate drafts for same appointment
   - Handle race conditions (last write wins)

8. **Performance**
   - Save response time: <500ms
   - Retrieve response time: <300ms
   - Support 1000 concurrent saves (load testing)

9. **Security**
   - Verify user is patient or clinic staff authorized to view appointment
   - Don't expose other patients' draft data
   - Log all draft operations for audit trail
   - Encrypt sensitive data in database (medical info)

## Implementation Steps

1. Create IntakeDraft model/entity
2. Create IntakeDrafts database table with indexes
3. Implement save draft endpoint (POST)
4. Implement retrieve draft endpoint (GET)
5. Implement clear draft endpoint (DELETE)
6. Add validation for all endpoints
7. Add authentication/authorization checks
8. Implement draft expiration logic (24-hour TTL)
9. Create scheduled job to clean expired drafts
10. Add logging for audit trail
11. Write integration tests (5+ test cases)
12. Load test: 1000 concurrent saves
13. Document API endpoints
