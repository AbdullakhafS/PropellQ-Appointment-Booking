# Task 001: Implement Intake Mode State Management

**Task ID:** task_014_001  
**User Story:** US-014 (Enable Intake Mode Switching - AI ↔ Manual)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Frontend Developer (React)  
**Status:** Not Started

## Objective

Implement centralized React context/Redux state management for tracking intake mode, data, switch count, and session persistence using localStorage.

## Requirements

1. **Intake State Structure**
   ```typescript
   interface IntakeState {
     mode: 'ai' | 'manual';
     hasInitialized: boolean;
     data: {
       chief_complaint: string;
       medical_history: object;
       medications: array;
       allergies: object;
       insurance: object;
     };
     switchCount: number;
     lastSwitchedAt: Date | null;
     status: 'in-progress' | 'completed';
     appointmentId: string;
   }
   ```

2. **Context Actions**
   - `initializeIntake(mode)` - Set initial mode at intake start
   - `updateData(section, data)` - Update partial intake data
   - `switchMode(newMode)` - Attempt mode switch (checks switch limit)
   - `saveProgress()` - Save current state to localStorage
   - `resumeFromDraft()` - Load previous session state
   - `completeIntake()` - Mark intake as complete
   - `resetIntake()` - Clear all data and start over

3. **Switch Limit Enforcement**
   - Track switchCount in state
   - Prevent switch if switchCount >= 2
   - Display warning if switchCount == 1: "This is your last mode switch"
   - Record timestamp of each switch (lastSwitchedAt)

4. **Data Persistence**
   - Save intake state to localStorage on every update
   - Use key: `intake_${appointmentId}_draft`
   - Include: mode, data, switchCount, lastSwitchedAt
   - Persist session for 24 hours (add expiry timestamp)
   - Clear from localStorage on intake completion

5. **Context Provider**
   - Create IntakeContext with default values
   - Create IntakeProvider wrapper component
   - Provide dispatch actions and state to child components
   - Enable component access via useIntake hook

6. **Data Immutability**
   - Use spread operator or Immer for state updates (no mutations)
   - Maintain data structure consistency across mode switches
   - Handle nested objects (medical_history, allergies, insurance) without loss

7. **Performance Optimization**
   - Memoize context value to prevent unnecessary re-renders
   - Use useCallback for dispatch actions
   - Lazy load draft from localStorage (avoid blocking on mount)

## Implementation Steps

1. Create IntakeContext with TypeScript interface
2. Create IntakeProvider component with reducer logic
3. Implement initialization action with mode selection
4. Implement updateData action for partial updates

7. Create useIntake hook for component consumption
8. Test state updates and data immutability
9. Test localStorage persistence and retrieval
10. Test switch limit enforcement
