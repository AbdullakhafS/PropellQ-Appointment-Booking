# US-090: Optimize Database Queries with Indexes

**Story ID:** US-090  
**Epic:** EP-008 (System Reliability & Performance)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** Database Engineer  
**Status:** Draft

## User Story

As a **database engineer**, I want high-frequency queries optimized with indexes so that the system can support low-latency responses and scale under load.

## Business Value

- Improves response times for key application workflows
- Reduces database load and resource consumption
- Supports reliability and performance targets

## Scope

In scope:
- Indexing strategy for appointment, user, queue, and audit queries
- Query plan analysis and optimization
- Verification of index effectiveness under expected workloads

Out of scope:
- Full database refactor or schema redesign beyond targeted index improvements

## Acceptance Criteria

1. Given the most common queries, then appropriate indexes are created and query plans improve.
2. Given index changes, then the database performance is validated under load.
3. Given query patterns, then index selection avoids unnecessary write amplification.
4. Given database statistics, then query latency p95 improves toward target thresholds.

## Technical Notes

- Focus on high-read, high-frequency access patterns for booking and queue operations.
- Use covering indexes where appropriate to avoid extra lookups.
- Monitor write impact and balance with query performance.

## Dependencies

- Data schema from EP-DATA-001.
- Performance targets from EP-008.

## Definition of Done

- Indexes implemented and validated.
- Query performance improved and documented.
- No adverse write-side impact detected.
- Review completed with database team.
