# US-002: Display Appointment Slots with Calendar

**Epic:** EP-001 (Patient Appointment Booking)  
**Story Points:** 5  
**Priority:** CRITICAL  
**Status:** Not Started  
**Created:** 2026-06-17  

---

## User Story Statement

As a **patient**, I want to **view available appointment slots on an interactive calendar with visual indicators for slot availability and pricing**, so that **I can quickly identify and select the best appointment time for my needs**.

---

## Business Value

- **Outcome 1:** Visual calendar interface is intuitive and reduces cognitive load vs. list-based slot display
- **Outcome 2:** Enables patients to spot patterns (e.g., "Tuesday mornings are full") and plan flexibly
- **Outcome 3:** Calendar-first UX increases booking confidence and completion rate

---

## Scope

**In Scope:**
- Interactive calendar widget (month/week view with toggle)
- Visual indicators: available slots (green), booked/full (gray), preferred slots (highlighted)
- Slot cards showing: date, time, provider name, location, duration
- Hover/click on slot to see provider details and availability status
- Calendar navigation (previous/next month)
- Responsive design for mobile (pinch-zoom on mobile calendar)

**Out of Scope:**
- Timezone handling beyond patient's local timezone (out of Phase 1)
- Waitlist join functionality (handled in waitlist epic EP-004)
- Pricing display/insurance coverage check (handled in EP-002)
- Appointment notes or reason for visit (handled in EP-002 intake)

---

## Acceptance Criteria

1. **Calendar displays full month view by default** with all available slots color-coded: green (available), gray (booked), blue (preferred slot); system fetches calendar data from appointment search endpoint (US-001)
2. **Week view available as toggle:** Clicking "Week View" button switches to 7-day calendar showing hourly time slots with appointment start times
3. **Slot card interaction:** Clicking on a colored slot displays provider name, specialty, appointment duration, location, and "Select" button without leaving calendar
4. **Provider preview on hover:** Hovering over provider name shows small popover with provider photo, credentials, and patient review count (if available)
5. **Calendar navigation works:** "Previous/Next Month" buttons update calendar; clicking specific date jumps to that month (keyboard accessible)
6. **Mobile responsiveness:** On 375px width, calendar displays as scrollable month grid; on 768px+, standard calendar layout; pinch-zoom enabled on mobile
7. **Timezone accuracy:** All times display in patient's local timezone (read from browser or user profile); UTC offset shown in footer
8. **Performance:** Calendar loads within 3 seconds; month view with 30+ slots renders without lag

## Technical Notes

**Requirement Mapping:** FR-001 (Patient Self-Service Appointment Booking)  
**Related Requirements:** NFR-002 (sub-500ms API latency), NFR-005 (HIPAA), UI-001 (mobile-responsive design)  

**Technology Stack:**
- Frontend: React with calendar library (e.g., React Calendar, FullCalendar, or custom)
- API: Reuses `GET /api/appointments/search` from US-001; adds optional `?view=calendar` param
- CSS: Responsive grid; Flexbox for slot cards

**Calendar Data Model:**
- Slots organized by date; each date has array of time slots with provider_id, duration, status
- Month view: fetch all slots for current month + 2 adjacent months (prefetch for UX)
- Week view: fetch 7-day range on demand

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Calendar library is heavy; adds 200KB to bundle size | Medium | Evaluate lightweight alternatives; use code-splitting to lazy-load calendar on appointments page |
| Timezone calculation is incorrect → shows wrong times to patient | High | Use `date-fns` or `luxon` for timezone handling; test with multiple TZ offsets in QA; store all times as UTC in DB |
| Month view with 30+ slots is slow on low-end mobile | Medium | Virtualize slot rendering; display only visible slots in viewport |
| User scrolls calendar and loses search filter context | Low | Keep search filters sticky in sidebar on desktop; show summary of applied filters above calendar |
