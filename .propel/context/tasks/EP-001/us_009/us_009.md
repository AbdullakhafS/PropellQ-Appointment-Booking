# US-009: Bidirectional Calendar Sync (External Updates)

**Epic:** EP-001 (Patient Appointment Booking)  
**Story Points:** 8  
**Priority:** HIGH  
**Status:** Not Started  
**Created:** 2026-06-17  

---

## User Story Statement

As a **patient**, I want to **have my appointment changes (cancellations, reschedules) reflected automatically in my Google Calendar and Outlook Calendar when I change them in PropellQ, and vice versa**, so that **my external calendars stay in sync and I never have stale appointment information**.

---

## Business Value

- **Outcome 1:** Bi-directional sync ensures patient has single source of truth; eliminates confusion
- **Outcome 2:** If patient reschedules via PropellQ, external calendar updates automatically → seamless experience
- **Outcome 3:** If patient cancels in external calendar, PropellQ reflects cancellation → prevents no-shows from external cancellations

---

## Scope

**In Scope:**
- PropellQ → External: When appointment is rescheduled or cancelled in PropellQ, update corresponding Google/Outlook event
- External → PropellQ: Poll Google/Outlook calendars periodically; if appointment event is deleted/modified externally, sync back to PropellQ
- Conflict resolution: If both systems change simultaneously, PropellQ is source of truth (PropellQ update wins)
- Event ID mapping: Store Google/Outlook event IDs in PropellQ database to track external references
- Soft delete handling: Cancelled appointments remain in PropellQ (never hard-deleted); external calendar event is deleted

**Out of Scope:**
- Patient initiates reschedule from external calendar → PropellQ updates (requires write-back, Phase 2 feature)
- Appointment status changes (e.g., "confirmed" → "tentative" in external calendar) — calendar event stays as-is
- Sync of availability/free-busy data (Phase 2)

---

## Acceptance Criteria

1. **PropellQ-to-External sync (push):** When patient reschedules appointment in PropellQ, system detects change and:
   - If Google Calendar connected: Update Google Calendar event with new date/time via Microsoft Graph API
   - If Outlook connected: Update Outlook Calendar event with new date/time via Google Calendar API
   - Update succeeds within 10 seconds of appointment change
2. **External-to-PropellQ sync (pull):** Background job runs every 30 minutes; polls Google/Outlook calendars:
   - If appointment event is deleted externally → mark appointment as "cancelled" in PropellQ; don't delete the record
   - If appointment event is rescheduled externally → log event for manual review (don't auto-update in Phase 1)
3. **Event ID tracking:** Store Google event ID and Outlook event ID in `Appointments` table; maintain reference throughout appointment lifecycle
4. **Cancellation sync:** When appointment is cancelled in PropellQ:
   - Delete corresponding calendar event from Google/Outlook within 10 seconds
   - Log deletion in audit trail
5. **Conflict resolution:** If both PropellQ and external calendar are modified simultaneously:
   - PropellQ change is source of truth; external calendar is updated to match PropellQ
   - Log conflict in audit trail for review
6. **Error handling:** If calendar API fails (rate-limit, network error, invalid token):
   - Queue failed sync for retry (exponential backoff)
   - Alert admin if retry fails after 3 attempts
   - Don't block appointment updates in PropellQ
7. **Webhook support (optional):** If Google/Outlook provides push notifications via webhooks, implement webhook handlers to sync in real-time (vs. polling)

## Technical Notes

**Requirement Mapping:** FR-004 (Calendar Integration - Google/Outlook)  
**Related Requirements:** NFR-001 (uptime), NFR-002 (latency <500ms), NFR-005 (HIPAA), NFR-007 (logging), TR-006 (logging), TR-007 (tracing), TR-008 (resilience)  

**Technology Stack:**
- Google Calendar API: `https://www.googleapis.com/calendar/v3/calendars/{calendarId}/events/{eventId}`
- Microsoft Graph API: `https://graph.microsoft.com/v1.0/me/calendar/events/{eventId}`
- Background Job: .NET BackgroundService (runs every 30 minutes for polling)
- Event ID Storage: `Appointments.google_event_id`, `Appointments.outlook_event_id`
- Sync Queue: `CalendarSyncQueue` table for failed syncs

**Database Schema:**
```sql
ALTER TABLE Appointments ADD COLUMN (
  google_event_id VARCHAR (nullable),
  outlook_event_id VARCHAR (nullable),
  last_synced_at DATETIME (nullable),
  sync_status VARCHAR ('synced', 'pending', 'failed')
);

CREATE TABLE CalendarSyncQueue (
  id INT PRIMARY KEY,
  appointment_id INT FOREIGN KEY,
  action VARCHAR ('update', 'delete'),
  calendar_type VARCHAR ('google', 'outlook'),
  enqueued_at DATETIME,
  retry_count INT DEFAULT 0,
  last_error VARCHAR (nullable),
  scheduled_retry_at DATETIME (nullable)
);
```

**Push Sync Flow (Reschedule/Cancel in PropellQ):**
```
UPDATE Appointments SET slot_id = new_slot
  → Trigger: AFTER UPDATE appointment
  
  → ENQUEUE to CalendarSyncQueue:
  INSERT CalendarSyncQueue 
    (appointment_id, action='update', calendar_type='google')
  INSERT CalendarSyncQueue 
    (appointment_id, action='update', calendar_type='outlook')

→ Async worker processes queue:
  FOREACH row IN CalendarSyncQueue WHERE retry_count < 3:
    
    IF calendar_type = 'google':
      PATCH /calendar/v3/calendars/primary/events/{google_event_id}
        { start, end, description (updated time) }
    
    IF calendar_type = 'outlook':
      PATCH /graph/v1.0/me/calendar/events/{outlook_event_id}
        { start, end, subject (updated time) }
    
    IF success:
      DELETE FROM CalendarSyncQueue
      UPDATE Appointments SET sync_status='synced', last_synced_at=now
    ELSE:
      UPDATE CalendarSyncQueue 
        SET retry_count++, 
            scheduled_retry_at = now + (2 ^ retry_count) minutes,
            last_error = error_message
      INSERT AuditLog (event='calendar_sync_failed', details={error})
```

**Pull Sync Flow (Poll External Calendars every 30 min):**
```
FOREACH patient WHERE google_refresh_token IS NOT NULL:
  GET /calendar/v3/calendars/primary/events
    ?timeMin=24h_ago &updatedMin=last_sync_time
  
  FOREACH event IN response:
    IF event.id NOT IN (SELECT google_event_id FROM Appointments):
      → Unknown event; skip
    ELSE:
      appointment = SELECT * FROM Appointments WHERE google_event_id = event.id
      
      IF event.status = 'cancelled':
        UPDATE Appointments SET status='cancelled' WHERE id=appointment.id
        INSERT AuditLog (event='appointment_cancelled_external')
      
      ELSE IF event.start != appointment.appointment_time:
        → Reschedule detected externally
        INSERT AuditLog (event='external_reschedule_detected', 
          original_time=appointment.appointment_time,
          new_time=event.start)
        → ALERT admin for manual review (don't auto-update)
```

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Race condition: PropellQ reschedule + external reschedule simultaneously → data inconsistency | High | Use optimistic locking (version column); if conflict detected, PropellQ wins; log conflict for audit |
| Google/Outlook API rate-limit → syncs fail, external calendar not updated | Medium | Implement exponential backoff; batch updates (e.g., 10 events per request); monitor quota usage |
| Event ID is deleted/corrupted → sync logic breaks | Medium | Validate event ID exists in database before sync; if missing, log as error but don't crash; re-create event if needed |
| Patient revokes calendar access (token invalid) → sync permanently fails | Low | Detect invalid token errors; set `google_auth_status='expired'`; prompt user to re-authorize |
| Webhook delivery is delayed or lost → sync is out-of-sync | Low | Use polling as primary mechanism; webhooks are optional acceleration; polling catches missed updates |
