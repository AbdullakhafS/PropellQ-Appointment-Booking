# US-049: Implement Secure Password Reset (Email Link)

**Story ID:** US-049  
**Epic:** EP-005 (User Access & Role Management)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** Security / Backend Engineer  
**Status:** Draft

## User Story

As a **user**, I want a secure password reset flow using email links so that I can restore access without risking credential theft or insecure reset processes.

## Business Value

- Increases user self-service while preserving security
- Reduces support load for forgotten password cases
- Ensures password reset flow complies with security standards

## Scope

In scope:
- Email-based password reset request and token issuance
- Secure one-time reset links with expiration
- Proper validation and rate limiting for reset requests
- Notification of password reset activity to account email

Out of scope:
- Passwordless login flows
- Third-party identity provider password resets

## Acceptance Criteria

1. Given a valid email, when a password reset is requested, then the system sends a one-time reset link to the user's email.
2. Given an expired or invalid reset link, when the user attempts to use it, then the system rejects it with a clear error.
3. Given a valid reset link, when the user submits a new password, then the password is updated and the token invalidated.
4. Given multiple reset requests, then tokens are rate-limited and earlier unused tokens are invalidated.
5. Given a password reset occurs, then the user receives notification of the activity.

## Technical Notes

- Use cryptographically secure token generation and storage.
- Set reset token expiration to a strict window (e.g. 30 minutes).
- Avoid disclosing whether an email is registered in reset request responses.

## Dependencies

- User account management APIs from US-048.
- Email delivery service integration.

## Definition of Done

- Password reset flow implemented end-to-end and tested.
- Reset tokens expire and cannot be reused.
- Rate limiting protects against abuse.
- Security review completed.
