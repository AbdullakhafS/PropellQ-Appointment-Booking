# US-052: Build Admin User Audit Log Viewer

**Story ID:** US-052  
**Epic:** EP-005 (User Access & Role Management)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** Admin / Security Engineer  
**Status:** Draft

## User Story

As an **admin user**, I want an audit log viewer for user management actions so that I can review account changes, detect suspicious activity, and support compliance reporting.

## Business Value

- Provides transparency into user administration operations
- Supports compliance and internal review processes
- Enables faster investigation of account-related incidents

## Scope

In scope:
- Admin-accessible audit log viewer for user lifecycle actions
- Filtering by actor, action type, date range, and affected user
- Read-only view with export capability
- Secure access restricted to Admin role only

Out of scope:
- Full system-wide audit analytics (EP-007 broader scope)
- External audit export formats beyond CSV/JSON

## Acceptance Criteria

1. Given an Admin user is authenticated, when they view the audit log page, then only Admin role users can access it.
2. Given a log viewer request, then the interface supports filtering by actor, action type, affected user, and date range.
3. Given an audit log entry is selected, then the details display actor, action, target user, timestamp, and reason.
4. Given an admin requests export, then the log viewer provides CSV or JSON download.
5. Given an unauthorized user attempts access, then the system returns 403 Forbidden.

## Technical Notes

- Leverage existing audit log storage from EP-007.
- Keep the viewer read-only and role-restricted.
- Ensure exported logs do not include more sensitive PHI than needed.

## Dependencies

- Audit log infrastructure from EP-007.
- Admin role permissions from US-046.

## Definition of Done

- Admin audit log viewer implemented and tested.
- Filtering and export functionality works correctly.
- Access restricted to admin role.
- Security review completed.
