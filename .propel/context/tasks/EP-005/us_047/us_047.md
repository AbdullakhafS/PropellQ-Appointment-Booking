# US-047: Build User Account Management UI (Admin Only)

**Story ID:** US-047  
**Epic:** EP-005 (User Access & Role Management)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Frontend / Admin Engineer  
**Status:** Draft

## User Story

As an **admin user**, I want a secure account management interface so that I can create, edit, and deactivate staff and patient accounts without exposing sensitive operations to unauthorized roles.

## Business Value

- Streamlines onboarding and offboarding of users
- Reduces administrative overhead for staff management
- Ensures user lifecycle operations are auditable and restricted

## Scope

In scope:
- Admin UI for creating new user accounts and assigning roles
- Edit user details, role assignment, and active/inactive status
- Search and filter users by name, email, role, and status
- Confirmation workflows for deactivations and role changes

Out of scope:
- Self-service patient registration
- External identity provider setup

## Acceptance Criteria

1. Given an Admin user is authenticated, when they open the user management page, then they see a list of all active users with role and status.
2. Given an Admin user clicks "Create User", then a secure form opens with role, email, display name, and initial status fields.
3. Given an Admin user updates a user’s role or status, then the update is saved and audit-logged.
4. Given an Admin user deactivates a user, then the user cannot log in and the change is confirmed before applying.
5. Given an Admin user searches users, then filter results are returned accurately by name, email, role, and status.

## Technical Notes

- UI routes must be restricted to Admin role only.
- Use secure form validation and avoid exposing sensitive user details in lists.
- Backend endpoints should require Admin authorization and log changes.

## Dependencies

- RBAC model from US-043.
- Admin role permissions from US-046.
- Audit logging from EP-007.

## Definition of Done

- Admin user management UI implemented and access-restricted.
- Create/edit/deactivate workflows validated.
- Audit logging for all account changes.
- UX review and security review completed.
