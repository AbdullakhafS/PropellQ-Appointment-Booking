User Story: US-051

## Objective
Ensure role/status changes propagate by revoking or refreshing existing tokens.

## Acceptance Criteria
- [ ] Role changes trigger token invalidation or short-lived refresh policy.
- [ ] Post-change requests with stale claims are rejected.
- [ ] Behavior is documented for operations/support teams.

## Definition of Done
Role-change token consistency is enforced.
