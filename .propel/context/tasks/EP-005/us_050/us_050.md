# US-050: Add Permission Checks to All API Endpoints

**Story ID:** US-050  
**Epic:** EP-005 (User Access & Role Management)  
**Priority:** CRITICAL  
**Story Points:** 5  
**Owner Role:** Backend Engineer  
**Status:** Draft

## User Story

As a **security engineer**, I want all API endpoints to enforce role-based permission checks so that unauthorized users cannot access data or actions outside their permitted scope.

## Business Value

- Prevents unauthorized access and privilege escalation
- Strengthens compliance with data protection requirements
- Ensures access control is consistently enforced across the platform

## Scope

In scope:
- Permission validation on all protected API endpoints
- Standardized 403 response behavior for unauthorized access
- Integration with RBAC model and token claims
- Logging of denied authorization attempts

Out of scope:
- New endpoint development unrelated to access control
- UI authorization messaging beyond standard error handling

## Acceptance Criteria

1. Given any protected API endpoint, when a request arrives, then role and permission checks execute before processing.
2. Given a request with insufficient permissions, then the API returns 403 Forbidden and logs the denied request.
3. Given a request from a valid role, then the endpoint proceeds only if the role is authorized for that action.
4. Given a request for sensitive data, then access is additionally scoped by ownership or assignment.
5. Given authorization failures occur, then denied events are tracked for auditing.

## Technical Notes

- Use a centralized authorization middleware or service layer.
- Map endpoints to permission assertions rather than broad role checks where feasible.
- Ensure 403 responses do not leak sensitive resource details.

## Dependencies

- RBAC definitions from US-043.
- Staff, patient, and admin role policies from US-044, US-045, US-046.

## Definition of Done

- All protected endpoints enforce permission checks.
- Unauthorized access attempts return standard 403 semantics.
- Audit log includes denied access actions.
- Regression tests verify enforcement for each role.
