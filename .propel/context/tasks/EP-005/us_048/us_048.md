# US-048: Implement User Create/Edit/Deactivate

**Story ID:** US-048  
**Epic:** EP-005 (User Access & Role Management)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** Backend Engineer  
**Status:** Draft

## User Story

As an **admin user**, I want secure backend APIs for creating, editing, and deactivating user accounts so that account lifecycle operations are managed reliably and auditably.

## Business Value

- Enables controlled user onboarding and offboarding
- Supports compliance by preventing inactive accounts from logging in
- Reduces manual admin burden on user maintenance

## Scope

In scope:
- API endpoints for create, update, and deactivate user accounts
- Validation of role assignment and required user fields
- Deactivation and reactive activation workflows
- Audit logging for all lifecycle changes

Out of scope:
- Password reset flow (handled in US-049)
- Self-service enrollment portal

## Acceptance Criteria

1. Given Admin credentials, when a create account request is submitted, then a new user record is created with assigned role and active status.
2. Given Admin credentials, when user details are edited, then the updates are persisted and returned in the API response.
3. Given Admin credentials, when a user is deactivated, then the user's status becomes inactive and they can no longer authenticate.
4. Given invalid user data is submitted, then the API returns a validation error with clear messages.
5. Given any create/edit/deactivate action occurs, then it is audit-logged with actor, timestamp, and changes.

## Technical Notes

- Enforce Admin-only authorization on all user management endpoints.
- Use strong input validation and avoid leaking existing user secrets.
- Ensure deactivated users are excluded from authentication token issuance.

## Dependencies

- RBAC model and Admin permissions from EP-005.
- Audit log infrastructure from EP-007.

## Definition of Done

- Backend user management APIs implemented, tested, and documented.
- Audit logs capture all lifecycle operations.
- Deactivated users cannot log in.
- Security review completed.
