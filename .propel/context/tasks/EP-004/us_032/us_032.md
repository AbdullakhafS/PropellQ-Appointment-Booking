# US-032: Assign Walk-In to Available Slots

**Story ID:** US-032  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** HIGH  
**Story Points:** 3  
**Owner Role:** Product / Platform Engineer  
**Status:** Draft

## User Story

As a **staff scheduler**, I want walk-in patients to be assigned to available appointment slots automatically so that I can minimize manual slot selection and reduce scheduling errors.

## Business Value

- Streamlines walk-in appointment creation
- Reduces likelihood of conflicting or unavailable slot selection
- Increases staff productivity and scheduling accuracy

## Scope

In scope:
- Lookup of available slots for the selected clinic/provider
- Automatic slot assignment during walk-in booking
- Validation of walk-in slot availability at save time

Out of scope:
- Full patient intake or clinical triage
- Waiting room seat assignment

## Acceptance Criteria

1. Given a walk-in patient is ready to be booked, when the staff user views available slots, then the system suggests open slots for the selected provider or clinic.
2. Given a slot is selected, when the walk-in booking is submitted, then the appointment persists in that slot and no double-booking occurs.
3. Given no open slots exist, when the staff user tries to save the walk-in, then a clear message is shown and alternative options are offered.
4. Given the walk-in is scheduled, then the appointment record includes the selected slot and walk-in flag.

## Technical Notes

- Reuse availability logic from EP-001 where possible.
- Treat walk-in appointments as real-time schedule changes with immediate conflict checks.
- Provide a fallback message if providers are fully booked.

## Dependencies

- Provider scheduling and availability API
- Appointment slot conflict detection
- Staff authentication from EP-005

## Definition of Done

- Walk-in slot selection integrated into booking flow
- Real-time availability check executed before save
- Conflicts prevented during walk-in assignment
- Acceptance criteria validated with staff workflow scenarios
