# Task 001: Implement Walk-In Slot Availability Lookup Service

**Task ID:** task_032_001  
**User Story:** US-032 (Assign Walk-In to Available Slots)  
**Epic:** EP-004  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Backend Developer  
**Status:** Not Started

## Objective

Build a backend service that returns open appointment slots for walk-in scheduling based on selected clinic and provider context.

## Requirements

1. Availability lookup:
   - Reuse EP-001 availability logic where possible
   - Support provider-specific and clinic-wide lookups
   - Return only open slots that are bookable now

2. Response contract:
   - slot_id, provider_id, provider_name, start_time, end_time, location
   - optional priority/ranking score for best-fit suggestion

3. Filtering:
   - Support current-day and near-term windows for walk-ins
   - Exclude blocked, canceled, and already-booked slots

4. Performance:
   - API response target suitable for front-desk use (<500ms)

## Implementation Steps

1. Create availability query endpoint/service for walk-ins.
2. Integrate provider and clinic filters.
3. Add reusable mapper for slot DTO output.
4. Add unit tests for filtering and ranking behavior.
5. Validate response time with realistic slot volume.

## Acceptance Criteria

- [ ] Open slots are returned for selected provider/clinic
- [ ] Unavailable/conflicting slots are excluded
- [ ] Response contract supports UI slot suggestion
- [ ] API meets expected desk workflow responsiveness

## Dependencies

- EP-001 scheduling availability services
- Provider and clinic master data

## Validation

- [ ] Tests cover provider and clinic lookup cases
- [ ] Fully booked scenario returns empty list
- [ ] Performance baseline recorded

## Definition of Done

- Walk-in slot availability lookup is implemented and consumable by booking UI.
