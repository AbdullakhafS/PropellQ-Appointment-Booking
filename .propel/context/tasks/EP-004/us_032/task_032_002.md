# Task 002: Build Slot Suggestion and Selection UI in Walk-In Flow

**Task ID:** task_032_002  
**User Story:** US-032 (Assign Walk-In to Available Slots)  
**Epic:** EP-004  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Frontend Developer  
**Status:** Done

## Objective

Add slot suggestion and slot selection components to walk-in booking so staff can quickly choose from open appointment options.

## Requirements

1. Slot suggestion view:
   - Display suggested slots for selected provider or clinic
   - Show provider name, time range, and slot status

2. Selection behavior:
   - Allow one active slot selection before submit
   - Clearly indicate selected slot in UI

3. UX considerations:
   - Fast refresh when provider/clinic changes
   - Empty state for no slots with guidance
   - Accessible keyboard navigation for slot list

4. Integration:
   - Consume backend availability endpoint from task_032_001
   - Pass selected slot details into booking payload

## Implementation Steps

1. Build slot list/suggestion component.
2. Integrate with availability lookup API.
3. Add selection state and visual indicators.
4. Handle no-slot and loading/error states.
5. Add component tests for selection and refresh behavior.

## Acceptance Criteria

- [x] Staff sees open slot suggestions during walk-in booking
- [x] Slot selection is clear and persists until submit
- [x] Slot list updates on provider/clinic change
- [x] Empty/error states are user-friendly

## Dependencies

- Task 032_001 availability service
- Existing walk-in booking UI from US-031

## Validation

- [x] Slot selection reflected in submit payload
- [x] No-slot state appears correctly when availability is empty
- [x] Accessibility checks pass for slot navigation

## Definition of Done

- Slot suggestion and selection UI is fully integrated into walk-in flow.
