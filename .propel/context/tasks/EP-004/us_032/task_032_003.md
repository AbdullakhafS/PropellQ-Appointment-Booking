# Task 003: Enforce Real-Time Conflict Check and Safe Slot Assignment

**Task ID:** task_032_003  
**User Story:** US-032 (Assign Walk-In to Available Slots)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Backend Developer  
**Status:** Done

## Objective

Ensure walk-in booking performs real-time slot conflict validation at save time and prevents double-booking.

## Requirements

1. Save-time validation:
   - Re-check slot availability when booking request is submitted
   - Reject booking if slot became unavailable between lookup and submit

2. Concurrency handling:
   - Prevent race-condition double booking
   - Use transactional lock/version check or equivalent strategy

3. Persistence:
   - Save appointment with selected slot_id and walk_in flag
   - Return definitive booking result to UI

4. Error contract:
   - Provide conflict-specific error code/message
   - Include actionable response for reselection

## Implementation Steps

1. Add final availability check in booking command handler.
2. Implement concurrency-safe slot assignment logic.
3. Persist appointment with walk-in metadata.
4. Return structured conflict/success responses.
5. Add integration tests for collision scenarios.

## Acceptance Criteria

- [x] No double-booking occurs for the same slot
- [x] Conflict at save time is detected and blocked
- [x] Booking persists selected slot and walk-in flag on success
- [x] API provides clear conflict response for UI handling

## Dependencies

- Appointment booking backend APIs
- Conflict detection logic from EP-001

## Validation

- [x] Parallel booking simulation confirms conflict prevention
- [x] Successful booking stores slot and walk-in metadata
- [x] Conflict errors handled predictably

## Definition of Done

- Real-time conflict-safe walk-in slot assignment is implemented.
