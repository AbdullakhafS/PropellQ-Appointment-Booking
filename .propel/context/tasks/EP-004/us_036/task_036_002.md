# Task 002: Build Queue Statistics Display Panel with Threshold Warnings

**Task ID:** task_036_002  
**User Story:** US-036 (Display Queue Statistics - Wait Time, Patient Count)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Done

## Objective

Create a staff-facing queue statistics panel showing key metrics and warning state when wait-time thresholds are exceeded.

## Requirements

1. Display metrics:
   - Average wait time
   - Active patient count
   - Number of walk-ins

2. Warning indicators:
   - Show warning style when wait time exceeds configured threshold
   - Show normal state when under threshold

3. UI behavior:
   - Compact panel suitable for queue dashboard
   - Clear labels and units (minutes, counts)

4. Accessibility:
   - Status indicators with text labels, not color-only cues

## Implementation Steps

1. Build stats panel component.
2. Map metrics response to display cards/rows.
3. Add threshold state logic and visual treatment.
4. Add UI tests for warning/normal display states.

## Acceptance Criteria

- [x] All required metrics are visible in staff UI
- [x] Wait-time warning state appears at threshold breach
- [x] Normal/warning states are clearly distinguishable
- [x] Accessibility checks pass for indicator semantics

## Dependencies

- Task 036_001 metrics service
- Staff queue dashboard layout

## Validation

- [x] Warning state triggers with test threshold values
- [x] Metrics labels and values are accurate

## Definition of Done

- Queue statistics panel is implemented with threshold warnings.
