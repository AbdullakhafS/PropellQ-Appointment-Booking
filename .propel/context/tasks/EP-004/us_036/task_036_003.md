# Task 003: Integrate Real-Time Metric Updates from Queue Events

**Task ID:** task_036_003  
**User Story:** US-036 (Display Queue Statistics - Wait Time, Patient Count)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Full-Stack Developer  
**Status:** Done

## Objective

Update queue metrics in real-time when queue status changes, without requiring manual page refresh.

## Requirements

1. Real-time updates:
   - Subscribe stats panel to queue event channel (US-034)
   - Recompute or refresh metrics when relevant events arrive

2. Update strategy:
   - Keep updates lightweight to avoid UI jitter
   - Debounce/coalesce rapid event bursts if needed

3. Consistency:
   - Ensure panel remains aligned with visible queue data

## Implementation Steps

1. Add real-time subscription hook for metrics updates.
2. Trigger metrics refresh/recalculation on queue events.
3. Optimize update frequency for performance.
4. Add integration tests for live metric updates.

## Acceptance Criteria

- [x] Metrics update automatically on queue changes
- [x] No manual refresh required for current values
- [x] Panel values stay consistent with queue state
- [x] Integration tests validate live update behavior

## Dependencies

- US-034 real-time queue updates
- Tasks 036_001 and 036_002

## Validation

- [x] Simulated queue changes update panel in near real-time
- [x] Burst update behavior remains stable

## Definition of Done

- Queue metrics refresh in real-time and remain accurate.
