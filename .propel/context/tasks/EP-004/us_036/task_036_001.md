# Task 001: Implement Queue Statistics Aggregation Service

**Task ID:** task_036_001  
**User Story:** US-036 (Display Queue Statistics - Wait Time, Patient Count)  
**Epic:** EP-004  
**Priority:** CRITICAL  
**Story Points:** 1  
**Owner Role:** Backend Developer  
**Status:** Done

## Objective

Compute core queue metrics (average wait time, active patient count, walk-in count) from current queue state and check-in timestamps.

## Requirements

1. Metrics to compute:
   - Average wait time for active queue items
   - Active patient count
   - Current walk-in count

2. Data source:
   - Current queue state + appointment/check-in timestamps

3. Aggregation behavior:
   - Exclude completed/canceled items from active metrics
   - Handle missing timestamps safely

4. Performance:
   - Lightweight computation suitable for frequent updates

## Implementation Steps

1. Define metrics service contract.
2. Implement aggregation queries/calculations.
3. Add guardrails for incomplete data rows.
4. Add unit tests for calculation correctness.

- [x] Walk-in count included in output
- [x] Aggregation handles edge cases safely
- [x] Unit tests cover metric formulas
