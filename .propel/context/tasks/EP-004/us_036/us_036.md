# US-036: Display Queue Statistics (Wait Time, Patient Count)

**Story ID:** US-036  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** MEDIUM  
**Story Points:** 2  
**Owner Role:** Product / Analytics Engineer  
**Status:** Draft

## User Story

As a **clinic supervisor**, I want to see queue statistics such as average wait time and active patient count so that I can make staffing and throughput decisions quickly.

## Business Value

- Provides quick operational insights for staff
- Helps identify bottlenecks in same-day workflows
- Supports data-driven decisions for staffing adjustments

## Scope

In scope:
- Display of queue metrics in the staff portal
- Average wait time, active patients, and queued appointments
- Visual indicators for queue health and target thresholds

Out of scope:
- Historical trend analytics dashboard
- Machine-learning based predictions

## Acceptance Criteria

1. Given the queue is open, then the system displays average wait time, active patient count, and number of walk-ins.
2. Given queue status changes, then the statistics update in real-time.
3. Given wait time exceeds threshold, then the UI indicates a warning state.
4. Given the queue metrics are visible, then staff can use them to prioritize urgent actions.

## Technical Notes

- Derive statistics from current queue state and check-in timestamps.
- Keep metric updates lightweight to avoid performance issues.
- Provide threshold values for warning and normal states.

## Dependencies

- Queue status and check-in event data
- Real-time queue updates from US-034

## Definition of Done

- Queue statistics display implemented in staff UI
- Metrics update automatically as queue state changes
- Warning indicators shown for high wait times
- Acceptance criteria validated with sample data
