# Task 003: Expose Arrival Timestamp and Status in Appointment APIs

Task ID: task_038_003  
User Story: US-038 (Record Check-In Timestamp & Status)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Backend Developer

## Objective

Expose arrival timestamp and current status in appointment and queue-related API responses.

## Requirements

1. Include arrival timestamp in appointment detail/list DTOs.
2. Include arrival timestamp in queue payloads used by staff UI.
3. Keep response contracts consistent across endpoints.
4. Maintain backward compatibility where possible.

## Implementation Steps

1. Update API DTOs and serializers.
2. Map model fields into queue and appointment endpoints.
3. Add API contract tests for arrived and non-arrived records.
4. Update API documentation examples.

## Acceptance Criteria

- [x] Arrival timestamp appears in relevant API responses.
- [x] Status and timestamp values are accurate.
- [x] API tests cover both arrived and non-arrived states.

## Dependencies

- Task 038_001 and task 038_002.
- Queue API contracts.

## Validation
