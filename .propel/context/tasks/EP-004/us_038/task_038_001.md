# Task 001: Add Arrival Timestamp Field and Status Support in Appointment Model

Task ID: task_038_001  
User Story: US-038 (Record Check-In Timestamp & Status)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 1  
Owner Role: Backend Developer / Data Engineer  
Status: Done

## Objective

Extend the appointment data model to persist arrival timestamps in UTC and support consistent Arrived status semantics.

## Requirements

1. Add arrival_timestamp_utc field to appointment entity/table.
2. Ensure appointment status supports Arrived state consistently.
3. Backward compatibility for non-checked-in records.
4. Migration default handling for existing appointments.

## Implementation Steps

1. Create DB migration for arrival timestamp column.
2. Update ORM/entity mappings.
3. Ensure status enum includes Arrived and matches workflow constants.
4. Add tests for schema and model mapping.

## Acceptance Criteria

- [x] Appointment model includes UTC arrival timestamp field.
- [x] Arrived status is available and consistent in model.
- [x] Existing records remain valid after migration.
- [x] Automated tests confirm mapping correctness.

## Dependencies

- Appointment schema and migration tooling.

## Validation

- [x] New check-in records can persist arrival timestamp.
- [x] Existing non-arrived appointments are unaffected.
