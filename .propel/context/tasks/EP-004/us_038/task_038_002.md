# Task 002: Implement Atomic Check-In Update (Status + Timestamp)

Task ID: task_038_002  
User Story: US-038 (Record Check-In Timestamp & Status)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 1  
Owner Role: Backend Developer  
Status: Not Started

## Objective

Guarantee that check-in updates Arrived status and arrival timestamp atomically in one transactional operation.

## Requirements

1. On check-in, set status to Arrived.
2. On check-in, persist arrival_timestamp_utc in UTC.
3. Ensure atomic write semantics to avoid partial updates.
4. Prevent duplicate or invalid transitions.

## Implementation Steps

1. Update check-in command/service to set both fields together.
2. Wrap state transition in transaction.
3. Add guard for repeat check-in behavior.
4. Add unit/integration tests for atomicity.

## Acceptance Criteria

- [ ] Check-in always writes Arrived plus timestamp together.
- [ ] Partial writes cannot occur.
- [ ] Duplicate check-in handling is defined and enforced.
- [ ] Tests validate transactional behavior.

## Dependencies

- Task 038_001 model update.
- Check-in workflow from US-037.

## Validation

- [ ] Simulated failure rollback leaves record unchanged.
- [ ] Valid check-in persists both fields correctly.

## Definition of Done

- Atomic status and timestamp check-in logic is implemented.
