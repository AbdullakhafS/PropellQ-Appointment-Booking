# US-038: Record Check-In Timestamp & Status

**Story ID:** US-038  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** MEDIUM  
**Story Points:** 2  
**Owner Role:** Platform / Data Engineer  
**Status:** Draft

## User Story

As a **clinic operations analyst**, I want each check-in action to record a timestamp and status so that patient flow metrics and arrival order can be measured accurately.

## Business Value

- Enables operational analytics on wait times and arrival patterns
- Supports SLA tracking for appointment adherence
- Improves queue visibility and patient flow management

## Scope

In scope:
- Persisting arrival timestamp on check-in
- Updating appointment status to reflect arrival
- Exposing status and arrival time in queue and appointment views

Out of scope:
- Historical analytics beyond current day
- Billing or encounter documentation based on arrival time

## Acceptance Criteria

1. Given a patient is checked in, then the appointment record saves an arrival timestamp.
2. Given check-in occurs, then the appointment status updates to "Arrived".
3. Given staff reviews the appointment, then they can see arrival time and current status.
4. Given queue or reporting UIs request appointment metadata, then arrival timestamp is included.

## Technical Notes

- Use UTC timestamps for arrival time storage.
- Keep status values consistent with the appointment workflow.
- Ensure timestamp updates are atomic with status transition.

## Dependencies

- Appointment data model
- Check-in workflow from US-037

## Definition of Done

- Arrival timestamp and status persisted for check-ins
- Queue and appointment views include the timestamp
- Acceptance criteria verified with sample check-in events
