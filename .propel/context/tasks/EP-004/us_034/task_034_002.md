# Task 002: Publish Queue Events from Appointment and Walk-In Workflows

**Task ID:** task_034_002  
**User Story:** US-034 (Build Real-Time Queue Management UI)  
**Epic:** EP-004  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Backend Developer  
**Status:** Not Started

## Objective

Emit queue update events whenever same-day appointments, walk-ins, or check-in statuses change.

## Requirements

1. Event triggers:
   - New walk-in appointment created
   - Check-in status change
   - Queue-relevant appointment status updates

2. Event consistency:
   - Ensure emitted data reflects committed state (post-transaction publish)
   - Avoid duplicate or out-of-order updates where possible

3. Payload content:
   - appointment_id, patient_name, provider, appointment_type, scheduled_time, status, walk_in flag

4. Performance:
   - Keep payload minimal for high-concurrency clinics

## Implementation Steps

1. Hook event publishing into appointment lifecycle handlers.
2. Add post-commit publisher logic to avoid stale payloads.
3. Implement idempotency/sequence metadata if needed.
4. Add integration tests for event emission.

## Acceptance Criteria

- [ ] Queue events emitted for all queue-relevant mutations
- [ ] Event payloads contain required UI fields
- [ ] Duplicate/stale event behavior controlled
- [ ] Integration tests validate trigger coverage

## Dependencies

- Task 034_001 real-time channel
- EP-001/EP-004 appointment status flows

## Validation

- [ ] Walk-in creation emits queue-add event
- [ ] Check-in/status change emits queue-update event
- [ ] Event payload matches expected schema

## Definition of Done

- Queue event publishing is integrated across key appointment workflows.
