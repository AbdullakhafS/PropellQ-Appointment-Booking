# Task 001: Implement Real-Time Queue Event Channel

**Task ID:** task_034_001  
**User Story:** US-034 (Build Real-Time Queue Management UI)  
**Epic:** EP-004  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Backend Developer  
**Status:** Done

## Objective

Establish a real-time channel (WebSocket/SSE) for broadcasting queue updates to all authorized staff sessions.

## Requirements

1. Real-time transport:
   - Implement WebSocket or SSE endpoint for queue events
   - Authenticate and authorize staff connections

2. Event model:
   - Define minimal payload for queue changes (created/updated/status-changed)
   - Include identifiers needed for efficient client patching

3. Session sync:
   - Broadcast queue updates to all connected staff clients in same clinic context

4. Reliability:
   - Support reconnect handling and heartbeat/ping strategy

## Implementation Steps

1. Choose transport pattern (WebSocket or SSE) per platform standard.
2. Implement connection endpoint and auth gate.
3. Define queue event payload contract.
4. Implement event broadcaster/service.
5. Add tests for connection and publish flows.

## Acceptance Criteria

- [x] Real-time endpoint established and secured
- [x] Staff clients can connect and receive queue events
- [x] Broadcast reaches all active staff sessions
- [x] Reconnect path works for dropped connections

## Dependencies

- Staff authentication and permissions
- Queue state service

## Validation

- [x] Two-session test confirms synchronized event delivery
- [x] Unauthorized connection attempts rejected

## Definition of Done

- Real-time queue event channel is available and stable.
