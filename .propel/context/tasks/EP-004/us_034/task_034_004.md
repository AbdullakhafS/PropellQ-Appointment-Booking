# Task 004: Implement Multi-Session Queue Synchronization and Permissions

**Task ID:** task_034_004  
**User Story:** US-034 (Build Real-Time Queue Management UI)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Full-Stack Developer  
**Status:** Done

## Objective

Ensure queue updates synchronize correctly across multiple staff sessions with proper role-based visibility controls.

## Requirements

1. Session synchronization:
   - Changes made in one session appear in other active sessions in real-time
   - Scope updates by clinic/location where applicable

2. Permissions:
   - Only authorized staff can subscribe and view queue events
   - Enforce role checks consistently in API and real-time channel

3. Data isolation:
   - Prevent cross-clinic event leakage
   - Verify tenancy/organization boundaries

## Implementation Steps

1. Add clinic/role scoping to real-time subscriptions.
2. Enforce permission checks in event channel and queue APIs.
3. Add multi-session integration tests.
4. Validate isolation across tenant/clinic contexts.

## Acceptance Criteria

- [x] Multi-session updates synchronize correctly
- [x] Unauthorized users cannot receive queue updates
- [x] Cross-clinic data isolation is enforced
- [x] Security tests cover subscription authorization

## Dependencies

- Tasks 034_001 through 034_003
- Staff RBAC and tenant context

## Validation

- [x] Two concurrent staff sessions stay in sync
- [x] Cross-clinic visibility blocked as expected

## Definition of Done

- Queue synchronization is secure, scoped, and reliable.
