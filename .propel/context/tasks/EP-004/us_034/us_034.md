# US-034: Build Real-Time Queue Management UI

**Story ID:** US-034  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Platform Engineer  
**Status:** Draft

## User Story

As a **clinic operations staff member**, I want a real-time appointment queue interface so that I can monitor same-day patients, walk-ins, and check-ins without refreshing the page.

## Business Value

- Enables accurate, up-to-date queue management
- Reduces delays caused by stale schedule views
- Enhances staff collaboration across front desk and clinical teams

## Scope

In scope:
- Real-time queue updates for same-day appointments and walk-ins
- Queue display with patient, provider, status, and wait-time summary
- Support for refreshless updates across staff sessions

Out of scope:
- Real-time patient-facing notifications
- Machine learning queue prioritization

## Acceptance Criteria

1. Given multiple staff users are logged in, when any user updates the queue, then all sessions see the queue change in real-time.
2. Given a new walk-in or check-in event occurs, then the queue updates without manual refresh.
3. Given queue items change status, then the UI reflects the current status immediately.
4. Given the queue displays appointments, then each item shows patient name, appointment type, provider, scheduled time, and current status.

## Technical Notes

- Leverage WebSockets, Server-Sent Events, or real-time messaging infrastructure.
- Synchronize queue state across staff clients.
- Keep payloads minimal to preserve performance under high concurrency.

## Dependencies

- Queue state management service
- Staff authentication and permissions
- Appointment status updates from EP-001/EP-004 flows

## Definition of Done

- Real-time queue UI built and connected to live updates
- Queue item metadata displayed accurately
- Queue refreshes automatically on status changes
- Tested with at least two concurrent staff sessions
