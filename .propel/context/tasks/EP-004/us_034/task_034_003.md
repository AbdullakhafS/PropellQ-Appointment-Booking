# Task 003: Build Real-Time Queue UI with Live State Updates

**Task ID:** task_034_003  
**User Story:** US-034 (Build Real-Time Queue Management UI)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Implement the queue interface that renders same-day patients and applies real-time updates without page refresh.

## Requirements

1. Queue item display:
   - Patient name
   - Appointment type
   - Provider
   - Scheduled time
   - Current status
   - Wait-time summary where available

2. Live updates:
   - Subscribe to queue event channel
   - Insert/update/remove queue items based on event type
   - No manual refresh required

3. UI states:
   - Loading state for initial queue load
   - Connection status indicator (live/reconnecting)
   - Graceful fallback if realtime channel is unavailable

4. Accessibility:
   - Semantic table/list structure
   - Screen-reader friendly status updates

## Implementation Steps

1. Build queue list/table component and item renderer.
2. Load initial queue snapshot from API.
3. Subscribe to real-time event stream and patch local state.
4. Add connection status and fallback behavior.
5. Add component tests for live update rendering.

## Acceptance Criteria

- [ ] Queue shows required metadata fields
- [ ] New/updated queue items appear without refresh
- [ ] Status changes reflect immediately in UI
- [ ] Connection disruptions are surfaced clearly
- [ ] Accessibility checks pass for queue display

## Dependencies

- Task 034_001 and task_034_002
- Existing queue API for initial snapshot

## Validation

- [ ] Live update simulation confirms UI patch behavior
- [ ] Connection loss/reconnect behavior verified

## Definition of Done

- Real-time queue UI is functional and refreshless.
