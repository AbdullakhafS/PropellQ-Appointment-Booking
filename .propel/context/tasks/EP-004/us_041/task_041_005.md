# Task 005: Send Notifications for Cancellation and Reschedule Changes

Task ID: task_041_005  
User Story: US-041 (Implement Cancellation/Reschedule Logic)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Backend Developer

## Objective

Notify affected patients and assigned providers when appointments are canceled or rescheduled.

## Requirements

1. Send patient notification on cancellation.
2. Send patient and provider notifications on reschedule.
3. Include updated appointment details in messages.
4. Track delivery outcome for auditing.

## Implementation Steps

1. Integrate cancel/reschedule handlers with notification service.
2. Build template payloads for each change type.
3. Add retry/error handling for notification dispatch failures.
4. Add integration tests for notification triggers.

## Acceptance Criteria

- [x] Notifications are sent for cancellation and reschedule events.
- [x] Message payload includes correct updated details.
- [x] Delivery outcomes are recorded.
- [x] Integration tests validate notification behavior.

## Dependencies

- Notification service.
- Tasks 041_001 and 041_002.

## Validation

- [x] Notification logs show expected recipients and payloads.

## Definition of Done

- Notification delivery for schedule changes is implemented and verifiable.
