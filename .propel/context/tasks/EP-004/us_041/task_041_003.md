# Task 003: Sync Queue and Real-Time Sessions on Cancellation/Reschedule

Task ID: task_041_003

Epic: EP-004  
Priority: HIGH

Owner Role: Full-Stack Developer

## Objective

Ensure queue and schedule views update immediately across staff sessions when appointments are canceled or rescheduled.

## Requirements

1. Emit queue/schedule update events for cancellation and reschedule actions.
2. Patch active client sessions without manual refresh.
3. Keep queue ordering and status display consistent post-update.

## Implementation Steps

1. Hook cancel/reschedule handlers to event publisher.
2. Update client subscribers to process change events.
3. Add integration tests for multi-session synchronization.

## Acceptance Criteria

- [ ] Queue updates immediately after cancellation/reschedule.
- [ ] Multi-session views stay synchronized.
- [ ] No stale entries remain after update events.

## Dependencies

- US-034 real-time event infrastructure.
- Tasks 041_001 and 041_002.

## Validation

- [ ] Two-session test confirms synchronized updates.

## Definition of Done

- Queue synchronization is consistent for cancel/reschedule operations.
