# Task 001: Implement Staff Waitlist Join Flow for Full Slots

Task ID: task_039_001  
User Story: US-039 (Implement Waitlist Join & Accept/Decline Flow)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 2  
Owner Role: Frontend/Backend Developer  
Status: Done

## Objective

Allow staff to add patients to a waitlist when desired appointment slots are full.

## Requirements

1. Add waitlist action when slot unavailable/full.
2. Capture minimum waitlist entry data (patient, provider/clinic, preferred slot context).
3. Persist waitlist entry with queued state and timestamp.
4. Prevent duplicate active waitlist entries for same slot/patient.

## Implementation Steps

1. Add waitlist join UI action in booking flow.
2. Implement waitlist create endpoint.
3. Add duplicate-check guard.
4. Add tests for successful join and duplicate prevention.

## Acceptance Criteria

- [x] Staff can add patient to waitlist for full slot.
- [x] Waitlist entry is saved with queued state.
- [x] Duplicate active entries are prevented.
- [x] Join flow works without leaving booking context.

## Dependencies

- Appointment scheduling engine.
- Staff auth from EP-005.

## Validation

- [x] Waitlist DB record created with correct context.
- [x] Duplicate entry scenario handled correctly.

## Definition of Done

- Waitlist join flow is functional and reliable.
