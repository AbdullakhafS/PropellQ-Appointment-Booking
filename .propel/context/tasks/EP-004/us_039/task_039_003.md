# Task 003: Build Accept/Decline Response Endpoints and State Updates

Task ID: task_039_003  
User Story: US-039 (Implement Waitlist Join & Accept/Decline Flow)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Backend Developer  
Status: Done

## Objective

Enable accept/decline responses for waitlist offers and update offer/waitlist states accordingly.

## Requirements

1. Response endpoints/actions for accept and decline.
2. Validate response only for pending, non-expired offers.
3. Accept path marks offer accepted and initiates booking conversion.
4. Decline path marks offer declined and advances waitlist progression.

## Implementation Steps

1. Implement accept/decline API handlers.
2. Add pending/expiry guards.
3. Wire accept to booking conversion workflow.
4. Wire decline to next-candidate flow.
5. Add integration tests.

## Acceptance Criteria

- [x] Pending offers can be accepted or declined.
- [x] Expired/non-pending offers cannot be responded to.
- [x] Accept and decline states persist correctly.
- [x] Integration tests validate response outcomes.

## Dependencies

- Task_039_002 offer lifecycle.
- Appointment booking service.

## Validation

- [x] Accept response updates offer and waitlist state.
- [x] Decline response triggers proper fallback state.

## Definition of Done

- Accept/decline response flow is implemented and robust.
