# Task 002: Implement Waitlist Offer Model and Expiry Window

Task ID: task_039_002  
User Story: US-039 (Implement Waitlist Join & Accept/Decline Flow)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 2  
Owner Role: Backend Developer  
Status: Done

## Objective

Create the waitlist offer lifecycle model with pending, accepted, declined, and expired states including a 30-minute response window.

## Requirements

1. Offer states: pending, accepted, declined, expired.
2. Store created_at and expires_at for each offer.
3. Default expiry window: 30 minutes.
4. Link offers to waitlist entries and target slot.

## Implementation Steps

1. Add waitlist offer schema/entities.
2. Implement state transition rules.
3. Add expiry calculation and persistence logic.
4. Add tests for valid/invalid transitions.

## Acceptance Criteria

- [x] Offers persist with explicit lifecycle states.
- [x] Expiry timestamp is set to 30-minute window.
- [x] State transitions are validated and auditable.
- [x] Tests cover lifecycle transitions.

## Dependencies

- Waitlist entry model from task_039_001.
- Notification pipeline integration points.

## Validation

- [x] Offer record includes expiry and current state.
- [x] Invalid transition attempts are rejected.

## Definition of Done

- Waitlist offer lifecycle model is implemented and tested.
