## Objective

Handle 30-minute offer timeout and automatically progress to next eligible waitlist patient.

## Requirements

1. Detect expired pending offers.
2. Mark expired offers and update waitlist state.
3. Trigger next patient offer according to ordering rules.
4. Stop gracefully if no eligible patients remain.

## Implementation Steps

1. Implement expiry scheduler or event-driven checker.
2. Update expired offers and linked waitlist entries.
3. Invoke candidate progression logic.
4. Add tests for chained expiry progression.

## Acceptance Criteria

- [ ] Offers expire after 30 minutes if no response.
- [ ] Next eligible patient receives follow-up offer.
- [ ] No-candidate path is handled without failure.
- [ ] Tests validate timeout progression.

## Dependencies

- Task_039_002 offer lifecycle.
- Candidate selection logic.

## Validation

- [ ] Timeout simulation confirms expiry and next-offer behavior.
- [ ] Progression remains consistent across multiple expiries.

## Definition of Done
