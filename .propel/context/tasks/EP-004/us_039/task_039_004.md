# Task 004: Convert Accepted Waitlist Offer to Confirmed Appointment

Task ID: task_039_004  
User Story: US-039 (Implement Waitlist Join & Accept/Decline Flow)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Backend Developer  
Status: Not Started

## Objective

On accepted offer, create or confirm appointment booking and close corresponding waitlist entry.

## Requirements

1. Confirm slot assignment for accepted offer.
2. Create/confirm appointment record transactionally.
3. Mark waitlist entry fulfilled and offer accepted.
4. Prevent duplicate booking on repeated accept attempts.

## Implementation Steps

1. Implement transactional accept-to-book pipeline.
2. Add idempotency guard for repeated accepts.
3. Persist final states across offer/waitlist/appointment records.
4. Add integration tests for success and conflict scenarios.

## Acceptance Criteria

- [ ] Accepted offers convert to confirmed appointments.
- [ ] Waitlist entry transitions to fulfilled state.
- [ ] Repeated accept attempts do not duplicate bookings.
- [ ] Integration tests pass.

## Dependencies

- Task_039_003 response endpoints.
- Scheduling and conflict detection services.

## Validation

- [ ] Confirmed appointment contains correct patient and slot.
- [ ] Waitlist and offer states remain consistent.

## Definition of Done

- Accepted waitlist offers reliably become confirmed appointments.
