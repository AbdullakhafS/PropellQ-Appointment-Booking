# US-042: Implement Waitlist Processing on Slot Release

**Story ID:** US-042  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** Platform / Automation Engineer  
**Status:** Draft

## User Story

As a **clinic operations manager**, I want the system to process waitlist entries automatically when a slot is released so that available capacity is offered to patients without manual intervention.

## Business Value

- Increases conversion of open slots to confirmed visits
- Reduces staff burden for manual waitlist processing
- Ensures fast patient notification when availability changes

## Scope

In scope:
- Detect slot release events from cancellations or reschedules
- Evaluate waitlist entries and select eligible candidates
- Create offers and update waitlist state automatically

Out of scope:
- Complex prioritization heuristics beyond queue order
- Patient-specific eligibility scoring beyond basic rules

## Acceptance Criteria

1. Given a slot is released, then the waitlist processor evaluates eligible entries.
2. Given an eligible patient is found, then the system creates an offer and marks the waitlist entry as pending.
3. Given a waitlist patient accepts, then the appointment is confirmed.
4. Given an offer expires, then the next eligible patient is offered the slot.

## Technical Notes

- Use event-driven processing when slot allocations change.
- Keep the waitlist processor idempotent for repeated slot release events.
- Prefer first-in-first-out ordering with ability to skip ineligible patients.

## Dependencies

- Appointment cancellation/reschedule processing from US-041
- Waitlist join and offer flow from US-039 and US-040
- Notification service for offer delivery

## Definition of Done

- Waitlist processing is triggered on slot release
- Eligible candidates receive offers automatically
- Offer expiry and fallback logic works correctly
- Acceptance criteria validated through system tests
