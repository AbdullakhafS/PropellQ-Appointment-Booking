## Objective

Validate full automation pipeline from slot release to offer creation, acceptance/expiry handling, and fallback candidate progression.

## Requirements

1. End-to-end scenarios:
   - Slot release creates pending offer
   - Offer accepted leads to confirmed appointment
   - Offer expired leads to next candidate offer
2. Idempotency checks:
   - Replayed release events do not duplicate offers
3. Observability checks:
   - Logs and status transitions are traceable end-to-end

1. Build test matrix mapped to acceptance criteria.
2. Execute integration/system tests for primary scenarios.
3. Execute replay/idempotency tests.
4. Validate state transitions across waitlist and offers.
5. Produce QA sign-off and defect report.
