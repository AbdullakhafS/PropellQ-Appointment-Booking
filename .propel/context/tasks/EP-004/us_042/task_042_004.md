# Task 004: Implement Offer Acceptance and Slot Confirmation Flow

Task ID: task_042_004

Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Full-Stack Developer  
Status: Done

## Objective

Convert accepted waitlist offers into confirmed appointments safely and update waitlist and slot states.

## Requirements

1. Acceptance handling:
   - On accept, reserve and confirm slot for offer holder
   - Mark offer state accepted
   - Mark waitlist entry fulfilled/closed
2. Conflict safety:
   - Reject acceptance if slot already taken
   - Return clear fallback response for next step
3. Data consistency:
   - Transactionally update offer, appointment, and waitlist state

## Implementation Steps

1. Implement offer acceptance endpoint/handler.
2. Add transactional slot confirmation logic.
3. Handle already-filled-slot conflicts.
4. Add integration tests for success and conflict cases.

## Acceptance Criteria

- [x] Accepted offers confirm appointments successfully.
- [x] Waitlist and offer states update consistently.
- [x] Slot conflict on accept is handled safely.
- [x] Integration tests validate state transitions.

## Dependencies

- Task 042_003 offer creation pipeline.
- Appointment booking confirmation APIs.

## Validation

- [x] Acceptance creates confirmed appointment record.
- [x] Conflict acceptance test returns expected response.

## Definition of Done

- Offer acceptance flow reliably confirms released slots.
