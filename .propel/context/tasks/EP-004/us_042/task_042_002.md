# Task 002: Implement Idempotent Waitlist Candidate Selection Engine

Task ID: task_042_002  
User Story: US-042 (Implement Waitlist Processing on Slot Release)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 2  
Owner Role: Backend Developer  
Status: Not Started

## Objective

Select eligible waitlist candidates in FIFO order with skip support for ineligible entries, while remaining idempotent for repeated processing events.

## Requirements

1. Candidate ordering:
   - FIFO by waitlist join time
2. Eligibility checks:
   - Basic rules only (slot compatibility, active waitlist state)
   - Skip ineligible entries and continue search
3. Idempotency:
   - Reprocessing same slot event does not duplicate offers
4. State transitions:
   - Move selected entry to pending-offer state

## Implementation Steps

1. Implement waitlist query strategy with FIFO ordering.
2. Add eligibility evaluator and skip logic.
3. Add idempotency keys/guards per slot release cycle.
4. Persist candidate state transition transactionally.
5. Add unit and integration tests for repeated events.

## Acceptance Criteria

- [ ] Eligible candidate selection follows FIFO rules.
- [ ] Ineligible entries are skipped correctly.
- [ ] Duplicate offers are prevented for repeated triggers.
- [ ] Candidate state transitions to pending offer.

## Dependencies

- Task 042_001 release trigger.
- Waitlist domain model from US-039 and US-040.

## Validation

- [ ] Replayed event does not create duplicate offer.
- [ ] Mixed eligible/ineligible test selects correct candidate.

## Definition of Done

- Candidate selection engine is accurate, idempotent, and tested.
