# Task 005: Implement Offer Expiry and Next-Candidate Fallback

Task ID: task_042_005  
User Story: US-042 (Implement Waitlist Processing on Slot Release)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Backend Developer  
Status: Done

## Objective

When an offer expires or is declined, automatically progress the released slot to the next eligible waitlist candidate.

## Requirements

1. Expiry handling:
   - Detect expired pending offers
   - Mark offer state expired
2. Fallback progression:
   - Re-run candidate selection for next eligible entry
   - Generate next offer automatically
3. Safety:
   - Avoid infinite retry loops
   - Stop gracefully when no eligible candidates remain

## Implementation Steps

1. Add scheduled or event-driven expiry checker.
2. Implement offer expiry state update logic.
3. Trigger next-candidate processing path.
4. Add tests for multiple fallback iterations.

## Acceptance Criteria

- [x] Expired offers transition to expired state.
- [x] Next eligible candidate receives offer automatically.
- [x] Process stops cleanly when queue is exhausted.
- [x] Tests validate multi-step fallback logic.

## Dependencies

- Tasks 042_002 and 042_003.
- Waitlist state model and offer TTL configuration.

## Validation

- [x] Expiry simulation triggers next offer.
- [x] No-candidate scenario ends without failure.

## Definition of Done

- Offer expiry and fallback progression are automated and reliable.
