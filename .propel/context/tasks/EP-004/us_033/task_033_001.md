# Task 001: Add Walk-In Flag to Appointment Data Model

**Task ID:** task_033_001  
**User Story:** US-033 (Flag Walk-In Appointments)  
**Epic:** EP-004  
**Priority:** CRITICAL  
**Story Points:** 1  
**Owner Role:** Backend Developer / DBA  
**Status:** Not Started

## Objective

Extend the appointment model to store walk-in status as explicit metadata that persists across updates and moves.

## Requirements

1. Data model update:
   - Add `is_walk_in` boolean (or enum) to appointment record
   - Default value false for non-walk-in appointments

2. Migration and compatibility:
   - Backfill existing records with non-walk-in default
   - Ensure no breaking changes to existing booking flows

3. Update behavior:
   - Walk-in flag must persist on appointment edits and reschedules
   - Flag cannot be dropped unintentionally during update operations

## Implementation Steps

1. Add schema migration for walk-in column/field.
2. Update appointment entity and ORM mapping.
3. Update create/update command handlers to preserve flag.
4. Add unit tests for default and persistence behavior.

## Acceptance Criteria

- [ ] Appointment model includes walk-in flag
- [ ] Existing data migration completes safely
- [ ] Walk-in flag persists through updates and moves
- [ ] Unit tests validate flag lifecycle behavior

## Dependencies

- Appointment service schema management
- Existing appointment update workflows

## Validation

- [ ] New walk-in appointment stores `is_walk_in=true`
- [ ] Existing appointment defaults to `is_walk_in=false`
- [ ] Update/reschedule retains walk-in value

## Definition of Done

- Walk-in flag is fully supported at the appointment data-model layer.
