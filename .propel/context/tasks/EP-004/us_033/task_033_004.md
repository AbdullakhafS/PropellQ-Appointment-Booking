# Task 004: Add Walk-In Filtering and Search Support

**Task ID:** task_033_004

**Epic:** EP-004  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** Full-Stack Developer

## Objective

Enable staff to filter queue and schedule data to view only walk-in appointments.

## Requirements

1. Filter controls:
   - Add Walk-In filter option in queue and schedule views
   - Support toggling between all, walk-in-only, and non-walk-in (if needed)

2. API/query support:
   - Pass walk-in filter parameter to backend endpoints
   - Backend returns filtered result set accurately

3. UX behavior:
   - Preserve filter selection while navigating within view
   - Show empty state if no walk-ins match current context

## Implementation Steps

1. Add UI filter controls for walk-in flag.
2. Add query parameter handling in API layer.
3. Implement backend filtering logic where required.
4. Add tests for filter combinations and empty states.

## Acceptance Criteria

- [x] Walk-in filter returns only walk-in appointments
- [x] Default view remains unchanged when filter off
- [x] Filter state behaves consistently across reload/navigation
- [x] Filter tests pass
