# US-033: Flag Walk-In Appointments

**Story ID:** US-033  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** MEDIUM  
**Story Points:** 2  
**Owner Role:** Platform / Product Engineer  
**Status:** Draft

## User Story

As a **clinic manager**, I want walk-in appointments to be clearly identified in the schedule so that staff can differentiate them from pre-booked appointments and adjust operational flow accordingly.

## Business Value

- Improves operational visibility for staff
- Helps triage same-day arrivals efficiently
- Enables analytics on walk-in volume and performance

## Scope

In scope:
- Walk-in flag on appointment records
- Visual indicator in the queue and schedule views
- Filter and search support for walk-in appointments

Out of scope:
- Historical walk-in analytics dashboard
- Advanced walk-in priority scoring

## Acceptance Criteria

1. Given a walk-in appointment is created, then the appointment record is marked with a "walk-in" indicator.
2. Given staff views daily schedule or queue, then walk-in appointments have a distinct visual style or badge.
3. Given staff filters for walk-ins, then only walk-in appointments are returned.
4. Given a walk-in appointment is updated or moved, then the walk-in flag persists.

## Technical Notes

- Store a boolean or enumerated type for walk-in status in appointment metadata.
- Ensure presentation layer displays the indicator consistently.
- Include walk-in status in API responses used by queue and schedule UIs.

## Dependencies

- Appointment data model updates
- Staff schedule and queue UI

## Definition of Done

- Walk-in flag present on appointment model
- UI indicator shown for walk-ins in schedule and queue
- Filters for walk-in appointments functional
- Basic automation tests cover walk-in flag persistence
