# Task 003: Implement Walk-In Visual Badge in Queue and Schedule UI

**Task ID:** task_033_003  
**User Story:** US-033 (Flag Walk-In Appointments)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Display a distinct visual indicator for walk-in appointments in both queue and schedule views.

## Requirements

1. Visual treatment:
   - Add walk-in badge/chip/tag style
   - Ensure indicator is distinct from other statuses

2. Coverage:
   - Show indicator in queue rows/cards
   - Show indicator in schedule list/calendar entries

3. Accessibility:
   - Ensure badge has accessible label text
   - Maintain color contrast requirements

4. Consistency:
   - Reuse shared status badge component where possible
   - Same terminology across all views (Walk-In)

## Implementation Steps

1. Create or extend status badge component.
2. Bind `is_walk_in` field to badge display logic.
3. Add badge rendering in queue and schedule components.
4. Add unit/UI tests for badge visibility behavior.

## Acceptance Criteria

- [ ] Walk-in appointments show clear badge in queue view
- [ ] Walk-in appointments show clear badge in schedule view
- [ ] Non-walk-in appointments do not show walk-in badge
- [ ] Accessibility checks pass for label and contrast

## Dependencies

- Task 033_002 API walk-in field availability
- Queue and schedule UI components

## Validation

- [ ] Visual QA confirms badge in all relevant placements
- [ ] Screen reader announces walk-in status where appropriate

## Definition of Done

- Walk-in indicator is implemented consistently across queue and schedule UI.
