# Task 005: Validate Walk-In Flag Persistence and Staff Workflow

**Epic:** EP-004  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** QA Engineer / Full-Stack Developer

## Objective

Validate end-to-end walk-in flag behavior from appointment creation through update, queue/schedule display, and filtering.

## Requirements

1. E2E scenarios:
   - Create walk-in appointment and confirm flag set
   - Update/move walk-in appointment and confirm flag persists
   - Confirm badge appears in queue and schedule
   - Apply walk-in filter and verify result set

2. Data integrity:
   - Walk-in flag stored correctly in appointment record
   - API responses carry consistent walk-in status

3. Regression safety:
   - Non-walk-in appointments unaffected by new logic
   - Existing queue/schedule workflows still function

## Implementation Steps

1. Build test matrix from acceptance criteria.
2. Execute integration tests for create/update/filter/display flows.
3. Validate API payloads and DB records.
4. Log and retest defects.
5. Publish QA sign-off summary.

- [x] Walk-in indicator visible where expected
- [x] Walk-in filtering works accurately
