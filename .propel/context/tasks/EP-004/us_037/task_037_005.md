## Objective

Validate full staff check-in workflow from action click to persisted arrival data and queue updates.

## Requirements

1. E2E scenarios for same-day and walk-in appointments.
2. Verify status transition and arrival timestamp persistence.
3. Verify queue and real-time session updates.
4. Verify error handling for invalid transitions.

## Implementation Steps

1. Build test matrix mapped to acceptance criteria.
2. Execute integration and manual tests.
3. Validate API responses, DB records, and queue UI.
4. Capture defects and retest after fixes.

## Acceptance Criteria

- [ ] Staff can check in eligible appointments successfully.
- [ ] Arrived status and arrival time are persisted correctly.
- [ ] Queue updates are visible in real-time.
- [ ] Invalid operations are blocked and clearly reported.

## Dependencies

- Tasks 037_001 through 037_004.

## Validation

- [ ] QA evidence covers each acceptance criterion.
- [ ] No critical defects remain open.

## Definition of Done

- US-037 check-in workflow is validated and release-ready.
