# Task 003: Update Queue Rendering for Arrived Status and Arrival Time

Task ID: task_037_003  
User Story: US-037 (Implement Patient Check-In Workflow)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Frontend Developer  
Status: Not Started

## Objective

Ensure queue UI shows checked-in patients with Arrived status and arrival timestamp clearly.

## Requirements

1. Display Arrived status style for checked-in appointments.
2. Show arrival time in queue row/card metadata.
3. Keep visual consistency with other status badges.
4. Support mobile and desktop queue views.

## Implementation Steps

1. Extend queue item renderer with arrival timestamp field.
2. Add Arrived status styling and label.
3. Handle missing arrival time gracefully for non-arrived items.
4. Add UI tests for status and timestamp rendering.

## Acceptance Criteria

- [ ] Queue reflects Arrived status after check-in.
- [ ] Arrival time is visible for checked-in patients.
- [ ] Non-arrived items remain unaffected.
- [ ] UI tests validate rendering behavior.

## Dependencies

- Check-in API payload from task_037_002.
- Queue UI foundation from EP-004 and US-034.

## Validation

- [ ] Check-in action updates queue item display in real-time.
- [ ] Visual QA confirms readable timestamp format.

## Definition of Done

- Queue accurately displays checked-in state and arrival time.
