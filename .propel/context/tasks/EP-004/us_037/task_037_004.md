## Objective

Publish check-in changes to real-time listeners and trigger related waitlist or slot allocation updates.

## Requirements

1. Emit queue event when check-in is completed.
2. Update all active staff sessions without refresh.
3. Trigger dependent waitlist or slot-allocation logic if required.
4. Ensure event payload includes status and arrival time.

## Implementation Steps

1. Hook check-in completion to event publisher.
2. Add event consumer updates in queue clients.
3. Integrate any downstream waitlist/allocation hooks.
4. Add integration tests for event propagation.

## Acceptance Criteria

- [ ] Check-in events are broadcast to queue listeners.
- [ ] Other staff sessions reflect updates in real-time.
- [ ] Related waitlist or allocation logic receives updates.
- [ ] Integration tests validate event flow.

## Dependencies

- Task_037_002 check-in backend logic.

## Validation

- [ ] Downstream workflow update is verifiable.
