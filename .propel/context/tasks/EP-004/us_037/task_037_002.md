# Task 002: Implement Appointment Check-In API and Status Transition Logic

Task ID: task_037_002  
User Story: US-037 (Implement Patient Check-In Workflow)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 1  
Owner Role: Backend Developer  
Status: Not Started

## Objective

Implement backend check-in command that transitions appointment status to Arrived and records arrival timestamp.

## Requirements

1. Add check-in endpoint for appointment status update.
2. Enforce valid status transition path (Scheduled to Arrived).
3. Persist arrival timestamp on successful check-in.
4. Return updated appointment payload for UI refresh.

## Implementation Steps

1. Define endpoint contract for check-in action.
2. Implement transition validation in service layer.
3. Persist arrival_time and updated status transactionally.
4. Add API tests for valid and invalid transitions.

## Acceptance Criteria

- [ ] Check-in endpoint updates status to Arrived.
- [ ] Arrival timestamp is recorded for each check-in event.
- [ ] Invalid transitions are rejected with clear errors.
- [ ] API tests cover transition logic and timestamp persistence.

## Dependencies

- Appointment state model and repository.
- Staff authorization context from EP-005.

## Validation

- [ ] Database record shows Arrived plus arrival timestamp.
- [ ] Transition validation prevents illegal state changes.

## Definition of Done

- Check-in API and transition rules are implemented and tested.
