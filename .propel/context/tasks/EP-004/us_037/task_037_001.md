# Task 001: Build Check-In Action in Staff Appointment View

Task ID: task_037_001  
User Story: US-037 (Implement Patient Check-In Workflow)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 1  
Owner Role: Frontend Developer  
Status: Not Started

## Objective

Enable front-desk staff to perform patient check-in directly from same-day and walk-in appointment views.

## Requirements

1. Add a visible Check In action on eligible appointments.
2. Restrict action based on appointment status rules.
3. Prevent duplicate check-in submissions.
4. Provide immediate success or error feedback.

## Implementation Steps

1. Add check-in button/action to appointment row and detail panel.
2. Gate action for allowed statuses (for example Scheduled).
3. Disable action during in-flight request.
4. Show completion toast/banner with updated status.
5. Add component tests for action visibility and state transitions.

## Acceptance Criteria

- [ ] Staff can trigger check-in from appointment UI.
- [ ] Check-in action is only shown for eligible appointments.
- [ ] Duplicate submissions are prevented.
- [ ] Success and failure feedback is clear.

## Dependencies

- Appointment list/detail UI.
- Backend check-in endpoint from task_037_002.

## Validation

- [ ] UI tests pass for eligible and ineligible states.
- [ ] Manual check confirms action usability in front-desk workflow.

## Definition of Done

- Check-in action is available, reliable, and user-friendly in staff UI.
