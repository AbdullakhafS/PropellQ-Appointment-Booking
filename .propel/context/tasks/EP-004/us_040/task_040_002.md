# Task 002: Implement First-Eligible FIFO Candidate Selection

Task ID: task_040_002  
User Story: US-040 (Auto-Offer First Waitlist Patient)  
Epic: EP-004  
Priority: CRITICAL  
Story Points: 1  
Owner Role: Backend Developer  
Status: Not Started

## Objective

Select the first eligible waitlisted patient according to FIFO ordering.

## Requirements

1. Query waitlist entries in FIFO order.
2. Evaluate basic eligibility constraints.
3. Skip ineligible entries and continue selection.
4. Return first eligible candidate only.

## Implementation Steps

1. Build selection query and ordering logic.
2. Add eligibility check service.
3. Implement skip-and-continue behavior.
4. Add unit tests for mixed eligibility queues.

## Acceptance Criteria

- [ ] First eligible FIFO patient is selected.
- [ ] Ineligible entries are skipped safely.
- [ ] Selection logic is test-covered.

## Dependencies

- Waitlist entry model from US-039.

## Validation

- [ ] Selection output matches expected ordering in sample datasets.

## Definition of Done

- FIFO first-eligible selection is deterministic and reliable.
