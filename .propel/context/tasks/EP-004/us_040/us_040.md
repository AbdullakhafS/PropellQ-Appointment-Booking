# US-040: Auto-Offer First Waitlist Patient

**Story ID:** US-040  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** HIGH  
**Story Points:** 4  
**Owner Role:** Platform / Automation Engineer  
**Status:** Draft

## User Story

As a **clinic operations lead**, I want the system to automatically offer the first eligible waitlisted patient a newly available slot so that vacant capacity is filled quickly.

## Business Value

- Increases throughput by filling open slots faster
- Reduces manual coordination for waitlist management
- Improves patient experience with proactive booking offers

## Scope

In scope:
- Automatic detection of released slots and waitlist eligibility
- Offering the first eligible patient in FIFO order
- Tracking offer status and expiry

Out of scope:
- Predictive waitlist selection or advanced priority algorithms
- Manual override logic beyond standard staff actions

## Acceptance Criteria

1. Given a slot is released, then the system identifies the first eligible waitlisted patient.
2. Given the first waitlisted patient is eligible, then the system creates and sends an offer notification.
3. Given the patient accepts the offer, then the appointment is confirmed automatically.
4. Given the offer expires or is declined, then the next patient receives an offer.

## Technical Notes

- Use queue ordering and eligibility rules to select candidates.
- Offer creation should be idempotent for slot release events.
- Keep offer notifications consistent with channel preferences.

## Dependencies

- Waitlist entries from US-039
- Notification delivery system
- Appointment release and rescheduling logic

## Definition of Done

- First waitlist patient auto-offered when a slot becomes available
- Offers track expiration and response state
- Accepted offers convert to booked appointments automatically
- Acceptance criteria validated by test cases
