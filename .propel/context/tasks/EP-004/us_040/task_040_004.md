# Task 004: Automate Offer Response Outcomes (Accept, Decline, Expire)

Task ID: task_040_004  
User Story: US-040 (Auto-Offer First Waitlist Patient)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Full-Stack Developer  
Status: Not Started

## Objective

Handle offer outcomes so accepts confirm appointments and declines/expiries progress to next patient.

## Requirements

1. Accept response confirms appointment automatically.
2. Decline response triggers next eligible offer.
3. Expired offer triggers next eligible offer.
4. State transitions remain consistent across offer, waitlist, appointment.

## Implementation Steps

1. Wire accept handler to booking confirmation.
2. Wire decline and expiry paths to progression logic.
3. Add transactional updates for related entities.
4. Add integration tests for all response outcomes.

## Acceptance Criteria

- [ ] Accept confirms appointment automatically.
- [ ] Decline/expire advances to next candidate.
- [ ] State transitions are consistent and auditable.

## Dependencies

- US-039 offer response model.
- Booking confirmation services.

## Validation

- [ ] Outcome tests pass for accept, decline, and timeout scenarios.

## Definition of Done

- Offer outcome automation works end-to-end.
