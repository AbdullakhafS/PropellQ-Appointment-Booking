# Task 003: Create Idempotent Offer Generation and Notification Dispatch

Task ID: task_040_003  
User Story: US-040 (Auto-Offer First Waitlist Patient)  
Epic: EP-004  
Priority: HIGH  
Story Points: 1  
Owner Role: Backend Developer  
Status: Not Started

## Objective

Generate and send an offer to the selected patient with idempotency safeguards for repeated slot-release events.

## Requirements

1. Create offer record for selected waitlist entry.
2. Dispatch notification via configured channels.
3. Prevent duplicate offers for same slot/event window.
4. Persist offer status and expiry metadata.

## Implementation Steps

1. Add idempotency key strategy for offer creation.
2. Implement offer persistence and notification dispatch.
3. Add duplicate suppression checks.
4. Add integration tests for replay scenarios.

## Acceptance Criteria

- [ ] Offer is created and notification sent once per slot release cycle.
- [ ] Replayed events do not create duplicate offers.
- [ ] Offer metadata is persisted correctly.

## Dependencies

- Task_040_001 and task_040_002.
- Notification service.

## Validation

- [ ] Replay test confirms idempotent behavior.

## Definition of Done

- Offer creation/dispatch is idempotent and production-safe.
