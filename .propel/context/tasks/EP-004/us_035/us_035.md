# US-035: Implement Drag-Drop Queue Reordering

**Story ID:** US-035  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** MEDIUM  
**Story Points:** 3  
**Owner Role:** UX / Front-End Engineer  
**Status:** Draft

## User Story

As a **staff queue coordinator**, I want to reorder same-day queue items with drag-and-drop so that I can prioritize patients based on arrival, urgency, or provider availability.

## Business Value

- Simplifies queue optimization for front-desk staff
- Supports operational flexibility during busy periods
- Reduces manual queue management effort

## Scope

In scope:
- Drag-and-drop reordering of the staff queue
- Persisted queue order across sessions
- Conflict detection when multiple users reorder simultaneously

Out of scope:
- Automated queue ranking rules
- Staff performance dashboards

## Acceptance Criteria

1. Given the staff queue is displayed, then items can be reordered by dragging and dropping.
2. Given a queue order change is saved, then the new order persists for all staff users.
3. Given another user changes the queue concurrently, then the system resolves or warns about conflicting reorder operations.
4. Given the queue item order changes, then the updated order is reflected in downstream processing such as check-in workflows.

## Technical Notes

- Use a drag-and-drop library consistent with the UI stack.
- Persist order using a queue priority or position field.
- Handle concurrent reorder operations with optimistic updates or locks.

## Dependencies

- Queue management API
- Real-time queue updates from US-034

## Definition of Done

- Drag-and-drop queue reorder implemented and functional
- Persisted order is visible across staff sessions
- Conflicts handled gracefully
- UI/UX reviewed for usability and accessibility
