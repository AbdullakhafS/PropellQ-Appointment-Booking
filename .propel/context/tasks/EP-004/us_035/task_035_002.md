## Objective

Persist reordered queue positions so updated order is durable and visible across sessions/users.

## Requirements

1. Add queue position/priority persistence field(s) if needed.
2. Implement reorder API endpoint to submit new order.
3. Save position updates transactionally.
4. Return canonical ordered queue state after save.
5. Ensure reordered state is used by queue retrieval API.

## Implementation Steps

1. Add migration/model update for queue position.
2. Implement `PATCH/POST` reorder endpoint contract.
3. Apply position updates in single transaction.
4. Update queue query order-by logic.
5. Add integration tests for persistence.

## Acceptance Criteria

- [ ] Reorder API saves queue order successfully
- [ ] Queue order persists after reload/new session
- [ ] Queue retrieval reflects saved ordering
- [ ] Integration tests validate persistence behavior
