# Task 004: Sync Reordered Queue Across Sessions and Downstream Workflows

**Task ID:** task_035_004  
**User Story:** US-035 (Implement Drag-Drop Queue Reordering)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Full-Stack Developer  
**Status:** Not Started

## Objective

Ensure reordered queue state propagates in real-time to all staff sessions and is honored by downstream processing such as check-in workflows.

## Requirements

1. Publish reorder events on successful save.
2. Update all subscribed staff sessions without manual refresh.
3. Ensure downstream queue consumers use updated position.
4. Preserve order consistency between queue and check-in processing.

## Implementation Steps

1. Emit reorder event through real-time channel.
2. Patch queue state on client subscribers.
3. Validate downstream sort/selection uses new order.
4. Add integration tests for cross-session sync and check-in alignment.

## Acceptance Criteria

- [ ] Reordered queue appears for all active sessions
- [ ] Downstream check-in reflects updated order
- [ ] No stale ordering remains after reorder event

## Dependencies

- US-034 real-time infrastructure
- Task 035_002 reorder persistence

## Validation

- [ ] Multi-session test confirms synchronized order
- [ ] Check-in workflow test uses updated queue order

## Definition of Done

- Reorder synchronization works across sessions and queue-dependent workflows.
