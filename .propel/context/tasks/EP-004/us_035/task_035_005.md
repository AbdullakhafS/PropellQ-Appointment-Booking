## Objective

Validate drag-drop reorder behavior end-to-end, including persistence, conflicts, accessibility, and staff usability.

## Requirements

1. E2E scenarios:
   - Reorder queue item and confirm persisted order
   - Conflict scenario with concurrent staff users
   - Reconnect/session reload retains order

2. UX and accessibility:
   - Drag feedback usability review
   - Keyboard and assistive behavior checks

3. Reliability:
   - No item loss/duplication after repeated reorder actions

## Implementation Steps

1. Build test matrix from acceptance criteria.
2. Execute manual and automated tests.
3. Record defects and retest fixes.
4. Document QA sign-off.
