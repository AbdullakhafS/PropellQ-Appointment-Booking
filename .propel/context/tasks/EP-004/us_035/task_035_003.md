# Task 003: Handle Concurrent Reorder Conflicts Gracefully

**Task ID:** task_035_003  
**User Story:** US-035 (Implement Drag-Drop Queue Reordering)  
**Epic:** EP-004  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Full-Stack Developer  
**Status:** Not Started

## Objective

Detect and resolve concurrent reorder conflicts when multiple staff users modify queue order at the same time.

## Requirements

1. Add conflict detection strategy (version/etag/lock).
2. Reject stale reorder submissions with clear conflict response.
3. Provide UI warning and refresh/retry path on conflict.
4. Prevent silent overwrite of another user’s queue changes.

## Implementation Steps

1. Add queue version token to reorder payload/response.
2. Validate token server-side before applying reorder.
3. Return conflict-specific response code/message.
4. Add frontend conflict handling modal/banner.
5. Add integration tests for concurrent reorder scenario.

## Acceptance Criteria

- [ ] Concurrent reorder conflicts are detected
- [ ] Staff receives clear guidance on conflict
- [ ] Queue state can be refreshed and reorder retried
- [ ] No silent data overwrite occurs

## Dependencies

- Task 035_002 reorder API
- US-034 real-time sync layer

## Validation

- [ ] Two-session reorder collision test passes
- [ ] Conflict flow is recoverable by user

## Definition of Done

- Concurrency conflict handling for reorder is reliable and user-friendly.
