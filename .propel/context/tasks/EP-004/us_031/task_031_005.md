# Task 005: Validate Staff Workflow, Data Integrity, and Basic Performance

**Task ID:** task_031_005  
**User Story:** US-031 (Build Walk-In Booking UI - Search/Create Patient)  
**Epic:** EP-004  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** QA Engineer / Full-Stack Developer  
**Status:** Done

## Objective

Validate the complete walk-in booking workflow for front-desk users, ensuring correctness, speed, and usability across the key acceptance scenarios.

## Requirements

1. End-to-end scenarios:
   - Search existing patient -> book walk-in -> confirm summary
   - No match -> create patient -> book walk-in -> confirm summary

2. Data integrity checks:
   - Appointment is persisted with walk-in flag
   - Patient linkage is correct for selected/created records
   - Queue entry reflects walk-in context

3. Basic performance checks:
   - Search responsiveness acceptable for desk workflow
   - Booking submission/confirmation latency acceptable for staff use

4. Usability checks:
   - Front-desk flow can be completed without leaving the walk-in screen
   - Error states are recoverable and clear

## Implementation Steps

1. Create test matrix for all acceptance criteria.
2. Execute integration and manual tests for both primary paths.
3. Verify API payloads and queue visibility.
4. Log defects and retest fixes.
5. Publish QA sign-off notes.

## Acceptance Criteria

- [x] Walk-in flag and queue visibility validated
- [x] Error and retry behavior validated
- [x] No critical workflow defects remain
