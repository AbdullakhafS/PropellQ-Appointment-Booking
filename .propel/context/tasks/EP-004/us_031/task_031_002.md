# Task 002: Build Quick New Patient Registration Flow

**Task ID:** task_031_002  
**User Story:** US-031 (Build Walk-In Booking UI - Search/Create Patient)  
**Epic:** EP-004  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Frontend/Backend Developer  
**Status:** Done

## Objective

Enable front-desk staff to create a new patient record quickly from the walk-in screen when no existing patient match is found.

## Requirements

1. Quick registration form:
   - Required fields: first name, last name, DOB, phone, gender (as defined by patient service)
   - Optional fields: email, address, notes

2. Validation:
   - Required field and format validation before submit
   - Duplicate warning if similar patient exists

3. Creation flow:
   - Submit to patient create API
   - On success, auto-select newly created patient for booking
   - On failure, preserve entered data and display actionable error

4. UX constraints:
   - Keep flow lightweight for front-desk speed
   - Avoid full intake form complexity

## Implementation Steps

1. Build quick registration modal/panel UI.
2. Add form validation rules.
3. Integrate patient creation API.
4. Auto-select created patient and return to booking flow.
5. Add tests for required fields and error handling.

## Acceptance Criteria

- [x] Staff can create a new patient from walk-in screen
- [x] Required fields are validated correctly
- [x] Created patient is immediately available for booking
- [x] Duplicate or API errors are handled clearly
- [x] Registration flow does not require leaving walk-in screen

## Dependencies

- Patient create endpoint from EP-001/EP-002
- Front-desk authorization context

## Validation

- [x] Successful create path tested end-to-end
- [x] Validation errors displayed correctly
- [x] Duplicate warning flow validated

## Definition of Done
