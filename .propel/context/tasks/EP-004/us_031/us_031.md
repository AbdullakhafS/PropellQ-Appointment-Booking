# US-031: Build Walk-In Booking UI (Search/Create Patient)

**Story ID:** US-031  
**Epic:** EP-004 (Appointment Operations & Queue Management)  
**Priority:** HIGH  
**Story Points:** 3  
**Owner Role:** Product / UX Engineer  
**Status:** Draft

## User Story

As a **front-desk staff member**, I want a streamlined walk-in booking interface that lets me search for an existing patient or create a new patient record so that I can register and schedule walk-in appointments quickly.

## Business Value

- Reduces front-desk processing time for walk-in patients
- Improves patient experience with faster check-in
- Increases operational throughput for same-day visits

## Scope

In scope:
- Patient search by name, phone, email, or patient ID
- Quick new patient registration flow with required demographic fields
- Walk-in appointment booking workflow in staff portal
- Visual confirmation of walk-in appointment details

Out of scope:
- Full patient intake form
- Clinical documentation capture
- Mobile patient-facing self-service flow

## Acceptance Criteria

1. Given a staff user is on the walk-in booking screen, when they search for a patient, then matching patient records are displayed instantly.
2. Given a staff user cannot find the patient, when they create a new patient record, then the record is created and available for booking.
3. Given a patient is selected or created, when the staff user starts booking, then the booking flow proceeds with walk-in context.
4. Given the walk-in is saved, then the appointment record is flagged as a walk-in and visible in the staff queue.
5. Given the staff user submits the booking, then the system shows a confirmation summary with patient name, provider, time, and walk-in status.

## Technical Notes

- Use incremental search for patient lookup to minimize typing.
- Pre-validate required patient fields before booking.
- Persist walk-in context with an explicit flag in appointment records.

## Dependencies

- Patient record service from EP-001/EP-002
- Staff authentication and authorization from EP-005
- Appointment booking backend APIs

## Definition of Done

- Walk-in booking UI implemented with search/create flow
- New patient record creation linked to appointment booking
- Walk-in booking marked in the appointment model
- UI tested for basic staff workflows
- Acceptance criteria verified with sample data
