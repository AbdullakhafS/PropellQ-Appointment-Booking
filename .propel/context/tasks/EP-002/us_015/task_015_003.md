# Task 003: Build Insurance Pre-Check API and Verification Logging

**Task ID:** task_015_003  
**User Story:** US-015 (Implement Insurance Pre-Check Logic)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Backend Developer  
**Status:** Not Started

## Objective

Expose insurance pre-check via API and persist all verification attempts in InsuranceVerifications for auditability.

## Requirements

1. Endpoint:
   - POST /api/intake/insurance-check
2. Request payload:
   - appointment_id, insurance_name, member_id, group_number(optional), plan_name(optional)
3. Response payload:
   - matched_plan, confidence_score, verification_status, reason
4. Persist each check in InsuranceVerifications with:
   - patient_id, appointment_id, provided values, matched_plan_id, confidence_score, verification_status, checked_by=system, created_at
5. Error handling:
   - invalid input (400), unauthorized (401/403), internal errors (500)
6. Performance target:
   - median response <= 500ms

## Implementation Steps

1. Create InsuranceVerifications migration and indexes.
2. Implement API endpoint contract and validation.
3. Invoke matching/scoring engine from task_015_002.
4. Persist result row for each request.
5. Add integration tests for success/failure paths.

## Acceptance Criteria

- [ ] API endpoint returns expected verification payload
- [ ] Every check is logged in InsuranceVerifications
- [ ] Invalid payloads return clear errors
- [ ] Response time target is met
- [ ] Integration tests pass

## Dependencies

- Task 015_002 scoring engine
- Auth middleware and appointment context

## Validation

- [ ] API logs successful checks
- [ ] API logs unverified outcomes correctly
- [ ] API behavior matches acceptance criteria examples

## Definition of Done
