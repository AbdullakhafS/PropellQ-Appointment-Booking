## Objective

Implement the core matching engine that validates insurance name, member ID format, optional group number, and computes confidence/status.

## Requirements

1. Name matching:
   - Case-insensitive exact and alias matching
   - Partial match support (example: Aetna -> Aetna Health)
2. Member ID format validation using plan regex.
3. Optional group number validation using plan regex.
4. Confidence scoring rules:
   - Exact match + valid member ID: 100
   - Name match + unclear/invalid ID: 70 or 50 per rules
   - Recognized name without ID: 50
   - Unknown name: 0
5. Status assignment:
   - confidence >= 70 => verified
   - confidence < 70 => unverified
6. Provide reason codes/messages for downstream display.

## Implementation Steps

1. Implement plan lookup service (name/alias/partial).
2. Implement regex validation helpers for member ID and group number.
3. Implement confidence scoring function.
4. Add reason code map for result explanations.
5. Add unit tests for happy path and edge cases.
