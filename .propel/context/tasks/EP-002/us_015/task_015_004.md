# Task 004: Integrate Pre-Check Trigger into Intake Submission Flow

**Task ID:** task_015_004  
**User Story:** US-015 (Implement Insurance Pre-Check Logic)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Full-Stack Developer  
**Status:** Not Started

## Objective

Automatically trigger insurance pre-check during intake submission and ensure booking remains non-blocking for unverified outcomes.

## Requirements

1. Trigger pre-check after insurance data is collected from chatbot/form.
2. Do not block appointment flow on unverified result.
3. Store verification status with intake/appointment context.
4. If pre-check fails technically, allow booking and mark status for manual follow-up.
5. Ensure US-016 dashboard can consume resulting unverified records.

## Implementation Steps

1. Add pre-check call into intake submit workflow.
2. Map intake insurance fields to API request.
3. Persist returned verification status and confidence.
4. Add fallback handling for API failures/timeouts.
5. Add integration tests for full submit + pre-check path.

## Acceptance Criteria

- [ ] Intake submission invokes pre-check automatically
- [ ] Unverified result does not block booking
- [ ] Verification status persists with intake context
- [ ] Failure fallback preserves booking flow
- [ ] Integration tests confirm end-to-end behavior

## Dependencies

- Task 015_003 pre-check API
- Intake submit flow from US-011/US-013

## Validation

- [ ] Valid insurance path saves verified status
- [ ] Invalid insurance path saves unverified status
- [ ] Submission still succeeds in both cases

## Definition of Done

- Insurance pre-check is integrated into submission flow as a soft validation step.
