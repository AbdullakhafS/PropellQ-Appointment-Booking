# Task 001: Create Insurance Plans Dataset and Database Schema

**Task ID:** task_015_001  
**User Story:** US-015 (Implement Insurance Pre-Check Logic)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Backend Developer / DBA  
**Status:** Done

## Objective

Create the predefined insurance plans store and base schema needed for soft pre-check validation.

## Requirements

1. Create InsurancePlans table with fields:
   - id, name, aliases, member_id_format, group_number_format, created_at
2. Seed ~50 common plans with aliases and regex formats.
3. Include major plans: Aetna, BCBS, UnitedHealth, Humana, Cigna.
4. Add indexes for fast lookup by name/alias.
5. Add script/migration for deterministic seed loading.
6. Validate all regex patterns compile before seed insert.

## Implementation Steps

1. Add migration for InsurancePlans table.
2. Prepare plan seed data in SQL/JSON format.
3. Build seed loader with idempotent behavior.
4. Add regex validation test utility.
5. Document plan maintenance/update process.

## Acceptance Criteria

- [x] InsurancePlans table created with expected columns
- [x] 50+ plans seeded successfully
- [x] Name and alias matching fields populated
- [x] Regex patterns compile and pass sample checks
- [x] Indexes in place for query performance

## Dependencies

- Database migration framework
- Initial insurance list and sample formats

## Validation

- [x] Query by plan name returns correct row
- [x] Query by alias returns matching base plan
- [x] Regex samples pass expected valid/invalid cases

## Definition of Done

- Insurance plans schema and seed data are production-ready for pre-check logic.
