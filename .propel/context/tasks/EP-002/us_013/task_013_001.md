# Task 001: Build Auto-Population API for Previous Intake Data

**Task ID:** task_013_001  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 3  
**Owner Role:** Backend Developer  
**Status:** Not Started

## Objective

Create the backend API endpoint that retrieves the last intake record for a returning patient, enabling the manual form to auto-populate with previous visit data.

## Requirements

1. Implement endpoint:
   - `GET /api/patients/{patientId}/intake/last`
   - Returns the most recent completed intake data for the patient
   - Includes chief complaint, medical history, medications, allergies, insurance, and last_updated_at

2. Query logic:
   - Search `IntakeResponses` or equivalent intake store for the latest completed record
   - Filter by patient_id and completed_at not null
   - Handle no prior intake gracefully with 204 No Content or empty response

3. Data normalization:
   - Ensure returned JSON matches frontend form schema
   - Convert database structures to form-friendly shapes for repeating fields
   - Preserve typed fields for medical_history and allergies

4. Performance:
   - Response time <500ms for a patient with multiple historical records
   - Use indexes on `patient_id` and `completed_at`

5. Security:
   - Verify authenticated user is authorized for the patient record
   - Do not expose another patient’s data
   - Use current session/patient context for authorization

6. Error handling:
   - 401 Unauthorized if the request is unauthenticated
   - 403 Forbidden if the user is not authorized to view patient data
   - 404 Not Found if patient does not exist
   - 500 Internal Server Error for database failures

## Implementation Steps

1. Define response contract and form schema mapping.
2. Create or extend API controller for intake retrieval.
3. Implement database query and mapping logic.
4. Add authorization checks using current user context.
5. Test no-data and prior-data cases.
6. Add unit/integration tests for endpoint behavior.
7. Document endpoint in API spec.
