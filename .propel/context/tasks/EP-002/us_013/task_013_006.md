# Task 006: Test Mobile Responsiveness & Accessibility

**Task ID:** task_013_006  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** QA / Frontend Developer  
**Status:** Not Started

## Objective

Validate the manual intake form for mobile responsiveness and WCAG 2.1 AA accessibility, ensuring the form works across device sizes and assistive technologies.

## Requirements

1. Mobile layouts:
   - Single-column layout on 375px width
   - Two-column layout on 768px+ when space permits
   - Sticky submit button on mobile
   - Touch-friendly controls with 44px minimum tap targets

2. Accessibility:
   - All inputs have visible labels and ARIA labels where needed
   - Keyboard navigation supports section navigation and input focus
   - Error messages announced for assistive technology
   - Contrast ratios meet WCAG 2.1 AA

3. Performance:
   - Form loads within 1 second on mobile networks
   - Repeating field actions are responsive and do not lag

4. Testing:
   - Use browser dev tools or device lab for iPhone and Android viewport testing
   - Test with a screen reader or automated accessibility tool
   - Validate metrics from previous-data load and form submission flows

## Implementation Steps

1. Test the form on mobile breakpoints and adjust styles.
2. Verify sticky submit button behavior.
3. Run accessibility audits and fix issues.
4. Validate keyboard navigation and ARIA roles.
5. Document any remaining accessibility issues.

## Acceptance Criteria

- [ ] Form layout works on 375px and 768px breakpoints
- [ ] Sticky submit button behaves correctly on mobile
- [ ] Touch targets are large enough
- [ ] Accessibility audit passes WCAG 2.1 AA checks
- [ ] Keyboard navigation is complete
- [ ] Any issues are documented and resolved

## Dependencies

- Form implementation complete
- Auto-population and consent logic integrated

## Validation

- [ ] Mobile device tests passed
- [ ] Accessibility audit results recorded
- [ ] No critical accessibility violations remain

## Definition of Done

- Manual intake form is responsive and accessible.
