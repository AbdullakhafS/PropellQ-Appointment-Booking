# Task 005: Implement Auto-Population Consent & Fresh Start Flow

**Task ID:** task_013_005  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Implement the auto-population consent prompt and the ability for patients to choose between using previous intake data or starting fresh.

## Requirements

1. Consent prompt on form load:
   - Display message: "We have your previous information. Use it to save time?"
   - Buttons: [Use Previous] and [Start Fresh]

2. Data behavior:
   - If [Use Previous] selected, load and pre-fill the form with previous intake data
   - If [Start Fresh] selected, show empty form fields
   - Allow users to switch from previous data to fresh form before final submission

3. UI state:
   - Show loading indicator while retrieving previous data
   - Show last_updated_at timestamp if previous data exists
   - If no previous data exists, skip consent and load empty form directly

4. Accessibility:
   - Buttons and prompt are keyboard accessible and screen reader friendly
   - Provide descriptive labels for the consent action

5. Analytics:
   - Track which option patients choose
   - Track if they later clear or modify pre-populated data

## Implementation Steps

1. Create consent dialog or banner component.
2. Wire consent behavior to the auto-population API.
3. Implement fresh-start flow.
4. Display timestamp information when data is loaded.
5. Add analytics event tracking.
6. Test behavior with and without previous data.

## Acceptance Criteria

- [ ] Consent prompt appears on form load when prior intake exists
- [ ] [Use Previous] pre-populates the form
- [ ] [Start Fresh] loads an empty form
- [ ] Previous data can be cleared or edited after load
- [ ] No prior data flows directly to empty form
- [ ] Accessible and responsive UI
- [ ] User choice tracked in analytics

## Dependencies

- Auto-population API endpoint
- Manual intake form structure

## Validation

- [ ] Previous data loads correctly after consent
- [ ] Fresh-start option works and does not load old data
- [ ] Last updated timestamp displays when data exists
- [ ] Behavior works in mobile and desktop

## Definition of Done

- Consent flow and fresh-start behavior implemented.
