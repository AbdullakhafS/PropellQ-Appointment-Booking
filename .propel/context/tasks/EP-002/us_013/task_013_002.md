# Task 002: Build Manual Intake Form Structure & Sections

**Task ID:** task_013_002  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Build the manual intake form structure with clearly separated sections for chief complaint, medical history, medications, allergies, and insurance.

## Requirements

1. Form layout:
   - Sections: Chief Complaint, Medical History, Current Medications, Allergies, Insurance
   - Tabs or accordions for section navigation
   - Clear labels and section headers

2. Form fields:
   - Chief complaint: multiline text area
   - Medical history: checkboxes for common conditions + "Other" text field
   - Medications: repeating row fields for medication name, dosage, frequency, delete action
   - Allergies: radio/checkbox option for drug allergies and side effects, plus details input
   - Insurance: insurance name, member ID, group number, plan name

3. Form state:
   - Use React Hook Form or equivalent for state management
   - Initial state should handle empty and auto-populated payloads
   - Use controlled components for inputs

4. UX behavior:
   - Sections expand/collapse cleanly
   - Validation messages show only on submit
   - Buttons and field controls are responsive and touch-friendly

## Implementation Steps

1. Create form UI components for each section.
2. Build form wrapper with navigation controls.
3. Define form schema and initial values.
4. Wire fields to React Hook Form state.
5. Add submit button and section headers.
6. Test layout on desktop and mobile.

## Acceptance Criteria

- [ ] Manual intake form renders with all five sections
- [ ] Fields match the US-013 data model
- [ ] Navigation between sections works
- [ ] Form state is managed cleanly
- [ ] Submit button is present and sticky on mobile
- [ ] Layout is responsive and accessible

## Dependencies

- Form schema definitions from backend contract
- US-012 conversation structure for section alignment

## Validation

- [ ] User can navigate and enter data in every section
- [ ] Form fields behave correctly on mobile and desktop
- [ ] Initial skeleton loads without errors

## Definition of Done
