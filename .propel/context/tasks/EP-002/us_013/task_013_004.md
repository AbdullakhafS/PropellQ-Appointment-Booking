# Task 004: Implement Submit-Time Validation & Error Handling

**Task ID:** task_013_004  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Implement form submission validation rules and error handling so the manual intake form is forgiving during entry and only validates at submit time.

## Requirements

1. Validation rules:
   - Chief complaint required
   - Medical history may be empty if patient explicitly says none
   - Medications optional
   - Allergies optional
   - Insurance fields optional but if any insurance field is entered, require insurance_name and member_id

2. Error presentation:
   - Only display errors on submit
   - Highlight invalid fields with clear messages
   - Avoid inline validation while typing
   - Provide summary of validation issues at top of the form

3. Submission flow:
   - On valid submit, call `POST /api/intake/submit`
   - On invalid submit, prevent API call and show validation errors
   - Persist last_updated_at metadata if successful

4. Edge cases:
   - Handle empty arrays for medications and allergies gracefully
   - Handle invalid input lengths or malformed values
   - Allow user to clear pre-populated fields and submit empty optional data

## Implementation Steps

1. Define validation schema in React Hook Form.
2. Implement submit handler and field-level message display.
3. Add form error summary and field highlighting.
4. Wire submit action to backend API.
5. Add tests for valid and invalid submission cases.
6. Validate behavior across sections.

## Acceptance Criteria

- [ ] Form validates only on submit
- [ ] Required chief complaint is enforced
- [ ] Insurance partial entry validation works
- [ ] Error summary and field messages display clearly
- [ ] API is called only when validation passes
- [ ] Optional data does not block submission
- [ ] Submission tests pass

## Dependencies

- Form structure and field definitions
- API submit endpoint contract

## Validation

- [ ] Submit with missing chief complaint shows errors
- [ ] Submit with partial insurance fields shows errors
- [ ] Submit with optional fields empty succeeds
- [ ] Backend call occurs only after validation passes

## Definition of Done

- Submit-time validation and error handling implemented and tested.
