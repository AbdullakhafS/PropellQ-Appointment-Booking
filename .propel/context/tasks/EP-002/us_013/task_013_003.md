# Task 003: Implement Repeating Fields for Medications & Allergies

**Task ID:** task_013_003  
**User Story:** US-013 (Build Manual Intake Form with Auto-Population)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Done

## Objective

Implement add/remove repeating field behavior for medications and allergy entries in the manual intake form.

## Requirements

1. Medications:
   - Add medication rows with fields for name, dosage, frequency
   - Delete medication rows
   - Pre-populate rows with previous data when available
   - Support at least 10 medication rows without UI breakage

2. Allergies:
   - Add allergy entries with type classification (drug allergy vs. side effect)
   - Capture allergy details and reaction notes
   - Delete allergy entries
   - Pre-populate with previous allergy data

3. Field management:
   - Use React Hook Form `useFieldArray` or equivalent
   - Maintain stable keys for list items
   - Preserve unsaved entries when switching sections

4. UX:
   - Add buttons are visible and touch-friendly (44px+)
   - Deletions confirm on each row if needed to prevent accidental loss
   - Add/remove interaction is fluid and accessible

## Implementation Steps

1. Implement medication field array component.
2. Implement allergy field array component.
3. Wire add/delete logic into form state.
4. Add initial row defaults for empty state.
5. Test array behavior with populated and empty data.
6. Validate keyboard navigation and accessibility.

## Acceptance Criteria

- [x] Medications add/delete functionality works correctly
- [x] Allergy entries add/delete functionality works correctly
- [x] Pre-populated rows render and remain editable
- [x] No lost state when interacting across sections
- [x] UI works on mobile and desktop
- [x] Component tests cover add/remove behavior
