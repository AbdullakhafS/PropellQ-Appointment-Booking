# Task 001: Design AI Chatbot Architecture & Prompt Strategy

**Task ID:** task_011_001  
**User Story:** US-011 (Implement AI Chatbot Engine)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Solution Architect / Backend Developer  
**Status:** Not Started

## Objective

Define the architecture and prompt engineering strategy for the AI-powered medical intake chatbot, ensuring the solution supports conversational intake, context tracking, structured extraction, and HIPAA-safe logging.

## Requirements

1. **Architecture**
   - Define end-to-end flow for chatbot messages: frontend → backend → LLM service → response processing → persistence.
   - Choose integration pattern: synchronous API proxy vs. async inference queue.
   - Specify NLP engine options: OpenAI GPT-3.5/4, Azure OpenAI, or LLaMA inference service.
   - Ensure HIPAA-safe handling of PII in transit and storage.

2. **Prompt Strategy**
   - Create a system prompt that instructs the model to act as a medical intake assistant.
   - Define prompt templates for each intake domain: chief complaint, medical history, medications, allergies, insurance.
   - Include instructions for empathetic, conversational tone and follow-up question behavior.
   - Define extraction guidance for structured data in the background.

3. **Conversation Context**
   - Design how conversation history is passed to the model with each turn.
   - Define maximum context window and truncation strategy.
   - Ensure the model can recall prior answers (e.g., allergies previously stated) when asking follow-ups.

4. **Extraction Data Model**
   - Specify extracted JSON fields: chief_complaint, medical_history, medications, allergies, insurance_info.
   - Define confidence scores for each section.
   - Define when extraction updates should be persisted.

5. **Security & Compliance**
   - Document PII handling safeguards, including redaction rules and secure storage.
   - Specify logging policy: transcript logs must be encrypted and audit-ready.
   - Confirm no medical advice or diagnosis is produced.

## Implementation Steps

1. Review existing EP-002 requirements and current intake data model.
2. Evaluate LLM provider options and select the primary integration target.
3. Draft system prompt and domain-specific prompt templates.
4. Define API contract for chatbot backend endpoints.
5. Specify context payload shape and truncation logic.
6. Document structured extraction JSON schema.
7. Align on HIPAA-safe storage and logging policies.
8. Review architecture and prompt design with the team.
