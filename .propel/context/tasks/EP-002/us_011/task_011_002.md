## Objective

Build the backend service that proxies chatbot messages to the selected LLM provider, manages conversation context, extracts structured intake data, and returns assistant responses to the frontend.

## Requirements

1. **API Endpoints**
   - `POST /api/intake/chat/start` → starts a new chatbot session and returns conversation_id.
   - `POST /api/intake/chat/message` → accepts user message, appointment_id, conversation_id, and returns assistant message + extracted data.
   - `GET /api/intake/chat/{conversationId}` → retrieves conversation history.

2. **LLM Provider Integration**
   - Implement request construction for the chosen provider.
   - Include system prompt, conversation history, and new user message.
   - Implement retry/backoff logic for transient API failures.
   - Enforce a timeout to meet 3-second per-turn latency.

3. **Structured Extraction**
   - Parse model output for structured fields: chief_complaint, medical_history, medications, allergies, insurance_info.
   - Store confidence scores for each section.
   - If model response cannot be parsed, flag extraction fallback and continue conversation.

4. **Context Management**
   - Retrieve and append prior messages from `IntakeConversations`.
   - Truncate oldest history when payload size approaches provider limits.
   - Persist each turn after successful response.

5. **Error Handling**
   - Return user-friendly messages on provider failure.
   - Log errors with appointment_id and conversation_id.
   - If the provider is unavailable, optionally fall back to a default prompt or offer switch to manual intake.

6. **Security**
   - Sanitize incoming user messages before forwarding to the provider.
   - Do not store raw PII beyond encrypted transcript logs.
   - Ensure provider API keys are stored in secure configuration.

## Implementation Steps

1. Implement chatbot backend endpoints.
2. Wire up chosen LLM provider client.
3. Build request payload generator including system prompt and history.
4. Implement structured extraction parser.
5. Persist each message turn and extracted data.
6. Add timeout, retry, and fallback handling.
7. Write unit tests for payload construction, parsing, and error handling.
8. Review integration with security/compliance team.
