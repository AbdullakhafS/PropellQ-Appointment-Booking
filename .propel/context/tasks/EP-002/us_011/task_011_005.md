# Task 005: Implement Clarification Handling and Manual Fallback

**Task ID:** task_011_005  
**User Story:** US-011 (Implement AI Chatbot Engine)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Backend / Frontend Developer  
**Status:** Not Started

## Objective

Implement robust misunderstanding detection, clarification prompts, and a fallback path to manual intake when the chatbot cannot parse or confirm patient input.

## Requirements

1. **Misunderstanding Detection**
   - Detect unclear or off-topic responses using NLP confidence, response content, or explicit extraction failure.
   - Support up to 2 clarification attempts.
   - If still unclear after 2 attempts, offer the patient the option to switch to manual intake.

2. **Clarification Prompting**
   - Respond with friendly clarification messages such as: "I’m not sure I understood. Could you say that again in a different way?"
   - Keep clarification flow natural and non-frustrating.
   - Ensure the chatbot tracks re-attempt count per question.

3. **Manual Fallback**
   - Provide a visible button or link: "Switch to manual form".
   - If patient chooses fallback, persist `switched_to_manual_at` and transition to the manual intake form.
   - Preserve any collected data during the AI conversation.

4. **Transcript Logging**
   - Log clarification turns and fallback triggers.
   - Mark conversation metadata when clarification was required.

5. **UI Messaging**
   - Present clarification requests in a calm, empathetic tone.
   - Inform patients: "It looks like I didn't understand that. You can rephrase, or switch to the form if you prefer."
   - If fallback is used, show a summary of data captured so far.

6. **Testing**
   - Test scenarios where the chatbot fails to parse input.
   - Test re-attempt count and fallback transition.
   - Test data preservation after switching to manual intake.

## Implementation Steps

1. Implement misunderstanding detection in backend response processing.
2. Add clarification turn counter to session metadata.
3. Implement frontend clarification messaging and fallback control.
4. Preserve captured data when switching to manual mode.
5. Persist `switched_to_manual_at` and audit logs.
6. Create tests for clarification and fallback flows.
7. Conduct usability review on fallback messaging.

## Acceptance Criteria

- [ ] Clarification attempts are triggered on unclear responses.
- [ ] Patient can retry up to 2 times.
- [ ] Manual fallback is visible and functional.
- [ ] Data collected before fallback is preserved.
- [ ] Clarifications and fallback events are logged.
- [ ] UI messaging is user-friendly.
- [ ] Tests for fallback and clarification pass.

## Dependencies

- Chatbot backend and UI implemented
- Conversation persistence available
- Manual intake form implementation available

## Validation

- [ ] Simulate unclear patient input and verify clarification flow.
- [ ] Verify fallback works and preserves data.
- [ ] Confirm fallback metadata is stored correctly.
- [ ] Confirm patients can continue intake after fallback.

## Definition of Done

- Clarification and fallback paths are implemented end-to-end.
- Users can gracefully transition from AI to manual intake.
