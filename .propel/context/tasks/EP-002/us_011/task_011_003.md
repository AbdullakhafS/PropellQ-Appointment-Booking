# Task 003: Build React Chatbot UI & Conversation Experience

**Task ID:** task_011_003  
**User Story:** US-011 (Implement AI Chatbot Engine)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Create the patient-facing chatbot interface in React, enabling conversational intake with a message history view, input area, summary screen, and seamless transition from intake start to completion.

## Requirements

1. **Chatbot Interface**
   - Conversation panel showing user messages and assistant replies.
   - Input field with submit button and enter-to-send support.
   - Loading indicator while waiting for model response.
   - Responsive layout for desktop and mobile.

2. **Session Management**
   - Start chatbot session when user clicks "Start AI Intake".
   - Persist conversation_id in browser state.
   - Allow refresh/resume of the current conversation.

3. **Summary Screen**
   - After completion, show captured summary: chief complaint, medical history, medications, allergies.
   - Provide edit/confirm actions.
   - Allow patient to add missing info or correct data before final submission.

4. **UX Behavior**
   - Display initial greeting and first question from backend.
   - Support user editing of in-progress answers by repeating question flow.
   - Show contextual hint if chatbot asks for clarification.
   - Provide a visible switch to manual form (for fallback or preference).

5. **Accessibility**
   - Use semantic HTML for messages and inputs.
   - Ensure proper keyboard navigation and labels.
   - Maintain a11y contrast and readable font sizes.

6. **Analytics & Tracking**
   - Track session start, turn completion, fallback trigger, and summary confirmation.
   - Capture performance metrics for response wait time.

## Implementation Steps

1. Build React components for chatbot screen, message list, input box, and summary.
2. Connect UI to backend chat endpoints.
3. Implement loading state, error state, and retry behavior.
4. Add manual intake switch control.
5. Implement summary review after completion.
6. Add accessibility enhancements.
7. Write component tests for chat flow and summary display.
8. Perform usability review.
