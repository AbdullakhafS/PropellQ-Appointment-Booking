# Task 004: Implement Conversation Storage & Structured Data Extraction

**Task ID:** task_011_004  
**User Story:** US-011 (Implement AI Chatbot Engine)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Backend Developer / Data Engineer  
**Status:** Not Started

## Objective

Create the database schema and persistence layer for storing chatbot transcripts, extracted intake data, confidence scores, and session metadata.

## Requirements

1. **Database Tables**
   - `IntakeConversations`:
     - id, appointment_id, patient_id, mode ('ai'|'manual'), conversation_transcript JSON, extracted_data JSON, confidence_scores JSON, completed_at, switched_to_manual_at, created_at.
   - `ChatbotPrompts`:
     - id, prompt_version, prompt_text, effective_date, deprecated_at.
   - Indexes on appointment_id, patient_id, and mode.

2. **Transcript Storage**
   - Store each user/assistant turn in `conversation_transcript` as an ordered array.
   - Ensure transcript JSON is encrypted at rest per HIPAA requirements.
   - Support retrieval of full conversation history for UI display and audit.

3. **Extracted Data Storage**
   - Persist `extracted_data` with keys: chief_complaint, medical_history, medications, allergies, insurance_info.
   - Persist `confidence_scores` for each extraction category.
   - Update extracted data on each turn as new information is captured.

4. **Session Metadata**
   - Store `completed_at` when intake summary is confirmed.
   - Store `switched_to_manual_at` if patient switches out of AI mode.
   - Persist `conversation_id` mapping to frontend session.

5. **Migration / Seed**
   - Create database migrations for schema changes.
   - Seed initial system prompt version into `ChatbotPrompts`.

6. **Performance & Querying**
   - Ensure retrieval of transcript and extracted data is efficient.
   - Support queries for recent AI conversations and status.
   - Design storage size limits to prevent unbounded transcript growth.

## Implementation Steps

1. Implement database migrations.
2. Create persistence models/entities.
3. Build repository/service methods for saving conversation turns and extraction data.
4. Encrypt transcript JSON and manage decryption for authorized access.
5. Seed initial prompt template into `ChatbotPrompts`.
6. Add unit tests for persistence logic.
7. Validate retrieval performance.
