# Task 003: Implement Edit Availability Rules and Intake Edit Mode

**Task ID:** task_018_003  
**User Story:** US-018 (Display Intake Responses in Patient Profile)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Enable conditional intake editing from profile only when appointment has not started, reusing manual intake form behavior.

## Requirements

1. Edit eligibility rules:
   - Show Edit Intake button only if appointment status != checked_in and appointment time > now
2. Edit mode behavior:
   - Reuse IntakeForm component from US-013
   - Pre-fill with current intake values
   - Provide Save Changes and Cancel actions
3. Metadata updates:
   - Update last_updated_at after successful save
   - Display "Updated on" timestamp
4. UX safeguards:
   - Disable edit controls when user lacks permission
   - Preserve unsaved changes warning on navigation away

## Implementation Steps

1. Add canEdit evaluation from appointment metadata.
2. Wire profile display to IntakeForm edit mode.
3. Implement save/cancel flow state transitions.
4. Update timestamp display post-save.
5. Add tests for edit visibility and mode transitions.

## Acceptance Criteria

- [ ] Edit button appears only when appointment is editable
- [ ] Edit mode opens with pre-filled intake data
- [ ] Save and cancel paths work correctly
- [ ] Updated timestamp refreshes after save
- [ ] Permission checks enforced in UI

## Dependencies

- Task 018_002 intake tab display
- US-013 form component
- Appointment status/time data availability

## Validation

- [ ] Non-editable appointments hide edit controls
- [ ] Editable appointments allow full update flow
- [ ] Unsaved changes warning works as expected

## Definition of Done

- Intake edit mode is conditionally available and functions correctly.
