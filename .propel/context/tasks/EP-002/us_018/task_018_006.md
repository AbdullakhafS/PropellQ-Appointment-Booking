# Task 006: Validate Accessibility, Mobile Responsiveness, and End-to-End Workflow

**Task ID:** task_018_006  
**User Story:** US-018 (Display Intake Responses in Patient Profile)  
**Epic:** EP-002  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** QA Engineer / Frontend Developer  
**Status:** Done

## Objective

Validate US-018 end-to-end behavior including profile loading, conditional editing, persistence, accessibility, and mobile responsiveness.

## Requirements

1. E2E workflow tests:
   - Open profile -> intake tab -> view latest intake
   - Edit when appointment is eligible -> save -> verify updated values
   - Confirm edit denied when appointment is checked_in/started
2. Accessibility checks:
   - Section headings semantic (<h3>)
   - Table header semantics (<th scope>)
   - Keyboard navigation through view and edit controls
   - Screen reader labels and focus behavior
3. Mobile responsiveness:
   - 375px stacked layout
   - Scrollable tables and accessible controls
   - Edit and export actions available on mobile
4. Performance:
   - Intake tab render and API load acceptable for profile usage
5. Regression:
   - Ensure US-017 data mapping remains correct in display/edit flows

## Implementation Steps

1. Author integration/E2E test scenarios.
2. Run accessibility audit and remediate defects.
3. Execute mobile viewport/device checks.
4. Validate conditional edit rules and update timestamps.
5. Record findings and confirm release readiness.

## Acceptance Criteria

- [x] Full intake view/edit workflow passes
- [x] Conditional edit logic enforced in all test cases
- [x] Accessibility checks meet WCAG 2.1 AA baseline
- [x] Mobile behavior verified across breakpoints
- [x] No critical defects remain
