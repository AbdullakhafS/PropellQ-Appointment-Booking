# US-018: Display Intake Responses in Patient Profile

**Epic:** EP-002 (Patient Intake & Insurance Verification)  
**Story Points:** 5  
**Priority:** HIGH  
**Status:** Not Started  
**Created:** 2026-06-17  

---

## User Story Statement

As a **patient or clinical staff**, I want to **view my completed intake responses in an organized, easy-to-read format within my patient profile**, so that **I can verify my information is correct, edit if needed, and clinical staff can reference my intake without returning to a separate intake form**.

---

## Business Value

- **Outcome 1:** Single source of truth for intake data; no need to hunt for form responses
- **Outcome 2:** Patients can verify accuracy; catch and correct errors early
- **Outcome 3:** Clinical staff can review intake during appointment without delay

---

## Scope

**In Scope:**
- Dedicated "Intake" tab or section in patient profile (alongside Demographics, Medications, Allergies)
- Display intake from most recent appointment/visit
- Show all four intake sections: chief complaint, medical history, medications, allergies, insurance
- Edit capability: Patient can edit intake responses if appointment hasn't started
- Timestamp: Display "Last updated: [date]" and "Completed via: [AI/Manual]"
- Print/export: Patient can print or export intake as PDF
- Accessibility: Fully keyboard navigable, screen reader compatible

**Out of Scope:**
- Ability to create new intake from profile (intakes are triggered by appointment only)
- Intake comparison across multiple visits (Phase 2 analytics)
- Real-time sync with external forms (Phase 2)

---

## Acceptance Criteria

1. **Intake tab in profile:** Patient profile includes "Intake" tab alongside "Demographics," "Medications," "Allergies"
2. **Most recent intake display:** Clicking "Intake" tab shows most recent completed intake (filtered by appointment_id or created_at DESC)
3. **Chief complaint section:** Displays chief complaint text; read-only display unless patient is editing
4. **Medical history section:** Displays list of conditions: [Condition Name] [Diagnosed Date] [Status]; each row is collapsible for notes
5. **Medications section:** Displays table: [Medication] [Dosage] [Frequency] [Route]; same format as form for consistency
6. **Allergies section:** Displays list of allergies: [Allergen] [Reaction Type] [Severity] [Description]
7. **Insurance section:** Displays: [Insurance Name] [Member ID] [Group Number] [Verification Status] (color-coded: green=verified, amber=unverified)
8. **Metadata:** Show "Completed: [date/time]", "Via: AI Chatbot / Manual Form", "Version: 1" (if multiple versions exist)
9. **Edit button:** If appointment hasn't started (status != 'checked_in' and appointment_time > now), display "Edit Intake" button
10. **Edit mode:** Clicking "Edit" switches to editable form; same as intake form (US-013) but pre-filled with current data; "Save Changes" button
11. **Edit timestamp:** When edited, update last_updated_at; display "Updated on [date]" in parentheses
12. **Print/Export:** "Print as PDF" button generates printable PDF of intake; "Download as PDF" button for export
13. **Mobile responsiveness:** Sections stack on 375px; tables are scrollable; all functionality available on mobile
14. **Accessibility:** Section headers are <h3> tags; table headers have <th> scope; form labels linked; keyboard Tab navigation through all editable fields

**Intake Display Component (React):**
```typescript
function IntakeTab({ patientId }) {
  const [intake, setIntake] = useState(null);
  const [editMode, setEditMode] = useState(false);
  const [canEdit, setCanEdit] = useState(false);

  useEffect(() => {
    loadLatestIntake(patientId);
  }, [patientId]);

  const loadLatestIntake = async (pId) => {
    const data = await fetch(`/api/patients/${pId}/intake/latest`).then(r => r.json());
    setIntake(data);
    // Check if appointment hasn't started
    setCanEdit(data.appointment?.status !== 'checked_in' && 
               new Date(data.appointment?.time) > new Date());
  };

  const handleSaveChanges = async (updatedData) => {
    await updateIntake(intake.id, updatedData);
    setEditMode(false);
    loadLatestIntake(patientId);
  };

  if (!intake) return <div>No intake found</div>;

  return (
    <div className="intake-tab">
      <div className="metadata">
        <p>Completed: {formatDate(intake.completed_at)}</p>
        <p>Method: {intake.mode === 'ai' ? 'AI Chatbot' : 'Manual Form'}</p>
      </div>

      {editMode ? (
        <IntakeForm 
          initialData={intake} 
          onSave={handleSaveChanges}
          onCancel={() => setEditMode(false)}
        />
      ) : (
        <>
          <IntakeDisplay intake={intake} />
          {canEdit && <button onClick={() => setEditMode(true)}>Edit Intake</button>}
          <button onClick={() => exportPDF(intake)}>Download as PDF</button>
        </>
      )}
    </div>
  );
}

function IntakeDisplay({ intake }) {
  return (
    <div className="intake-display">
      <section>
        <h3>Chief Complaint</h3>
        <p>{intake.chief_complaint}</p>
      </section>

      <section>
        <h3>Medical History</h3>
        <ul>
          {intake.medical_history.map(item => (
            <li key={item.id}>
              {item.condition_name} (since {item.diagnosed_date}) - {item.status}
            </li>
          ))}
        </ul>
      </section>

      <section>
        <h3>Medications</h3>
        <table>
          <thead>
            <tr><th>Medication</th><th>Dosage</th><th>Frequency</th></tr>
          </thead>
          <tbody>
            {intake.medications.map(med => (
              <tr key={med.id}>
                <td>{med.medication_name}</td>
                <td>{med.dosage}</td>
                <td>{med.frequency}</td>
              </tr>
            ))}
          </tbody>
        </table>
      </section>

      <section>
        <h3>Allergies</h3>
        <ul>
          {intake.allergies.map(allergy => (
            <li key={allergy.id}>
              {allergy.allergen_name} - {allergy.reaction_type} ({allergy.severity})
            </li>
          ))}
        </ul>
      </section>

      <section>
        <h3>Insurance</h3>
        <p><strong>{intake.insurance.insurance_name}</strong></p>
        <p>Member ID: {intake.insurance.member_id}</p>
        <p>Verification: <span className={intake.insurance.verification_status}>
          {intake.insurance.verification_status.toUpperCase()}
        </span></p>
      </section>
    </div>
  );
}
```

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Patient edits intake after appointment started → creates confusion | Low | Check appointment status before allowing edit; disable edit button if appointment started |
| PDF export fails → patient can't download intake | Low | Test PDF export with different browsers; provide fallback: print-friendly HTML page |
| Mobile display is cramped → hard to read on small screen | Medium | Test on real mobile devices; use responsive tables (horizontal scroll on mobile); ensure font sizes ≥14px |
| Edit permissions are too loose → patient can edit others' intakes | Medium | Verify user_id matches patient_id before allowing edit; check authorization on API endpoint |
