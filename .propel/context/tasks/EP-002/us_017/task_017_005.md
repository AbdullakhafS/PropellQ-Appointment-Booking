# Task 005: Optimize Query Performance and Common Retrieval Patterns

**Task ID:** task_017_005  
**User Story:** US-017 (Store Intake Data in Structured Format)  
**Epic:** EP-002  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** Backend Developer / DBA

## Objective

Optimize intake storage retrieval paths for common query scenarios used by profile rendering, clinical logic, and reporting.

## Requirements

1. Validate and tune indexes on:
   - IntakeResponses(patient_id)
   - IntakeResponses(appointment_id)
   - IntakeResponses(created_at)
   - Detail tables by intake_id

2. Common query support:
   - Retrieve latest intake by patient
   - Retrieve all medications/allergies for patient
   - Retrieve intake by appointment_id

3. Performance targets:
   - Common read queries complete in <300ms under normal load
   - Bulk insert/read tests remain within acceptable limits

4. Query safety:
   - Avoid N+1 patterns in ORM usage
   - Ensure proper includes/joins for detail records

## Implementation Steps

1. Profile key queries with realistic dataset.
2. Tune indexes and query plans.
3. Refactor repository methods for efficient joins.
4. Add benchmark tests for key read paths.

## Acceptance Criteria

- [ ] Key retrieval queries meet performance target
- [ ] No N+1 behavior in primary data fetch paths
- [ ] Index strategy documented and applied
- [ ] Benchmark tests added
