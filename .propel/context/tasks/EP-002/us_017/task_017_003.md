# Task 003: Build Intake Submission API with Validation

**Task ID:** task_017_003  
**User Story:** US-017 (Store Intake Data in Structured Format)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Backend Developer  
**Status:** Done

## Objective

Implement POST /api/intake/submit with strict request validation and integration to the intake persistence service.

## Requirements

1. Endpoint:
   - POST /api/intake/submit
   - Accepts normalized payload sections for chief complaint, history, medications, allergies, insurance

2. Validation rules:
   - Required appointment_id and mode
   - Data type checks for all nested objects and arrays
   - Referential integrity checks (appointment and patient existence)
   - Reject malformed or unsupported fields

3. Response:
   - success flag
   - intake_id
   - appointment_id
   - completed_at timestamp

4. Error handling:
   - 400 for invalid payload
   - 404 for missing related entities
   - 500 for server/database failures

## Implementation Steps

1. Define request/response DTOs.
2. Implement payload validators.
3. Wire API to persistence service.
4. Return canonical response payload.
5. Add API tests for valid and invalid input cases.

## Acceptance Criteria

- [x] Referential checks enforced
