# Task 001: Define Chatbot Conversation Stages & Flow

**Task ID:** task_012_001  
**User Story:** US-012 (Design Multi-Step Chatbot Conversation Flow)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Product Designer  
**Status:** Done

## Objective

Define the end-to-end conversation stages, user journey, and flow structure for the AI intake chatbot so the experience is clear, empathetic, and optimized for completion.

## Requirements

1. Define all conversation stages:
   - Greeting
   - Chief complaint
   - Medical history
   - Current medications
   - Allergies
   - Insurance
   - Summary and confirmation
   - Fallback / manual switch

2. Establish the expected time budget for each stage:
   - Greeting: 30 seconds
   - Chief complaint: 1-2 minutes
   - Medical history: 1-2 minutes
   - Medications: 1 minute
   - Allergies: 1 minute
   - Insurance: 1 minute
   - Summary: 30 seconds

3. Document the required UX outcomes for each stage:
   - Data capture quality
   - Minimal friction
   - Progress awareness
   - Empathetic tone

4. Create a flow diagram showing stage transitions, conditional branching, skip options, and fallback paths.

5. Align stage design to the completion target of 5-7 minutes and 85%+ completion rate.

## Implementation Steps

1. Review existing US-012 requirements and EP-002 goals.
2. Define stage objectives and user intent for each phase.
3. Draft flow diagram with stage transitions and outcomes.
4. Identify stage-level decision points and fallback triggers.
5. Validate the flow with product and UX stakeholders.
6. Iterate based on feedback.

## Acceptance Criteria

- [x] Conversation stages documented and reviewed.
- [x] Time budget assigned for each stage.
- [x] Flow diagram created with stage transitions.
- [x] Fallback and skip options mapped.
- [x] Stakeholders agree flow supports 5-7 minute session.
- [x] Flow is ready for question-level breakdown.

## Dependencies

- US-011 chatbot engine requirements
- Product goals for completion rate and satisfaction

## Validation

- [x] Stakeholder review completed.
- [x] Flow diagram passes design checklist.
- [x] No stage exceeds the scoped time budget.

## Definition of Done

- End-to-end conversation stages and flow structure fully documented.
- Design can be handed off to implementation and user research.
