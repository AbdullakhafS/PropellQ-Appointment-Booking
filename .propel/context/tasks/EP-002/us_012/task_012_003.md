# Task 003: Design Conditional Branching & Fallback Matrix

**Task ID:** task_012_003  
**User Story:** US-012 (Design Multi-Step Chatbot Conversation Flow)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Product Designer  
**Status:** Not Started

## Objective

Design the chatbot’s conditional branching rules, skip/fallback behavior, and logic for adapting questions based on patient responses to keep the intake efficient and relevant.

## Requirements

1. Create a branching matrix for core paths:
   - Symptom-driven follow-ups in chief complaint
   - Chronic condition follow-ups in medical history
   - Medication dosage/frequency follow-ups
   - Allergy reaction differentiation
   - Insurance yes/no and detail capture

2. Define skip logic for responses such as:
   - "I don’t know"
   - "I don’t remember"
   - "I prefer not to answer"
   - Off-topic responses

3. Define fallback rules:
   - If the chatbot cannot extract usable data after 2 attempts
   - If patient expresses frustration or wants help
   - If the response is nonsensical or unrelated

4. Capture how the chatbot should handle branching when multiple conditions apply.

5. Document priority ordering for follow-ups to keep flow within 5-7 minutes.

## Implementation Steps

1. Review common intake response patterns and edge cases.
2. Draft a branching matrix with triggers, conditions, and next steps.
3. Add skip and fallback rules for each stage.
4. Align the matrix with the question library.
5. Validate with product and UX leads.

## Acceptance Criteria

- [ ] Branching matrix covers all core intake paths.
- [ ] Skip logic is documented clearly.
- [ ] Fallback rules are defined for unclear and frustrated responses.
- [ ] Priority ordering ensures conversational efficiency.
- [ ] Stakeholders approve the branching logic.
