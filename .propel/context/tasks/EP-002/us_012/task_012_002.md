# Task 002: Create Detailed Question Library & A/B Variants

**Task ID:** task_012_002  
**User Story:** US-012 (Design Multi-Step Chatbot Conversation Flow)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 2  
**Owner Role:** Product Designer  
**Status:** Done

## Objective

Build a detailed question library for each chatbot stage, including A/B phrasing options, clarifying follow-ups, and conditional variants to maximize comprehension and patient comfort.

## Requirements

1. Create question sets for each stage:
   - Greeting/preamble
   - Chief complaint
   - Medical history
   - Medications
   - Allergies
   - Insurance
   - Summary/confirmation

2. Provide at least 2 A/B phrasing variants for key questions:
   - Chief complaint: "What brings you in today?" / "What's bothering you?"
   - Medical history: "Do you have any chronic conditions?" / "Are you being treated for any long-term conditions?"
   - Medications: "What medications are you currently taking?" / "What pills or treatments do you use?"
   - Allergies: "Do you have any drug allergies?" / "Are you allergic to any medications?"
   - Insurance: "Do you have insurance?" / "Will your visit be covered by insurance?"

3. Define follow-up prompts and clarifiers for common patient replies.

4. List skip/resolution phrases for when a patient doesn’t know or can’t answer.

5. Document question intent and success criteria for each variant.

## Implementation Steps

1. Draft primary questions for each stage.
2. Create A/B variants for critical questions.
3. Define follow-up questions for conditional paths.
4. Map question variants to stage outcomes.
5. Review with UX and product team.
6. Prepare a question library document for handoff.

## Acceptance Criteria

- [x] Question library completed for all stages.
- [x] At least 2 A/B variants created for each key question.
- [x] Follow-up prompts defined for conditional logic.
- [x] Skip/clarification phrases defined.
- [x] Question intent and success criteria documented.
- [x] Review by product and UX stakeholders completed.

## Dependencies

- Conversation stages from task_012_001
- Chatbot tone and voice guidelines

## Validation

- [x] Question library reviewed by 3 stakeholders.
- [x] Variant options are distinct and measurable.
- [x] Ready for A/B test planning.

## Definition of Done

- Detailed question library and A/B plan ready for implementation and testing.
