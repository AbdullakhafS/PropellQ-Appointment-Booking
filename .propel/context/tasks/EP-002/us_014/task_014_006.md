# Task 006: Validate Mode Switch Experience and Performance

**Task ID:** task_014_006  
**User Story:** US-014 (Enable Intake Mode Switching)  
**Epic:** EP-002  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** QA / Frontend Developer  
**Status:** Done

## Objective

Validate the mode switch experience across devices, confirm the switch limit enforcement, and ensure performance stays within acceptable bounds.

## Requirements

1. Functional validation:
   - Switch from chatbot → form and verify preserved data
   - Switch from form → chatbot and verify preserved data
   - Confirm switch limit enforcement (max 2 switches)
   - Confirm confirmation modal behavior

2. Performance validation:
   - Mode switch completes in under 1 second
   - Draft save/retrieve operations complete in under 150ms
   - Mode restoration after page reload completes in under 600ms

3. UX validation:
   - Switch button is visible on mobile and desktop
   - Progress indicator updates correctly after switching
   - Confirmation messaging is clear
   - No layout glitches during mode transition

4. Error validation:
   - If mapping or draft save fails, the user sees a clear error
   - The original data remains available after failure

## Implementation Steps

1. Create test cases for all mode switch scenarios.
2. Execute mode switch flows on desktop and mobile.
3. Measure performance and timing for switch operations.
4. Verify switch count enforcement and disablement.
5. Document any issues for remediation.

## Acceptance Criteria

- [x] Chatbot ↔ form switching works reliably
- [x] Switch limit is enforced and explained to users
- [x] Mode switch completes under 1 second
- [x] UI and progress indicator are correct after switching
- [x] Failure cases are handled gracefully
- [x] Tests and device checks pass

## Dependencies

- End-to-end switch integration completed
- Draft persistence and mapping logic live

## Validation

- [x] All switch scenarios pass functional tests
- [x] Performance metrics within target thresholds
- [x] No critical UX or accessibility issues

## Definition of Done

- Mode switch behavior and performance validated.
