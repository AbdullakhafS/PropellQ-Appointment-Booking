# Task 002: Implement Data Mapping Between Chatbot and Form

**Task ID:** task_014_002  
**User Story:** US-014 (Enable Intake Mode Switching)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Full-Stack Developer  
**Status:** Done

## Objective

Build conversion functions that map chatbot-extracted intake data into manual form field structures and vice versa, preserving user entries across mode switches.

## Requirements

1. Chatbot → Form mapping:
   - Map `chief_complaint` directly
   - Map `medical_history` to checklist and "other" text field
   - Map chatbot medication objects to repeating form rows
   - Map allergy objects to allergy field format
   - Map insurance info to form insurance fields

2. Form → Chatbot mapping:
   - Convert form field data into a natural-language summary payload
   - Preserve structured data for chatbot context
   - Ensure mapping can resume chatbot conversation from current form information

3. Round-trip consistency:
   - `mapFormToChatbot(mapChatbotToForm(data))` should preserve meaningful values
   - `mapChatbotToForm(mapFormToChatbot(data))` should preserve usable data for the new mode

4. Validation:
   - Handle missing or partial data gracefully
   - Normalize keys, casing, and value shapes consistently
   - Include confidence/verification metadata where available

## Implementation Steps

1. Define mapping contracts for chatbot and form formats.
2. Implement conversion functions.
3. Add unit tests for round-trip mapping and edge cases.
4. Integrate mapping into mode switch handlers.
5. Document mapping rules and fields.
