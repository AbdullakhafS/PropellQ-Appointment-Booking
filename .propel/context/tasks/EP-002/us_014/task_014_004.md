# Task 004: Implement Draft Save/Retrieve API for Partial Intake State

**Task ID:** task_014_004  
**User Story:** US-014 (Enable Intake Mode Switching)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Backend Developer  
**Status:** Done

## Objective

Build backend support to save and retrieve partial intake drafts so intake mode switches and refreshes preserve user data.

## Requirements

1. API endpoints:
   - `POST /api/intake/{appointmentId}/draft`
     - Saves draft payload with mode, data, switchCount, and metadata
   - `GET /api/intake/{appointmentId}/draft`
     - Retrieves the latest draft for the current appointment
   - `DELETE /api/intake/{appointmentId}/draft`
     - Clears draft after successful intake submission or expiration

2. Draft schema:
   - appointmentId
   - mode
   - data (intake payload)
   - switchCount
   - lastUpdated
   - expiresAt

3. Persistence:
   - Save drafts in a secure backend store
   - Enforce draft expiry after 24 hours
   - Allow drafts to be overwritten on subsequent saves

4. Authorization and validation:
   - Ensure only authorized patient or staff can access drafts
   - Validate appointment ownership and data payload shape

5. Performance:
   - Save/retrieve operations should be fast (<150ms)
   - Avoid expensive serialization; store normalized JSON if possible

## Implementation Steps

1. Define draft database schema and migration.
2. Implement API controller and service methods.
3. Add authentication/authorization checks.
4. Add validation for draft payloads.
5. Handle expiration when retrieving drafts.
6. Write integration tests for save/retrieve/delete.
