# Task 003: Build Mode Switch UI & Confirmation Modal

**Task ID:** task_014_003  
**User Story:** US-014 (Enable Intake Mode Switching)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Done

## Objective

Implement the UI controls and confirmation flow for switching between chatbot and manual form intake modes.

## Requirements

1. Mode switch control:
   - Always visible during intake
   - Displays current alternate mode: "Switch to Form" or "Switch to Chatbot"
   - Accessible on mobile and desktop

2. Confirmation modal:
   - Prompt: "Switch to [other mode]? Your responses will be preserved."
   - Buttons: [Yes, Switch] and [Cancel]
   - Display summary of preserved data if available

3. Switch limit indicator:
   - Show remaining switches, e.g. "1 switch remaining"
   - Disable switch control after limit reached
   - Use clear messaging when switching is unavailable

4. UX behavior:
   - Transition should not reload the page if possible
   - Maintain user context and progress indicator
   - Display a success toast or banner after switch

## Implementation Steps

1. Add switch button to intake shell UI.
2. Implement confirmation modal.
3. Hook switch action to mode state and mapping logic.
4. Display remaining switch count.
5. Add accessible labels and focus management.
6. Test the switch flow end-to-end.

## Acceptance Criteria

- [x] Switch button visible and accessible in both modes
- [x] Confirmation modal appears before switching
- [x] Switch count is displayed and enforced
- [x] Mode transition happens without a full page reload
- [x] Success banner confirms preserved data
- [x] UI tests validate switch behavior
