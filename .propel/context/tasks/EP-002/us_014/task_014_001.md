# Task 001: Implement Intake Mode State Management and Draft Persistence

**Task ID:** task_014_001  
**User Story:** US-014 (Enable Intake Mode Switching)  
**Epic:** EP-002  
**Priority:** CRITICAL  
**Story Points:** 2  
**Owner Role:** Frontend/Backend Developer  
**Status:** Not Started

## Objective

Implement mode state management for intake sessions and persist partial intake drafts so users can switch modes without losing data.

## Requirements

1. Mode state tracking:
   - Track current mode: `ai` or `manual`
   - Track starting mode for the session
   - Track `switchCount` and `lastSwitchedAt`
   - Enforce maximum switches (1 or 2 per intake session)

2. Draft persistence:
   - Persist partial intake data on mode switch or page refresh
   - Use frontend storage and backend draft API where appropriate
   - Draft payload includes mode, data, switchCount, and lastUpdated

3. Session persistence:
   - Save mode state to localStorage with 24-hour expiry
   - Restore state on page reload if draft exists
   - Expire stale drafts after 24 hours

4. Data safety:
   - Avoid storing sensitive PII in insecure storage; use encrypted backend when needed
   - Ensure draft restores only for the current patient and appointment

5. Performance:
   - Mode switch should complete in <1 second
   - Draft save/retrieve operations should be efficient and reliable

## Implementation Steps

1. Define intake session state shape.
2. Implement React Context or Redux store for mode state.
3. Add persistence layer for localStorage with expiry.
4. Wire draft state save on switch and on session resume.
5. Test restore on page refresh and appointment reopen.

## Acceptance Criteria

- [ ] Mode state is tracked and persisted for intake sessions
- [ ] Drafts restore correctly on reload
- [ ] Switch count increments on each switch and enforces limit
- [ ] LocalStorage expiry behavior works correctly
- [ ] Mode switch finishes in under 1 second
- [ ] Sensitive data is protected during persistence

## Dependencies

- US-011 chatbot and US-013 manual form infrastructure
- localStorage and backend draft persistence APIs

## Validation

- [ ] Confirm mode state survives reloads
- [ ] Confirm only current session data is restored
- [ ] Verify stale drafts do not restore after 24 hours

## Definition of Done

- Intake mode state and draft persistence implemented and stable.
