# Task 006: End-to-End Testing, Access Control, and Performance Validation

**Task ID:** task_016_006  
**User Story:** US-016 (Flag Unverified Insurance for Staff Review)  
**Epic:** EP-002  
**Priority:** MEDIUM  
**Story Points:** 1  
**Owner Role:** QA Engineer / Full-Stack Developer  
**Status:** Done

## Objective

Validate the full insurance review workflow from unverified flag creation through staff verification, including RBAC, audit integrity, and dashboard performance.

## Requirements

1. E2E scenarios:
   - Unverified insurance appears in dashboard after US-015 pre-check
   - Staff filters/sorts and opens patient panel
   - Staff marks single and batch records as verified
   - Audit history records every status change

2. Access control tests:
   - Billing/admin roles can access dashboard and verify actions
   - Non-staff roles are denied access

3. Performance tests:
   - Dashboard loads 1000 unverified rows in <3 seconds
   - Filter/sort interactions respond in <1 second

4. Data integrity checks:
   - Verification status updates correctly in main table
   - Audit entries contain accurate user, timestamp, method, notes

5. Regression checks:
   - No impact on intake submission flow
   - No data loss when switching filters/pages

## Implementation Steps

1. Create integration and E2E test suite for dashboard workflow.
2. Add RBAC test cases for authorized and unauthorized users.
3. Run load/performance tests on large dataset.
4. Validate audit and status consistency.
5. Document defects and remediation actions.

## Acceptance Criteria

- [x] RBAC behavior validated for all relevant roles
- [x] Performance thresholds met
- [x] Audit and status data integrity confirmed
- [x] Regression checks completed
