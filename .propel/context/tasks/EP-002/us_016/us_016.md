# US-016: Flag Unverified Insurance for Staff Review

**Epic:** EP-002 (Patient Intake & Insurance Verification)  
**Story Points:** 5  
**Priority:** HIGH  
**Status:** Not Started  
**Created:** 2026-06-17  

---

## User Story Statement

As a **billing staff member**, I want to **see a dashboard of patients with unverified insurance, prioritized by appointment date**, so that **I can proactively follow up before appointments and resolve verification issues in time to prevent claim denials**.

---

## Business Value

- **Outcome 1:** Staff can batch-verify insurance before appointments → reduces post-visit administrative work
- **Outcome 2:** Early visibility into verification issues enables faster claims submission
- **Outcome 3:** Reduces revenue leakage from claim denials due to incorrect insurance data

---

## Scope

**In Scope:**
- Dedicated "Insurance Verifications" dashboard in staff portal
- Filter/sort: by appointment date (earliest first), by insurance plan, by verification status
- Details view: Patient name, insurance name/ID, provided data, matched plan (if any), confidence score, last verification attempt
- Manual update capability: Staff can mark "verified" after contacting insurance, with notes
- Audit trail: Track who verified, when, and what data was confirmed
- Notifications: Optional email alert when new unverified insurance is flagged
- Batch actions: Mark multiple insurances as verified at once (for efficiency)

**Out of Scope:**
- Real-time insurance API lookup (Phase 2)
- Automatic re-verification after manual update
- Integration with insurance provider portals (Phase 2)

---

## Acceptance Criteria

1. **Dashboard layout:** Dedicated page accessible from staff menu: "Billing & Insurance" → "Pending Verifications"
2. **Table display:** Shows columns: [Patient Name] [Insurance Name] [Member ID] [Confidence Score] [Status] [Appointment Date] [Action]
3. **Sorting:** Default sort by appointment date (earliest first); allow sort by insurance name, confidence score; sort descending by default
4. **Filtering:**
   - By verification status: [Verified] [Unverified] [Manual Review] (checkboxes)
   - By insurance plan: Dropdown with list of plans found in unverified records
   - By date range: "Appointments in next 7/14/30 days" (quick filters)
5. **Color coding:** Unverified insurances highlighted in amber/orange; verified in green
6. **Patient details link:** Clicking patient name opens side panel with: patient contact info, appointment details, complete insurance data
7. **Manual verification:** In side panel, "Mark as Verified" button; opens form: [Verification Method] dropdown (Phone, Email, Portal), [Notes] text area, [Staff Name] (auto-populated), [Date] (auto-set to today)
8. **Confidence score display:** Show score as percentage or visual indicator (e.g., "68% match - needs review")
9. **Audit trail:** Display history of verification attempts for each patient: "John Smith verified this on 2026-06-17 by Phone"
10. **Batch actions:** Select multiple rows (checkboxes); "Mark as Verified" button applies action to all selected records
11. **Export option:** "Export as CSV" for offline processing if needed
12. **Empty state:** If no unverified insurances, display: "All insurances verified! Great job keeping data clean."

**Dashboard Component (React):**
```typescript
function InsuranceVerificationDashboard() {
  const [data, setData] = useState([]);
  const [filters, setFilters] = useState({
    status: 'unverified',
    insurance: null,
    dateRange: '7days'
  });

  useEffect(() => {
    fetchPendingInsurances(filters);
  }, [filters]);

  const handleMarkVerified = async (selected) => {
    for (const id of selected) {
      await verifyInsurance(id, {
        status: 'verified',
        verification_method: 'manual',
        notes: 'Verified by staff'
      });
    }
    // Refresh data
    await fetchPendingInsurances(filters);
  };

  return (
    <div>
      <h1>Pending Insurance Verifications</h1>
      <FilterBar filters={filters} setFilters={setFilters} />
      <DataTable 
        data={data} 
        onMarkVerified={handleMarkVerified}
        onRowClick={openPatientPanel}
      />
    </div>
  );
}
```

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Staff doesn't use dashboard → unverified insurances slip through to claims | Medium | Implement email alerts for unverified insurances; integrate with staff calendar; make verification a pre-appointment checklist |
| Dashboard becomes cluttered → staff can't find unverified records | Low | Implement smart filtering; default sort by urgency (appointment date); provide summary statistics (e.g., "12 pending") |
| Staff overwrites valid data manually → introduces errors | Low | Require phone/email confirmation before marking verified; add audit trail; allow 48-hour undo period |
| Performance: Loading 1000s of records → slow dashboard | Medium | Implement pagination; lazy-load rows; add server-side filtering; cache results |
