**Owner Role:** Backend Developer  
**Status:** Done

## Objective

Implement the backend endpoint that returns pending insurance verifications for staff, including filtering, sorting, pagination, and role-based access control.

## Requirements

1. Endpoint:
   - `GET /api/insurance/pending`
   - Query params: `status`, `insurance`, `dateRange`, `sortBy`, `sortOrder`, `page`, `pageSize`

2. Data returned per row:
   - insurance verification id
   - appointment_id, appointment_date
   - patient_id, patient_name
   - provided insurance name and member ID
   - matched plan name/id (if any)
   - confidence score
   - verification status
   - last verification timestamp

3. Sorting and filtering:
   - Default sort: appointment date ascending (earliest first)
   - Additional sort fields: insurance name, confidence score
   - Filter by status (`verified`, `unverified`, `manual_review`)
   - Filter by insurance plan
   - Quick date filters: next 7/14/30 days

4. Performance:
   - Server-side pagination required
   - Query should support 1000+ records with response <500ms
   - Index use on status, appointment_date, patient_id

5. Authorization:
   - Restrict to billing/admin roles only
   - Return 403 for unauthorized users

## Implementation Steps

1. Define request/response DTOs for pending insurance list.
2. Implement repository query with filters, sorting, and pagination.
3. Add role-based authorization middleware check.
4. Add endpoint and controller tests.
5. Validate performance on large dataset.
