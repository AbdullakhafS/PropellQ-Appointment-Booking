# Task 004: Implement Side Panel Details and Manual Verification Form

**Task ID:** task_016_004  
**User Story:** US-016 (Flag Unverified Insurance for Staff Review)  
**Epic:** EP-002  
**Priority:** HIGH  
**Story Points:** 1  
**Owner Role:** Frontend Developer  
**Status:** Not Started

## Objective

Provide a detailed patient verification side panel that enables staff to review full insurance context and mark records as verified with method and notes.

## Requirements

1. Side panel contents:
   - Patient contact details
   - Appointment details
   - Full insurance values entered by patient
   - Matched plan and confidence context
   - Last verification attempt

2. Manual verify form:
   - Status selector (verified/manual_review)
   - Verification method dropdown (phone/email/portal)
   - Notes textarea
   - Staff name auto-populated from session
   - Date auto-set to current date

3. Submission behavior:
   - Call verify endpoint on submit
   - Refresh table row and status immediately
   - Show success/failure toast messages

4. Audit history view:
   - Display previous verification attempts in timeline form
   - Include who changed status, method, notes, and timestamp

## Implementation Steps

1. Build side panel component and open-on-row-click behavior.
2. Add manual verification form and validation.
3. Integrate with verify API endpoint.
4. Render audit history within panel.
5. Test form validation and submission.

## Acceptance Criteria

- [ ] Side panel shows complete patient and insurance details
- [ ] Manual verification form submits successfully
- [ ] Table refreshes with updated status after submission
- [ ] Audit history is visible and accurate
- [ ] Form handles errors gracefully
