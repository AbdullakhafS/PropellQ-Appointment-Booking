# Task 003: Build Insurance Verifications Dashboard Table

**Task ID:** task_016_003

## Objective

Create the staff dashboard page with a data table for pending insurance verifications, including status color coding, sorting, filtering, and row actions.

## Requirements

1. Navigation and layout:
   - Add route under Billing & Insurance: Pending Verifications
   - Display summary metrics: pending count, verified today, manual review count

2. Data table columns:
   - Patient Name, Insurance Name, Member ID, Confidence Score, Status, Appointment Date, Action

3. Sorting/filtering UI:
   - Sort by appointment date (default earliest first)
   - Optional sort by insurance name and confidence score
   - Filter by status, insurance plan, and date range (7/14/30 days)

4. Visual treatment:
   - `unverified` highlighted amber/orange
   - `verified` highlighted green
   - confidence shown as percentage badge

5. Empty/loading/error states:
   - Empty: "All insurances verified! Great job keeping data clean."
   - Show loading skeleton and retry state on API errors

## Implementation Steps

1. Create dashboard route and page shell.
2. Implement filter bar and sortable table.
3. Integrate with `GET /api/insurance/pending`.
4. Add status color coding and confidence badge.
5. Add empty/loading/error states.
6. Add component tests for sort/filter behavior.
