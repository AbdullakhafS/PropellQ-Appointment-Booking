# US-059: Make Patient Dashboard Mobile-Responsive

**Story ID:** US-059  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** MEDIUM  
**Story Points:** 3  
**Owner Role:** UX / Front-End Engineer  
**Status:** Draft

## User Story

As a **patient**, I want the dashboard to work well on my phone so that I can manage appointments and health information on the go.

## Business Value

- Improves mobile adoption and convenience
- Reduces friction for patients accessing care remotely
- Supports a consistent patient experience across devices

## Scope

In scope:
- Responsive dashboard layout for small screens
- Touch-friendly buttons and navigation
- Optimized card stacking and collapse behavior

Out of scope:
- Native mobile app development
- Mobile-specific offline mode

## Acceptance Criteria

1. Given the dashboard is viewed on a phone, then the layout adapts and remains usable.
2. Given the dashboard has multiple cards, then content stacks vertically without overflow.
3. Given taps are used, then buttons and links are large enough for touch input.
4. Given the patient loads the dashboard, then it performs acceptably on mobile network conditions.

## Technical Notes

- Use responsive breakpoints at 375px, 768px, and 1024px.
- Simplify complex UI elements on mobile.
- Ensure accessibility and touch targets meet guidelines.

## Dependencies

- Patient dashboard UI from US-053
- CSS/responsive design system

## Definition of Done

- Dashboard is mobile-responsive across major breakpoints
- Touch targets and spacing are optimized for phones
- Acceptance criteria validated on sample device sizes
