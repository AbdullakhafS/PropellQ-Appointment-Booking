# US-057: Enable Document Upload from Dashboard

**Story ID:** US-057  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** HIGH  
**Story Points:** 4  
**Owner Role:** UX / Platform Engineer  
**Status:** Draft

## User Story

As a **patient**, I want to upload documents from my dashboard so that I can share insurance cards, referrals, or clinical records ahead of my appointment.

## Business Value

- Speeds pre-visit preparation and reduces manual intake effort
- Improves document availability for clinical review
- Reduces paper handling and front-desk processing

## Scope

In scope:
- Document upload interface on the patient dashboard
- Support for PDF, JPG, PNG, and DOCX
- Status display for uploaded documents and processing state

Out of scope:
- Automated OCR or clinical extraction of uploaded documents
- Large document repositories beyond patient records

## Acceptance Criteria

1. Given a patient uploads a document, then the upload is accepted and saved.
2. Given supported file types are uploaded, then upload succeeds and a confirmation appears.
3. Given an unsupported file type is provided, then the system rejects it with a clear error.
4. Given a document is uploaded, then the patient can see its status and type in the dashboard.

## Technical Notes

- Validate file type and size on the client and server.
- Store documents securely with PHI protections.
- Show upload progress and completion feedback.

## Dependencies

- Document storage and API from EP-003/EP-DATA-001
- Authentication and authorization from EP-005

## Definition of Done

- Document upload feature implemented on patient dashboard
- Supported file types accepted and status displayed
- Error handling for unsupported files and failures
- Acceptance criteria verified with test uploads
