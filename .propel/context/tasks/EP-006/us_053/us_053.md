# US-053: Build Patient Dashboard UI

**Story ID:** US-053  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Front-End Engineer  
**Status:** Draft

## User Story

As a **patient**, I want a personal dashboard so that I can see upcoming appointments, recent activity, and my health profile information in one place.

## Business Value

- Improves patient engagement and self-service adoption
- Reduces support calls by centralizing important information
- Increases transparency for appointment status and care readiness

## Scope

In scope:
- Patient dashboard landing page with cards for appointments, profile, notifications, and documents
- Responsive layout for desktop and mobile
- Navigation to key patient portal actions

Out of scope:
- Deep analytics for admins
- Full clinical charting interfaces

## Acceptance Criteria

1. Given a patient is logged in, then they can access the dashboard from the patient portal.
2. Given the dashboard loads, then it displays upcoming appointments, recent activity, and health profile summary.
3. Given the patient is on mobile, then the dashboard layout adapts to smaller screens.
4. Given the dashboard data updates, then refreshed values appear without a full page reload.

## Technical Notes

- Use a modular component structure for dashboard cards.
- Lazy-load non-critical sections to improve perceived performance.
- Ensure dashboard is accessible and responsive.

## Dependencies

- Patient appointment and profile APIs from EP-001/EP-003
- Authentication from EP-005
- Front-end design system

## Definition of Done

- Patient dashboard UI built and rendered correctly
- Dashboard data sections structured for key patient actions
- Responsive behavior validated on common breakpoints
- Acceptance criteria verified with sample user flows
