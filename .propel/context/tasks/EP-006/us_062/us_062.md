# US-062: Display Average Wait Time Metrics

**Story ID:** US-062  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** MEDIUM  
**Story Points:** 3  
**Owner Role:** Product / Data Analyst  
**Status:** Draft

## User Story

As an **admin operations manager**, I want average wait time metrics so that I can gauge clinic efficiency and patient flow performance.

## Business Value

- Helps measure patient experience and workflow speed
- Supports operational improvements in staffing and resources
- Enables monitoring of queue and appointment throughput

## Scope

In scope:
- Average check-in to appointment start wait time
- Time-in-queue metrics for same-day and walk-in appointments
- Trend visualization over selectable date ranges

Out of scope:
- Predictive wait time forecasting
- Patient-specific wait time estimation

## Acceptance Criteria

1. Given the admin views the dashboard, then the average wait time KPI is visible.
2. Given the dashboard loads, then wait time metrics and trends are displayed.
3. Given a filter is applied, then wait time data updates for the selected period.
4. Given a high wait time threshold is exceeded, then the dashboard shows an alert or warning.

## Technical Notes

- Use arrival and appointment start timestamps for calculations.
- Keep metrics aggregated for performance.
- Display both average and percentile values if possible.

## Dependencies

- Check-in and appointment event data from EP-004
- Data aggregation or analytics services

## Definition of Done

- Wait time metrics display on admin dashboard
- Filters affect the displayed data correctly
- Alert/warning state appears when thresholds are exceeded
- Acceptance criteria validated with sample analytics data
