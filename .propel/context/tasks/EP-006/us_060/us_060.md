# US-060: Build Admin Operational Dashboard UI

**Story ID:** US-060  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** HIGH  
**Story Points:** 5  
**Owner Role:** UX / Product Engineer  
**Status:** Draft

## User Story

As an **admin operations manager**, I want an operational dashboard so that I can monitor appointment KPIs, wait times, and utilization metrics at a glance.

## Business Value

- Enables proactive operational oversight
- Helps identify performance bottlenecks quickly
- Supports data-driven staffing and scheduling decisions

## Scope

In scope:
- Admin dashboard landing page with KPI cards and charts
- Summary metrics for appointment utilization, wait times, and no-show rates
- Filtering by date range, provider, and location

Out of scope:
- Full BI reporting platform
- Historical forecasting beyond current trends

## Acceptance Criteria

1. Given an admin logs in, then they can navigate to the operational dashboard.
2. Given the dashboard loads, then KPI cards and charts are visible.
3. Given filters are applied, then the displayed metrics update accordingly.
4. Given the dashboard is refreshed, then the latest available data is shown.

## Technical Notes

- Build reusable chart and KPI components.
- Keep data refresh cadence efficient to avoid heavy loads.
- Provide drill-down links to source reports if needed.

## Dependencies

- Analytics APIs from EP-001, EP-004, EP-006
- Admin authentication and authorization from EP-005
- Dashboard UI framework

## Definition of Done

- Admin dashboard UI implemented with top-level KPIs
- Filters and refresh behavior working
- Metrics load correctly and appear readable
- Acceptance criteria validated with admin sample data
