# US-069: Implement CSV Export for Reports

**Story ID:** US-069  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** MEDIUM  
**Story Points:** 3  
**Owner Role:** Product / Platform Engineer  
**Status:** Draft

## User Story

As an **admin operations manager**, I want to export dashboard reports as CSV so that I can share data with other systems and teams.

## Business Value

- Supports offline analysis and reporting
- Enables easy data sharing with stakeholders
- Provides a simple export path for operational metrics

## Scope

In scope:
- CSV export for core dashboard tables and KPI summaries
- Export button on admin dashboard views
- Proper formatting for dates, numeric metrics, and labels

Out of scope:
- Full data warehouse export workflows
- Scheduled or automated exports

## Acceptance Criteria

1. Given the admin selects export, then a CSV file downloads successfully.
2. Given the current filters are applied, then the CSV matches the displayed data.
3. Given the export is triggered, then the file contains column headers and formatted rows.
4. Given a large dataset is exported, then the response remains performant.

## Technical Notes

- Generate CSV server-side from filtered analytics queries.
- Include clear headers and consistent formatting.
- Validate exported data against displayed dashboard metrics.

## Dependencies

- Analytics APIs and report data services
- Dashboard filtering logic

## Definition of Done

- CSV export available on admin dashboard
- Exported file matches filtered dashboard data
- Acceptance criteria validated with sample exports
