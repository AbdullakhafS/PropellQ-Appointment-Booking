# US-065: Display Insurance Verification Status

**Story ID:** US-065  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** MEDIUM  
**Story Points:** 3  
**Owner Role:** Product / Data Analyst  
**Status:** Draft

## User Story

As an **admin operations manager**, I want to see insurance verification status metrics so that I can monitor claims readiness and identify patients needing follow-up.

## Business Value

- Helps reduce claim denials from unverified insurance
- Enables staff to prioritize verification work
- Improves operational readiness for appointments

## Scope

In scope:
- Insurance verification status card with verified, pending, and failed counts
- Filterable view by date range, provider, and status
- Trend visualization for verification performance

Out of scope:
- Detailed insurance eligibility adjudication
- Claims processing workflows

## Acceptance Criteria

1. Given the admin views the dashboard, then insurance verification metrics are visible.
2. Given the dashboard filters are applied, then verification data updates accordingly.
3. Given verification issues exist, then the corresponding count is highlighted.
4. Given a status filter is selected, then only matching records are shown.

## Technical Notes

- Use insurance pre-check status from EP-002.
- Aggregate verification data for performant dashboard display.
- Distinguish verified, pending, and failed states clearly.

## Dependencies

- Insurance verification data from EP-002
- Analytics or reporting service

## Definition of Done

- Insurance verification metrics display on admin dashboard
- Filters and status breakdown function correctly
- Acceptance criteria validated with sample status data
