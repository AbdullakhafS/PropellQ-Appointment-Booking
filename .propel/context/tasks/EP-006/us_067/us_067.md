# US-067: Implement Dashboard Auto-Refresh (5 min)

**Story ID:** US-067  
**Epic:** EP-006 (Patient Portal & Analytics)  
**Priority:** MEDIUM  
**Story Points:** 3  
**Owner Role:** Platform Engineer  
**Status:** Draft

## User Story

As an **admin**, I want dashboards to auto-refresh every 5 minutes so that KPI data remains current without manual reloading.

## Business Value

- Keeps operational metrics up to date
- Reduces stale data exposure
- Improves awareness for time-sensitive staff decisions

## Scope

In scope:
- Auto-refresh mechanism for patient and admin dashboards
- Configurable refresh interval or default 5-minute cadence
- Minimal UI disruption during refresh

Out of scope:
- Real-time streaming for every KPI
- Custom refresh settings beyond the default cadence

## Acceptance Criteria

1. Given the dashboard is open, then data refreshes every 5 minutes automatically.
2. Given a refresh occurs, then current metrics update without a full page reload.
3. Given data is loading, then a subtle loading indicator is shown.
4. Given the dashboard is inactive, then refresh throttles or pauses to avoid unnecessary calls.

## Technical Notes

- Use polling or websocket-driven refresh depending on architecture.
- Keep refresh payloads scoped to changed data.
- Provide a visible last-updated timestamp.

## Dependencies

- Dashboard data APIs
- Front-end refresh control logic

## Definition of Done

- Auto-refresh implemented on dashboard pages
- Data updates without full page reloads
- Loading status is visible during refresh
- Acceptance criteria validated with live dashboard data
